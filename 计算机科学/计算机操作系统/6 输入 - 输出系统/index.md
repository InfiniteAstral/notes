# 第 6 章 输入 / 输出系统

到这里，操作系统课程中的几类核心资源已经基本都出现了：

- 处理机
- 存储器
- 文件
- 外部设备

如果说前几章更多在讨论“程序如何运行”“数据如何保存”，那么这一章讨论的就是另一个非常现实、也非常容易成为性能瓶颈的问题：

**系统怎样和外部世界交换信息。**

所谓输入 / 输出系统，最直观地看，就是：

- 键盘输入如何进入系统
- 显示器、打印机怎样输出结果
- 磁盘、U 盘、总线、控制器怎样协同工作
- 为什么 I/O 常常拖慢整个系统
- 操作系统如何通过中断、DMA、缓冲、调度等机制把 I/O 做得更高效

这一章的内容既有硬件色彩，又有操作系统管理色彩。学的时候不要把它当成纯“设备常识”，也不要把它当成纯“驱动细节”，而是要抓住一条更核心的主线：

**I/O 系统的任务，是把性能差异很大的 CPU、内存和外设组织起来，让它们能高效、统一、可控地协同工作。**

::: info 学习提示

本章最容易混淆的概念主要有以下几组：

- **字符设备、块设备**
- **独占设备、共享设备、虚拟设备**
- **程序直接控制、中断驱动、DMA、通道方式**
- **设备控制器、端口、寄存器、设备驱动程序**
- **单缓冲、双缓冲、环形缓冲、缓冲池**
- **设备分配、磁盘调度、SPOOLing**

学习时建议一直追问：

- 这个机制是在解决“谁来控制设备”，还是“怎样减少等待”，还是“怎样提高吞吐量”？
- 这个方案到底减轻了 CPU 的什么负担？

:::

## 6.1 概述

### 为什么 I/O 常常成为系统瓶颈

很多初学者容易把“CPU 很快”误认为“系统整体就很快”，但真实系统里，I/O 往往才是限制性能的关键环节。

原因主要有几类：

- I/O 设备通常比 CPU 慢得多
- CPU 与 I/O 之间的速度差距会随着处理器提升而进一步拉大
- I/O 请求频繁时，会带来更多中断和进程切换
- 外设种类多、特性杂、并发关系复杂，导致操作系统实现也更庞大

因此，I/O 管理不仅是“把数据搬进搬出”，它还直接影响：

- 系统吞吐量
- CPU 利用率
- 响应速度
- 文件系统性能

### 输入 / 输出系统在操作系统中的位置

I/O 系统并不是一个孤立模块，它和操作系统的很多部分都有强关联：

- 和 **进程管理** 相关，因为 I/O 等待会导致阻塞与唤醒
- 和 **存储管理** 相关，因为 DMA、缓冲区、缓存都要用到内存
- 和 **文件系统** 相关，因为大量文件访问最终都会落实为磁盘 I/O
- 和 **设备管理** 相关，因为要完成分配、驱动、控制和回收

可以把 I/O 子系统理解为：

**CPU、内存、文件系统与外设之间的一座桥。**

### 设备的分类

理解设备管理之前，先要知道设备能从哪些角度分类。

#### 按传输速率分类

- **低速设备**
- **中速设备**
- **高速设备**

这种分类主要影响：

- 设备管理方式
- 缓冲需求
- 中断频率

#### 按信息交换单位分类

##### 字符设备

字符设备通常以字符为单位传输信息，特点通常是：

- 速率较低
- 常采用中断驱动
- 信息按字符流方式处理

典型例子如：

- 键盘
- 终端
- 串口类设备

##### 块设备

块设备以块为单位传输信息，特点通常是：

- 速率较高
- 可随机访问任意块
- 常配合 DMA 使用

典型例子如：

- 磁盘
- 固态盘

#### 按资源管理方式分类

- **独占设备**
- **共享设备**
- **虚拟设备**

这组分类与后面的设备分配方式联系非常紧密。

#### 按从属关系分类

- **系统设备**
- **用户设备**

### 设备管理的目标

设备管理通常围绕三个核心目标展开：

- **实现设备独立性**
- **提高设备利用率**
- **实现设备的统一管理**

其中，“设备独立性”尤其重要。

#### 什么是设备独立性

设备独立性指：

**用户和上层软件尽量不必关心具体使用的是哪一台物理设备，而是通过统一逻辑接口访问设备。**

例如：

- 打印到哪台打印机
- 从哪个终端读入
- 使用哪个磁盘分区

这些细节最好由系统完成映射和调度，而不是交给用户程序硬编码。

### 设备管理的功能

设备管理通常需要完成以下工作：

- 监视系统中设备的状态
- 进行设备分配与回收
- 实施 I/O 控制

进一步展开后，还包括：

- 设备独立性支持
- 驱动程序调用
- 中断处理
- 缓冲管理
- 调度与优化

### 设备管理的数据结构

和进程管理有 `PCB`、文件管理有 `FCB` 一样，设备管理也依赖专门的数据结构。

#### DCB：设备控制块

**设备控制块（DCB, Device Control Block）** 是操作系统用来描述和管理某台设备的数据结构。

其内容通常包括：

- 设备标识符
- 设备属性
- I/O 总线地址
- 设备状态
- 等待队列指针

从功能上看，`DCB` 就是设备管理里的“档案卡”。

## 6.2 I/O 系统的硬件基础

I/O 系统不仅是软件问题，也离不开底层硬件组织。理解控制器、接口、端口和寄存器，是理解后面各种控制方式的前提。

### 设备与设备控制器

一个外设通常可以分为两部分：

- **物理部分**：真正执行机械或电气动作的设备本体
- **电子部分**：设备控制器

控制器负责把主机与设备连接起来。

### 设备控制器做什么

设备控制器通常承担以下任务：

- 地址译码
- 接收主机发来的数据与控制信号
- 向主机返回状态或结果信息
- 完成数字信号与设备可识别信号之间的转换
- 提供内部硬件缓冲和必要的数据加工功能

可以把它理解为：

**主机和设备之间的“翻译层 + 协调层”。**

### 设备接口

操作系统向控制器发命令、取状态、收发数据时，通常都要通过设备接口完成。

常见接口部件包括：

- **设备识别线路**
- **命令寄存器**
- **数据缓冲寄存器**
- **控制寄存器**
- **状态寄存器**

操作系统的典型动作通常是：

- 向命令寄存器写入命令
- 向数据寄存器写入或读取数据
- 从状态寄存器读取设备状态

### 端口编址方法

主机访问 I/O 端口时，通常有两种编址方式。

#### 内存映像编址

也叫 **内存映像 I/O**。

其思路是：

- 把 I/O 端口地址空间和内存地址空间统一编址

##### 优点

- 凡是能操作存储器的指令，通常都可用于 I/O 端口
- 不需要专门的 I/O 指令
- 端口可使用较大的地址空间

##### 缺点

- 会占用一部分内存地址空间

#### I/O 独立编址

也叫专用 I/O 指令方式。

其思路是：

- I/O 端口拥有独立于内存的地址空间
- 主机通过专门的 I/O 指令访问端口

##### 优点

- 不占用内存地址空间
- 程序设计时更容易区分“这是访存”还是“这是访问设备”

##### 缺点

- 可用的专用指令类型通常更少
- 灵活性相对较差

## 6.3 I/O 控制方式

当进程发出 I/O 请求后，最核心的问题是：

**谁来控制这次数据传输？CPU 要参与到什么程度？**

围绕这个问题，操作系统和硬件发展出了几种典型 I/O 控制方式。

### 控制方式的演进主线

整体演进方向可以概括为：

- 让 CPU 少做重复等待
- 让设备和 CPU 尽量并行
- 让数据传输尽量批量化

常见控制方式包括：

- **程序直接控制方式**
- **中断驱动方式**
- **DMA 方式**
- **通道方式**
- **外围处理机方式**

### 程序直接控制方式

#### 基本思想

CPU 向设备发出命令后，不断轮询设备状态，直到设备准备好，再进行数据传输。

这也常叫：

- 循环查询方式
- 忙等方式

#### 优点

- 实现最简单
- 控制逻辑直观

#### 缺点

- 会浪费大量 CPU 时间
- CPU 必须一直盯着设备
- 并行性差

因此它只适用于：

- 简单系统
- 低性能要求场景

### 中断驱动方式

#### 基本思想

CPU 发出 I/O 命令后，不必持续轮询，而是继续执行别的工作。等设备准备好时，由控制器发出中断，通知 CPU 前来处理。

#### 优点

- CPU 利用率比忙等方式高得多
- 设备和 CPU 可并行工作

#### 缺点

- 如果数据量大，而设备每次缓冲区满都要中断一次，那么中断次数会很多
- 设备数量多时，中断开销会显著上升

### DMA 方式

#### DMA 在解决什么问题

中断方式虽然减轻了 CPU 的等待，但数据传输本身仍常由 CPU 参与。DMA 进一步解决的是：

**让设备和内存之间的数据成块传输时，尽量不再由 CPU 一次次搬运。**

#### 基本思想

DMA 控制器直接在内存和设备之间传送一整块数据。

CPU 的工作通常变成：

1. 设置传送的内存起始地址
2. 设置传送字节数
3. 启动 DMA
4. 在整块传输结束后处理中断

#### DMA 方式的过程

以输入为例，通常是：

1. 进程提出输入请求
2. 操作系统设置 DMA 的地址寄存器和计数寄存器
3. 设备开始工作，原进程可阻塞，系统调度其他进程运行
4. 一批数据传输完成后，DMA 再发中断
5. 中断处理程序唤醒原进程

#### DMA 与中断方式的主要区别

两者都可能在结束时通知 CPU，但核心区别有两点：

- **中断方式**：通常每次缓冲寄存器满都可能中断一次
- **DMA 方式**：通常在整块数据传输完成后才中断一次

另外：

- **中断方式** 的数据传送更多由 CPU 控制
- **DMA 方式** 的数据传送主要由 DMA 控制器直接完成

### 通道方式

#### 为什么还要有通道

当系统很大、外设很多、I/O 任务更复杂时，DMA 仍然不够灵活。于是又引入了 **通道（Channel）**。

通道可以理解为：

**专门负责 I/O 控制的一种处理部件。**

它在 CPU 控制下工作，但能相对独立地执行通道程序，完成成批数据交换。

#### 通道程序与子通道

通道程序由一系列通道指令构成。

一个通道可以分时执行多道通道程序，其中每道通道程序通常控制一台外设，因此也常称为 **子通道**。

#### 通道的种类

常见通道类型包括：

- **字节多路通道**
- **数据选择通道**
- **数据多路通道**

不同类型适合不同设备特征和吞吐要求。

#### 通道的工作过程

典型过程是：

1. 进程通过系统调用提出 I/O 请求
2. 操作系统分配通道和设备
3. 根据请求生成通道程序并放入内存
4. CPU 把起始地址送入通道首址寄存器
5. 通道按通道指令独立执行 I/O 操作
6. 通道程序完成后，再中断通知 CPU

#### 通道的意义

与 DMA 相比，通道更像是：

- 不只是“搬一块数据”
- 而是“按一段程序组织整批 I/O 操作”

这进一步减轻了 CPU 负担。

### 外围处理机方式

外围处理机方式比通道更进一步，它使用专门的处理部件完成更完整的外围操作控制。

从演进趋势上看，它体现的是：

**把越来越多的 I/O 控制工作从 CPU 主体中分离出去。**

### 控制方式的比较

可以粗略理解为：

| 控制方式     | CPU 参与程度 | 并行性 | 特点             |
| ------------ | ------------ | ------ | ---------------- |
| 程序直接控制 | 很高         | 很差   | 简单但浪费 CPU   |
| 中断驱动     | 较高         | 较好   | 提高 CPU 利用率  |
| DMA          | 更低         | 更好   | 适合成块传输     |
| 通道方式     | 更低         | 更强   | 适合大型复杂 I/O |

## 6.4 I/O 系统的软件组成

I/O 系统不是只有硬件控制器，操作系统内部还需要一套分层的软件结构，把“统一接口”和“具体设备差异”隔离开来。

### 为什么 I/O 软件通常分层

不同设备在以下方面差异极大：

- 速度
- 数据单位
- 操作方式
- 出错条件

如果应用程序直接处理这些差异，系统会极难维护。因此 I/O 软件通常按层次组织，让上层看到统一接口，下层处理设备细节。

### 常见的软件层次

典型的 I/O 软件结构通常可分为几层：

- **用户层 I/O 软件**
- **与设备无关的 I/O 软件**
- **设备驱动程序**
- **中断处理程序**
- **硬件控制器**

其中最关键的两层是：

- **设备独立软件**
- **设备驱动程序**

### 设备独立的软件

#### 它解决什么问题

设备独立软件的目标，是尽量把上层使用方式从具体硬件差异中解放出来。

它通常负责：

- 统一命名和逻辑设备管理
- 提供一致的调用接口
- 进行缓冲管理
- 进行错误处理的一部分工作
- 完成逻辑设备名到物理设备名的映射

#### 逻辑设备表

实现设备独立性时，一个很重要的数据结构是 **逻辑设备表**。

它的作用是：

- 维护逻辑设备名到物理设备名的映射关系

这样，上层程序只需要使用逻辑名字，而不必写死某台物理设备。

### 设备驱动程序

#### 驱动程序是什么

设备驱动程序是最贴近具体设备的软件层，负责把操作系统的抽象请求转换成特定控制器能够识别和执行的命令序列。

#### 驱动程序做什么

通常包括：

- 启动设备
- 设置寄存器
- 处理设备特定时序
- 配合中断处理完成传输
- 返回状态和错误信息

可以把驱动程序理解为：

**设备的专用翻译器。**

### 中断处理程序

当设备完成操作或发生异常后，通常会通过中断通知 CPU。中断处理程序则负责：

- 接收中断
- 保存和恢复现场
- 判断中断来源
- 更新设备和请求状态
- 唤醒等待进程

## 6.5 缓冲技术

缓冲是 I/O 系统里极其重要的一类技术。它不一定改变设备本身速度，却常常能显著改善系统整体表现。

### 为什么要引入缓冲

缓冲技术主要是为了解决以下几个问题：

- 缓和 CPU 与 I/O 设备速度不匹配的矛盾
- 减少对 CPU 的中断频率
- 放宽对 CPU 响应时间的严格要求
- 提高 CPU 与设备之间的并行程度

一句话概括：

**缓冲不是让设备本身变快，而是让系统“更少因为速度不匹配而卡住”。**

### 缓冲技术的两大类

- **硬件缓冲器**
- **软件缓冲技术**

#### 硬件缓冲器

位于设备控制器内部，容量通常较小，但速度快。

#### 软件缓冲

由操作系统在内存中设置缓冲区，并提供相应管理机制。

常见形式包括：

- 单缓冲
- 双缓冲
- 环形缓冲
- 缓冲池

### 单缓冲

#### 基本思想

当进程发出 I/O 请求时，操作系统为其分配一个缓冲区。

对于输入：

- 设备先把数据送入缓冲区
- 操作系统再把数据从缓冲区交给进程

对于输出：

- 进程先把数据送入缓冲区
- 操作系统再把数据从缓冲区输出到设备

#### 作用

单缓冲能在一定程度上平滑设备与进程之间的数据交换，但它的并行能力有限。

通常不能从根本上显著加快整个进程执行速度，只是减少了直接耦合带来的问题。

### 双缓冲

#### 基本思想

设置两个缓冲区 `buf1` 和 `buf2`，交替使用。

例如输入时：

- 设备向 `buf1` 填数据时
- 进程可以从 `buf2` 取数据

或者反过来。

#### 优势

- 提高设备与进程之间的并行性
- 减少等待时间
- 在输入输出通信中非常实用

### 环形缓冲

#### 基本思想

在主存中分配一组大小相等的缓冲区，并把它们逻辑上首尾相接，形成环。

系统通常维护：

- `IN` 指针
- `OUT` 指针

来分别表示写入位置和读出位置。

#### 适合什么场景

环形缓冲适合：

- 连续流式输入输出
- 生产者和消费者并行工作

它的本质思想是：

**用多个缓冲区循环复用，而不是只靠一个或两个缓冲区。**

### 缓冲池

#### 基本思想

缓冲池由内存中一组大小相等的缓冲区组成，属于系统共享资源，由操作系统统一管理。

这些缓冲区通常会按状态组织成不同队列，例如：

- **空闲缓冲区队列**
- **装满输入数据的缓冲区队列**
- **装满输出数据的缓冲区队列**

#### 优势

- 适应性更强
- 更适合多进程、多设备并发 I/O
- 资源复用能力更高

## 6.6 磁盘驱动调度与优化

磁盘是最典型、也最重要的块设备之一。大量文件访问最终都会落实到磁盘上，因此磁盘 I/O 速度直接影响文件系统性能。

### 6.6.1 磁盘概述

#### 磁盘的基本结构

硬盘通常可分为：

- **固定头磁盘**
- **移动头磁盘**

固定头磁盘中，每个磁道都有对应磁头，切换磁道不需要机械移动，速度快但成本高。

移动头磁盘中，一个盘面通常只有一个磁头，访问不同磁道时要移动磁头，速度较慢但成本低。

#### 柱面、磁头、扇区

磁盘中常见的几个基本术语是：

- **磁道**
- **磁头**
- **扇区**
- **柱面**

其中：

- 每个盘面上有若干磁道
- 每个盘面有磁头
- 每个磁道被划分为若干扇区
- 所有盘面上同一磁道号的位置组成一个柱面

磁盘物理地址通常可写成：

- 柱面号
- 磁头号
- 扇区号

#### 磁盘访问时间由什么组成

磁盘访问通常由三部分时间构成：

- **寻道时间 `Ts`**：磁头移动到目标磁道
- **旋转延迟时间 `Tr`**：等待目标扇区转到磁头下
- **数据传输时间 `Tt`**：实际传送数据

其中最值得注意的是：

- 数据传输时间往往最短
- 机械移动相关的寻道和旋转延迟才是主要开销

因此，要提高磁盘速度，重点往往不在“传输得多快”，而在：

- **减少磁头移动**
- **减少等待旋转**

### 6.6.2 磁盘调度算法

当多个访盘请求同时等待时，系统需要决定：

**先服务哪个请求，后服务哪个请求。**

磁盘调度的目标通常有两个：

- **公平**：请求应在有限时间内得到满足
- **高效**：尽量减少机械运动造成的时间浪费

#### FCFS：先来先服务

##### 基本思想

按请求到达顺序依次服务。

##### 优点

- 简单
- 公平

##### 缺点

- 效率较低
- 磁头可能在磁盘两端频繁来回移动

#### SSTF：最短寻道时间优先

##### 基本思想

总是优先服务距离当前磁头最近的请求。

##### 优点

- 平均寻道时间通常比 FCFS 更好
- 磁头移动总距离明显减少

##### 缺点

- 可能导致某些远端请求长期等待
- 会出现“局部最优但整体不公平”的问题

#### SCAN：扫描算法

##### 基本思想

磁头沿一个方向移动，在移动过程中依次服务沿途遇到的请求；走到该方向尽头后，再反向扫描。

这很像电梯上下运行，因此也叫：

- **电梯算法**

##### 优点

- 比 SSTF 更公平
- 能避免长期饥饿

##### 局限

- 当请求分布较均匀时，靠近回头端的请求等待时间可能偏长

#### CSCAN：循环扫描算法

##### 基本思想

磁头始终按一个固定方向扫描。

当到达一端后：

- 立即快速返回另一端
- 返回过程中不为请求服务

然后重新开始下一轮扫描。

##### 优点

- 等待时间分布更均匀
- 对不同位置请求更公平

##### 代价

- 返回过程本身不服务请求，可能带来额外空转

#### 调度算法如何选择

不同算法各有适用场景：

- **FCFS**：磁盘负担轻时可用，简单公平
- **SSTF**：简单有效，实际系统中较常见
- **SCAN**：更适合磁盘负担较重的系统
- **CSCAN**：在追求更均匀等待时间时更有优势

### 6.6.3 提高磁盘 I/O 速度的方法

除了调度算法外，还可以从多个方面优化磁盘 I/O。

#### 方法一：磁盘高速缓存

##### 基本思想

由于磁盘速度比内存低很多，因此可以在内存中开辟一部分区域作为 **磁盘高速缓存**。

这样，频繁访问的数据不必每次都真的去访盘。

##### 两种常见组织方式

- 单独划出一块内存作为磁盘缓存
- 把未利用的内存空间组织成缓冲池，与分页系统和磁盘 I/O 共享

##### 数据交付方式

缓存中的数据交给请求者时，常见有两种方式：

- **数据交付**：把数据拷贝到进程空间
- **指针交付**：把指向缓存中数据的指针交给进程

##### 置换算法与周期性写回

当缓存已满时，需要淘汰部分盘块，常见算法包括：

- `LRU`
- `LFU`

如果缓存中有被修改但还没写回磁盘的数据，就会有一致性风险，因此系统通常还会采用：

- **周期性写回**

也就是定期强制把已修改块写回磁盘。

#### 方法二：优化数据分布

磁盘访问速度不仅取决于调度，也与数据本身在磁盘上的布局有关。

常见优化包括：

- 让文件物理块尽量连续分配
- 适当增大物理块大小，减少磁头移动
- 优化索引结点的位置
- 对磁道进行分组，让索引和数据布局更合理

#### 其他方法

##### 提前读

如果系统发现访问模式常常是顺序的，就可以在读取当前块时顺便预读下一块。

这叫：

- **提前读**
- **预取**

##### 延迟写

当缓存中的盘块被修改后，不一定立刻写回磁盘，而是先打上延迟写标志，等：

- 被替换时
- 周期性写回时

再真正写回。

这样能减少频繁写盘开销。

##### 虚拟盘

虚拟盘又叫：

- **RAM 盘**

它利用内存模拟磁盘，特点是：

- 内容对用户可见
- 速度很高

要注意它和磁盘高速缓存不同：

- **虚拟盘**：用户可见，内容由用户控制
- **磁盘缓存**：用户不可见，内容由系统控制

## 6.7 设备分配与 SPOOLing

I/O 管理不仅要控制数据怎么传，还要决定：

**设备分给谁、何时分、如何避免冲突和浪费。**

### 设备分配所需的数据结构

设备分配通常需要多种表结构协同工作，例如：

- **系统设备表 `SDT`**
- **控制器控制表 `COCT`**
- **通道控制表 `CHCT`**
- **逻辑设备表**

它们分别用于描述：

- 有哪些设备
- 设备挂在哪个控制器上
- 控制器又连接到哪个通道
- 逻辑设备名与物理设备名怎样映射

### 设备分配方式

#### 静态分配

##### 基本思想

在作业运行之前，一次性把所需全部设备分配好，并一直占用到作业结束。

##### 优点

- 不容易发生死锁

##### 缺点

- 设备利用率较低

#### 动态分配

##### 基本思想

进程在运行过程中按需申请设备，用完立即释放。

##### 优点

- 更有利于提高设备利用率

##### 缺点

- 可能引发死锁
- 管理更复杂

### 设备分配算法

常见算法比较直接，例如：

- **先请求先服务**
- **高优先级优先服务**

这些算法的选择，本质上是在平衡：

- 公平性
- 设备利用率
- 重要任务优先级

### 设备分配技术

根据设备特性，系统通常采用不同分配技术。

#### 独享分配

适用于独占设备。

当进程申请设备后，系统把设备独占地分配给它，直到释放为止。

这适用于：

- 打印机
- 某些终端类设备

#### 共享分配

适用于共享设备。

设备按动态方式分配，请求者使用完后立即释放。

典型共享设备如：

- 磁盘
- 某些磁带或磁鼓系统

#### 虚拟分配

##### 什么是虚拟设备

为了提高效率，系统可以在高速共享设备上模拟低速独占设备的功能，这种技术对应的对象就是 **虚拟设备**。

##### 基本思想

当用户申请独占设备时，系统不一定把真实设备直接交给它，而是：

- 先分配共享设备上的一部分存储空间
- 程序与这块空间交换信息
- 系统再在适当时机把这些信息送往真正设备

这样，低速独占设备就被高速共享资源“虚拟化”了。

#### SPOOLing 系统

虚拟设备思想最经典的实现之一，就是 **SPOOLing**。

##### 什么是 SPOOLing

`SPOOLing` 是：

`Simultaneous Peripheral Operations On-Line`

常译为：

- 外围设备联机并行操作
- 假脱机技术

它的核心思想是：

**在联机系统中，用磁盘等共享存储空间来模拟脱机批处理中的输入输出组织方式。**

##### SPOOLing 的组成

典型 SPOOLing 系统通常包括：

- **输入井**
- **输出井**
- **输入缓冲区**
- **输出缓冲区**
- **输入进程**
- **输出进程**

##### SPOOLing 的工作原理

典型过程是：

1. 作业执行前，程序和数据先进入输入井
2. 作业运行时，从输入井中取数据
3. 输出时不必直接启动慢速外设，而是先写入输出井
4. 作业运行完后，再由输出进程统一把结果送到实际设备

##### SPOOLing 的特点

SPOOLing 有三个非常重要的优点：

- **提高了 I/O 速度**
- **把独占设备改造成共享设备**
- **实现了虚拟设备功能**

这也是为什么打印系统、批量输出系统等场景中，SPOOLing 思想非常经典。

## 本章小结

学完本章后，你至少应该能把下面这些问题讲清楚：

1. 为什么 I/O 往往会成为整个系统的性能瓶颈？
2. 字符设备和块设备有什么区别？独占设备、共享设备、虚拟设备分别适合怎样的管理方式？
3. 设备独立性是什么意思？为什么逻辑设备名到物理设备名的映射很重要？
4. 设备控制器、接口、端口、寄存器分别在 I/O 硬件体系里扮演什么角色？
5. 内存映像编址和 I/O 独立编址有什么不同？
6. 程序直接控制、中断驱动、DMA、通道方式分别是谁在主导数据传输？它们的优缺点是什么？
7. 为什么说 DMA 相比中断驱动方式进一步减轻了 CPU 负担？
8. 缓冲技术为什么能提高系统整体效率？单缓冲、双缓冲、环形缓冲、缓冲池各解决什么问题？
9. 磁盘访问时间由哪些部分组成？为什么优化重点常常落在寻道和旋转延迟上？
10. FCFS、SSTF、SCAN、CSCAN 这几种磁盘调度算法各有什么特点？
11. 磁盘高速缓存、提前读、延迟写、优化数据分布、虚拟盘分别是在从哪个角度提升磁盘 I/O 性能？
12. 设备为什么既可以静态分配，也可以动态分配？两者的利弊是什么？
13. 独享分配、共享分配、虚拟分配分别对应什么样的设备与场景？
14. SPOOLing 的本质是什么？它为什么能把独占设备改造成逻辑上的共享设备？

::: details 参考答案提纲

1. 因为外设通常比 CPU 慢很多，而且 I/O 会带来中断、阻塞、唤醒和上下文切换，所以 I/O 常常限制系统整体性能。
2. 字符设备按字符流交换信息，速率较低；块设备按块交换信息，可随机访问。独占设备一次只给一个进程用，共享设备可轮流快速分配，虚拟设备则由共享资源模拟低速独占设备。
3. 设备独立性是让上层软件尽量不依赖具体物理设备；逻辑设备名到物理设备名的映射正是实现这种抽象的关键。
4. 控制器负责协调主机与设备，接口是交互入口，端口和寄存器用于传递命令、状态和数据。
5. 内存映像编址把端口和内存统一编址，访问灵活但占用内存地址空间；I/O 独立编址使用专门 I/O 指令，不占内存空间但灵活性较低。
6. 程序直接控制由 CPU 轮询等待；中断驱动由设备完成后中断 CPU；DMA 由 DMA 控制器直接搬整块数据；通道方式由专门通道执行通道程序控制更复杂的 I/O。
7. 因为 DMA 不再要求 CPU 逐次参与数据搬运，而是在整块传输结束后再中断通知 CPU。
8. 缓冲通过在内存中设置中间区来缓和速度不匹配、减少中断和等待；单缓冲最简单，双缓冲提高并行性，环形缓冲适合连续流，缓冲池适合多设备多进程共享。
9. 访问时间由寻道、旋转延迟和数据传输组成；其中前两者主要是机械等待，因此优化重点通常在减少磁头移动和等待旋转。
10. FCFS 简单公平但效率低；SSTF 平均寻道短但可能饥饿；SCAN 像电梯一样往返扫描，更公平；CSCAN 单向循环扫描，等待时间更均匀。
11. 磁盘高速缓存减少真实访盘次数；提前读利用顺序访问局部性；延迟写减少频繁写回；优化数据分布减少磁头移动；虚拟盘利用内存模拟磁盘以获得更高速度。
12. 静态分配在作业运行前一次分完，安全但利用率低；动态分配按需申请释放，利用率高但可能引发死锁。
13. 独享分配适合独占设备，共享分配适合共享设备，虚拟分配适合利用高速共享资源模拟低速独占设备。
14. SPOOLing 的本质是在联机系统中利用共享存储空间和专门输入输出进程，实现“假脱机”式的外围操作管理；它让慢速独占设备在逻辑上具备了共享特性。

:::

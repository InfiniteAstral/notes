# 第 6 章 无线网

无线网络这一章其实包含两条相关但不相同的主线：

- **无线（Wireless）**：通过无线链路通信，重点是信号传播、干扰、共享信道、Wi-Fi 等问题
- **移动性（Mobility）**：用户位置变化后，如何仍然保持联网，重点是切换、蜂窝网络、移动节点寻址与转发

很多初学者会把“无线”和“移动”混为一谈，但它们不是一回事：

- 设备可以是**无线但不移动**，例如固定位置的无线摄像头
- 设备也可以是**移动但不无线**，例如一台不断换有线接口接入位置的设备

学习这一章时，建议始终抓住三条主线：

- **无线链路主线**：无线链路为什么比有线更难
- **Wi-Fi 主线**：802.11 如何让很多设备共享无线信道
- **移动性主线**：设备移动后，网络如何继续找到它

::: info 学习提示

读这一章时，最好先把“无线链路特性”和“移动性管理”分开理解。

- 前者重点是信号、干扰、共享信道
- 后者重点是切换、寻址、转发路径

这样后面遇到 `Wi-Fi`、蜂窝网络、移动 IP 时就不容易混淆。

:::

## 6.1 无线网络概述

### 无线网络的组成

一个典型的无线网络通常包含三个基本部分：

- **无线主机**：笔记本、手机、平板等终端设备
- **基站**：负责把无线主机接入网络，例如蜂窝基站、Wi-Fi 接入点 `AP`
- **无线链路**：连接无线主机与基站，或直接连接无线设备之间的通信信道

### 无线主机

无线主机运行应用程序，和普通联网主机并没有本质区别。它们可以：

- 固定不动
- 在较小范围移动
- 在更大范围持续移动

所以：

- **无线不必然意味着移动**
- **移动也不必然意味着无线**

::: tip 易混点

“无线”描述的是**通信媒介**，而“移动”描述的是**接入点是否变化**。

- 手机连 Wi-Fi 但一直放桌上，是无线但几乎不移动
- 笔记本从一个有线工位换到另一个有线工位，是移动但不一定无线

:::

### 基站

基站的核心作用是：

- 作为无线网络与有线网络之间的桥梁
- 负责覆盖区域内无线设备的接入

常见例子：

- 蜂窝网络中的基站 / 基站塔
- 802.11 无线局域网中的接入点 `AP`

### 无线链路

无线链路既可以：

- 连接无线主机和基站

也可以：

- 直接连接无线节点之间

与有线链路相比，无线链路在数据率、传输距离、误码率、稳定性上变化更大，因此协议设计也更复杂。

### 两种无线网络模式

无线网络常见有两种模式：

#### 基础设施模式

在基础设施模式下：

- 无线主机通过基站接入网络
- 基站再连接到更大的有线网络或互联网

这是日常最常见的模式，例如：

- 手机连蜂窝网络
- 笔记本连 Wi-Fi

#### 自组织模式

在自组织模式下：

- 没有固定基站
- 节点之间直接通信
- 节点还可能自行承担转发与路由功能

这类网络也叫 **Ad Hoc** 网络。

## 6.2 无线链路及其特点

### 为什么无线链路比有线更难

无线链路不像网线那样有相对稳定、封闭的物理传输环境。它面临的问题更多，也更动态。

教材里最关键的几个特性包括：

- 信号衰减
- 干扰
- 多径传播
- 隐藏站和暴露站问题
- 信干噪比 `SINR`

### 信号衰减

无线电信号在空间中传播时会逐渐减弱，这叫**衰减（Attenuation）** 或路径损耗。

这意味着：

- 距离越远，接收到的信号通常越弱
- 墙壁、障碍物、人体等也可能继续削弱信号

### 干扰

无线频谱往往不是你一个人在用。其他设备也可能在相近频段工作，例如：

- 其他 Wi-Fi 网络
- 蓝牙设备
- 微波炉、电话等电磁设备

因此无线链路很容易受到**干扰**。

### 多径传播

无线电波会在地面、墙面、建筑物、金属物体上反射。

于是同一个信号可能：

- 经过多条不同路径
- 在略有不同的时间到达接收方

这就是**多径传播（Multipath Propagation）**。

它会导致：

- 信号叠加失真
- 误码率上升
- 接收端处理变复杂

### 隐藏站问题

隐藏站问题是无线里非常经典的现象。

例如：

- A 和 B 可以互相通信
- C 和 B 也可以互相通信
- 但 A 和 C 因距离远或障碍物影响，彼此听不到对方

这样一来：

- A 觉得信道空闲，就发
- C 也觉得信道空闲，也发
- 最终在 B 处发生冲突

这就是**隐藏站（Hidden Terminal）** 问题。

### 暴露站问题

暴露站问题与隐藏站相反。

例如某节点侦听到邻居在发送后，误以为自己一定不能发，但实际上：

- 自己若发送，也不一定会干扰真正目标接收方

结果就是：

- 明明可以并行发送
- 却因为过于保守而不发

这就是**暴露站（Exposed Terminal）** 问题。

::: warning 隐藏站与暴露站

这两个问题都说明了一件事：在无线环境里，“我听到什么”并不等于“整个网络当前真实发生了什么”。

- 隐藏站：本来会冲突，但发送方听不出来
- 暴露站：本来可以发，但发送方误以为不能发

这也是无线网络很难直接照搬有线以太网 `CSMA/CD` 的重要原因。

:::

### SINR：信干噪比

**SINR（Signal-to-Interference-plus-Noise Ratio）** 表示：

- 有效信号强度
- 相对于干扰和噪声强度

它是无线通信中的关键指标。

### SINR 与误码率、速率的关系

一般来说：

- `SINR` 越高，越容易正确恢复信号
- 误码率越低
- 可以选择更高的数据率和更复杂的调制方式

反过来：

- `SINR` 较差时
- 系统往往要降低发送速率
- 换取更可靠的传输

这也是为什么无线网络中的实际速率常常不是固定值，而会动态变化。

::: info 直观理解

可以把 `SINR` 理解成“目标声音相对于环境噪声有多清楚”。

- `SINR` 高：目标信号更突出，容易正确识别，可以跑更高数据率
- `SINR` 低：目标信号淹没在噪声和干扰里，只能降低速率换稳定性

所以无线网络里看到“离 AP 更近时速率更高”，本质上往往就是 `SINR` 更好了。

:::

## 6.3 CDMA：码分多址

### CDMA 的核心思想

**CDMA（Code Division Multiple Access，码分多址）** 的基本思想是：

- 多个用户共享同一频率
- 同时发送
- 但每个用户使用不同的“码片序列”编码自己的数据

接收方只要知道对应码片序列，就能从叠加信号中提取出目标用户的数据。

### 为什么 CDMA 能让大家同时发

其关键在于：

- 不同用户的码片序列设计得尽量正交或相关性很低

于是多个发送信号叠加后，接收端通过与目标码片做相关运算，仍可恢复对应数据。

### CDMA 的直观理解

可以把它理解成：

- 大家在同一个房间同时说话
- 但每个人都用自己独特的“节奏/暗号”说
- 听者只关注某一个特定节奏，就能较好地听出目标内容

### CDMA 的意义

CDMA 提供了一种不同于 `TDMA`、`FDMA` 的多址接入思路。教材中它常作为理解“共享无线媒介并不只有一种方法”的代表，也在早期蜂窝系统中广泛使用。

## 6.4 IEEE 802.11：Wi-Fi

### Wi-Fi 是什么

**Wi-Fi** 是最常见的无线局域网技术，对应的一组核心标准是 **IEEE 802.11**。

它解决的问题是：

- 在一个局部区域内
- 多台无线设备如何接入同一网络
- 并与有线网络连接起来

### 常见 802.11 标准

教材中常见的几个版本有：

- `802.11b`：2.4GHz，最高约 `11 Mbps`
- `802.11a/g`：5GHz / 2.4GHz，最高约 `54 Mbps`
- `802.11n`：支持多天线 `MIMO`，速率进一步提升

不同版本在：

- 频段
- 调制方式
- 速率
- 覆盖能力

等方面都有差异。

### 802.11 的架构

在基础设施模式下，802.11 的基本单元叫 **BSS（Basic Service Set，基本服务集）**。

一个 BSS 通常包括：

- 一个接入点 `AP`
- 若干与该 `AP` 关联的无线主机

### AP 是什么

`AP` 可以理解为 Wi-Fi 网络里的基站。

它通常：

- 向无线主机广播自己的存在
- 接收主机接入请求
- 把无线局域网连接到交换机、路由器或互联网

### 信道与频段

Wi-Fi 工作在特定频段上，例如 2.4GHz 或 5GHz。

这些频段又会进一步划分为多个**信道（Channel）**。

不同接入点若选用相同或相近信道，可能相互干扰，因此信道规划会影响网络质量。

## 6.5 802.11 的关联过程

### 为什么必须先关联

无线主机不是一上来就能直接通信，它必须先确定：

- 周围有哪些 AP
- 自己准备接入哪个 AP

这个过程叫**关联（Association）**。

### 被动扫描

在 **被动扫描** 中：

- AP 周期性发送**信标帧（Beacon）**
- 主机监听各信道上的信标
- 根据信号、SSID 等信息选择一个 AP
- 然后发送关联请求
- AP 返回关联响应

### 主动扫描

在 **主动扫描** 中：

- 主机主动广播探测请求
- 各 AP 返回探测响应
- 主机再选择目标 AP，发送关联请求
- AP 回复关联响应

### 关联之后还要做什么

关联成功后，主机通常还需要：

- 身份验证
- 获取 IP 地址

其中获取 IP 地址通常依赖：

- **DHCP**

也就是说，关联成功只表示“接入到 AP”，并不自动等于“已经拿到网络层地址”。

::: info 关联不等于联网完成

很多初学者会把“连上 Wi-Fi”理解成“已经能正常上网”，但中间其实还隔着几步：

1. 扫描并选择 `AP`
2. 完成关联
3. 通过认证机制
4. 通过 `DHCP` 获取 IP 地址等网络参数

只有这些步骤都成功后，设备才通常具备正常通信能力。

:::

## 6.6 802.11 的多路访问：CSMA/CA

### 为什么 Wi-Fi 不用 CSMA/CD

以太网中曾使用 `CSMA/CD`，但无线环境不适合直接照搬，因为：

- 发送时本地信号太强
- 很难同时准确接收别人信号
- 隐藏站问题让“检测到所有碰撞”本身就很难

因此 802.11 采用的是：

- **CSMA/CA（Collision Avoidance，碰撞避免）**

::: tip 记忆方式

- 有线以太网更强调“碰了再尽快发现”，所以是 `CD`
- 无线网络更强调“尽量别碰”，所以是 `CA`

不用死背，抓住“无线里很难可靠检测碰撞”这个原因就够了。

:::

### CSMA/CA 的基本思路

其核心不是“碰了再检测”，而是：

- 先尽量避免碰撞
- 若怀疑失败，再重传

### 基本发送过程

一个简化后的 802.11 发送流程可以理解为：

1. 先侦听信道
2. 若信道连续空闲达到 `DIFS`，则发送
3. 若信道忙，则选择一个随机退避时间
4. 信道空闲时才继续倒计时
5. 退避计时器归零后发送数据帧
6. 接收方正确收到后，在 `SIFS` 后返回 `ACK`
7. 若发送方没等到 `ACK`，则推测发送失败并重传

### DIFS 和 SIFS

这两个缩写容易背混，理解即可：

- **`DIFS`**：发送普通数据前需要等待的较长空闲间隔
- **`SIFS`**：用于立即回复 `ACK` 等控制帧的较短间隔

这样设计的意义是：

- 让 `ACK` 这类控制响应优先级更高

### 为什么 802.11 需要 ACK

在有线以太网里，链路层不一定用 `ACK`；但在无线里，802.11 常使用 `ACK`，原因包括：

- 无线误码率更高
- 无法像 `CSMA/CD` 那样可靠检测碰撞
- 隐藏站问题可能导致发送方并不知道自己是否成功送达

因此：

- 收到 `ACK` 才更能说明发送成功

::: warning 为什么无线里 ACK 更重要

在无线环境中，发送方即使“把帧发出去了”，也不代表它真的知道接收方收到了。

因为：

- 碰撞可能在别处发生
- 干扰和衰减可能导致接收失败
- 发送方又很难像有线那样边发边检测

所以 `ACK` 在 802.11 中承担了非常关键的“确认成功送达”作用。

:::

## 6.7 RTS/CTS：进一步避免碰撞

### RTS/CTS 要解决什么问题

如果直接发送长数据帧，一旦冲突，浪费很大。

因此 802.11 可以先交换较短的控制帧：

- `RTS`：Request To Send，请求发送
- `CTS`：Clear To Send，允许发送

### 工作过程

一个典型流程是：

1. 发送方先向 AP 发送 `RTS`
2. AP 若同意，就广播 `CTS`
3. 听到 `CTS` 的其他站点知道信道即将被占用，于是推迟发送
4. 发送方再发送真正的数据帧
5. 接收完成后返回 `ACK`

### RTS/CTS 的价值

RTS/CTS 的意义在于：

- 即使碰撞，也更可能发生在短小的 `RTS` 帧上
- 从而避免长数据帧的冲突代价
- 对隐藏站问题也有缓解作用

::: info 什么时候 RTS/CTS 更有价值

`RTS/CTS` 不是因为“控制帧更高级”，而是因为它能降低大帧冲突的浪费。

所以在这些场景里更有价值：

- 数据帧较长
- 隐藏站问题明显
- 冲突代价较高

如果数据本来就很短，额外交换控制帧反而可能得不偿失。

:::

## 6.8 802.11 帧与移动性

### 802.11 帧为什么地址字段更多

802.11 帧中常见多个地址字段，这是因为无线局域网里往往同时要表达：

- 当前无线发送者是谁
- 当前无线接收者是谁
- 最终要去往哪一侧网络

所以它的地址结构通常比普通以太网帧更复杂。

### 同一子网内移动

如果一个主机在同一个 IP 子网内，从一个 AP 移动到另一个 AP：

- 它的 **IP 地址通常可以保持不变**

为什么还能找到它？

因为：

- 底层交换机会继续通过**自学习**
- 观察主机从哪个端口方向发送帧
- 逐渐更新“该主机现在在哪边”

也就是说：

- **同子网内的移动，很多时候主要由链路层机制处理**

::: info 同子网内移动

只要设备仍留在同一个 IP 子网里，很多情况下就不需要更换 IP 地址。

变化的通常是：

- 它当前关联的是哪个 `AP`
- 交换机认为“去这个主机该走哪个端口”

这也是“Wi-Fi 漫游了但 IP 没变”这一现象背后的直观原因。

:::

## 6.9 蜂窝网络

### 什么是蜂窝网络

蜂窝网络是覆盖范围更广、支持大规模移动用户接入的无线网络体系。

其基本思路是：

- 把较大地理区域划分为许多小区（Cell）
- 每个小区由一个基站提供覆盖
- 用户移动时，在不同小区之间切换

### 蜂窝网络的发展

教材中通常按代际来梳理：

- **2G**：以语音为主，例如 `GSM`、`IS-95`
- **2.5G**：在语音基础上增强数据能力，例如 `GPRS`、`EDGE`
- **3G**：语音与数据能力进一步增强，例如 `WCDMA`、`CDMA2000`
- **4G / LTE**：高速分组数据成为核心
- **5G 及以后**：更高带宽、更低时延、更强连接能力

### 蜂窝网络和 Wi-Fi 的区别

二者都属于无线接入，但侧重点不同：

- **Wi-Fi**：局部范围、局域网接入、成本低
- **蜂窝网络**：广域覆盖、移动性强、运营商统一建设和管理

## 6.10 移动性管理

### 什么是移动性

移动性问题的核心是：

- 设备位置变了
- 网络连接点也变了
- 但通信还希望继续进行

也就是说，网络需要回答：

- 现在这个移动节点在哪里
- 发给它的数据该如何送到新位置

### 切换（Handoff）

当移动设备离开一个基站覆盖区并进入另一个基站覆盖区时，就会发生：

- **切换（Handoff）**

切换的本质是：

- 更换当前负责接入网络的基站

切换做得不好，就会导致：

- 短暂中断
- 时延突增
- 丢包

### 通过间接路由实现移动性

教材中常见一种基本思路是 **间接路由（Indirect Routing）**。

它的大致逻辑是：

- 通信方仍按移动设备原来的归属地址发送分组
- 归属网络中的代理先接收到这些分组
- 再把它们转发到移动设备当前所在的访问网络

### 归属网络与访问网络

理解移动性时，两个概念很关键：

- **归属网络（Home Network）**：设备原本所属的网络
- **访问网络（Visited / Foreign Network）**：设备当前临时接入的网络

### 归属代理与外地代理

在经典移动 IP 思想中，常涉及两个角色：

- **归属代理（Home Agent）**：位于归属网络，负责截获发往移动节点的数据
- **外地代理（Foreign Agent）**：位于访问网络，负责接收转发来的分组并交给移动节点

### 隧道

为了把“本来发往归属地址的数据”送到移动节点当前所在位置，常使用：

- **隧道（Tunneling）**

其基本思想是：

- 把原始分组再次封装
- 发到移动节点当前所在网络的代理或临时地址
- 到达后再解封装，交给移动节点

### 间接路由的代价

这种方式的优点是：

- 通信方不一定需要知道移动节点当前的精确位置

但也有明显代价：

- 路径可能绕远
- 增加时延
- 管理更复杂

教材里通常会把这种现象和“三角路由”直觉联系起来理解。

::: warning 间接路由的代价

间接路由的核心问题不是“能不能送到”，而是“能送到，但路径可能不够自然”。

典型现象是：

- 分组先绕到归属网络
- 再被转发到移动节点当前所在网络

这会导致路径变长、时延增大，也正是后续很多移动性优化方案要解决的问题。

:::

## 本章小结

学完无线网后，建议你至少能回答这些问题：

1. “无线”和“移动性”为什么是相关但不同的两个问题
2. 无线链路为什么比有线链路更容易出错、更难共享
3. 隐藏站、暴露站分别是什么
4. `SINR` 为什么会影响误码率和可选传输速率
5. `CDMA` 为什么能让多个用户共享同一频段同时发送
6. `802.11` 中 `BSS`、`AP`、关联、信标、扫描分别是什么意思
7. Wi-Fi 为什么使用 `CSMA/CA` 而不是 `CSMA/CD`
8. `DIFS`、`SIFS`、`ACK`、`RTS/CTS` 分别起什么作用
9. 同一子网内在不同 `AP` 间移动时，为什么 IP 地址通常可以不变
10. 蜂窝网络中的切换和移动性管理分别解决什么问题
11. 归属代理、外地代理、隧道分别承担什么角色

::: details 参考答案

1. “无线”强调通信媒介是无线信道，“移动性”强调节点接入位置会变化；

   两者经常一起出现，但并不是同一个问题。

2. 无线链路更容易出错、更难共享，是因为它存在衰减、干扰、多径传播，以及隐藏站、暴露站等问题。
3. 隐藏站是两个发送方彼此听不到，却会在接收端发生冲突；

   暴露站是某节点听到别人发送后误以为自己不能发，其实发送也不一定会造成干扰。

4. `SINR` 反映有效信号相对干扰和噪声的强弱，`SINR` 越高，误码率通常越低，也越可能支持更高的数据率。
5. `CDMA` 让多个用户共享同一频段同时发送，是因为每个用户使用不同码片序列编码，接收方可根据目标码片把信号分离出来。
6. `BSS` 是一个基本服务集，通常由一个 `AP` 和若干无线主机构成；

   `AP` 是无线接入点；

   关联是主机接入某个 AP 的过程；

   信标用于广播 AP 信息；

   扫描用于发现可接入的 AP。

7. Wi-Fi 使用 `CSMA/CA` 而不是 `CSMA/CD`，因为无线环境里很难边发边可靠检测碰撞，因此更适合尽量避免碰撞。
8. `DIFS` 是普通数据发送前的等待间隔，

   `SIFS` 是发送 `ACK` 等高优先级控制帧前的更短间隔，

   `ACK` 用于确认接收成功，

   `RTS/CTS` 用于在发送长数据前先预留信道、缓解隐藏站冲突。

9. 同一子网内在不同 `AP` 间移动时，设备通常仍处于同一个 IP 子网，因此很多情况下不需要更换 IP 地址，变化主要由链路层和交换转发表来适应。
10. 蜂窝网络中的切换解决“设备从一个基站移动到另一个基站时如何保持接入”，

    移动性管理解决“网络如何持续找到并把数据送到移动节点”。

11. 归属代理负责在归属网络中接收发往移动节点的分组，

    外地代理负责在当前访问网络中协助转发，

    隧道负责把原始分组封装后送往移动节点当前所在位置。

:::

如果这些问题已经能比较顺畅地解释出来，说明你对无线网与移动性已经建立起了比较完整的基础框架。

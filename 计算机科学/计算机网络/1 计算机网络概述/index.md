# 第 1 章 计算机网络概述

本章的目标不是一上来就记协议细节，而是先建立一张“全景图”：互联网由哪些部分组成、数据是怎样在网络里流动的、为什么要分层、网络性能通常怎么看。学完这一章后，再进入应用层、传输层、网络层等内容会轻松很多。

::: info 学习提示

这一章最重要的不是记住所有术语，而是先建立“地图感”：

- 互联网由什么组成
- 数据怎样在网络中前进
- 为什么网络要分层

只要这张大图先建立起来，后面每一章都会更容易理解。

:::

## 1.1 什么是互联网

### 两种理解视角

理解互联网，通常有两种常见视角：

- **组成视角**：互联网是由大量主机、链路、路由器、交换机以及 ISP 互联起来形成的“万网之网”。
- **服务视角**：互联网是一个为应用程序提供通信能力的基础设施，网页浏览、邮件、即时通信、在线游戏、视频会议等应用都建立在它之上。

也就是说，互联网既是一套真实存在的硬件和软件系统，也是一套面向应用提供通信服务的平台。

### 互联网的核心组成

- **端系统（End System）/ 主机（Host）**：运行网络应用的设备，如电脑、手机、服务器。端系统位于网络边缘，是用户最直接接触的部分。
- **客户端（Client）**：主动发起请求的一方，如浏览器、手机 App。
- **服务器（Server）**：持续在线、等待并处理请求的一方，通常部署在数据中心中。
- **通信链路（Link）**：连接不同设备的传输媒介，如双绞线、同轴电缆、光纤、无线电链路、卫星链路。
- **分组交换设备（Packet Switch）**：负责转发数据分组的设备，典型有路由器和链路层交换机。
- **ISP（Internet Service Provider，互联网服务提供商）**：向家庭、学校、企业等提供接入服务。不同层级的 ISP 彼此互联，最终形成全球互联网。

::: tip

“带宽”通常指链路的传输能力，单位是 `bps`（bit per second，每秒比特数）。它表示链路理论上的传输速率上限，不等于应用实际能跑到的速度。

:::

### 互联网提供什么

互联网不仅连接设备，还向应用提供通信基础设施：

- **为应用提供通信服务**：支持 Web、电子邮件、网络电话、文件传输、云服务等。
- **为程序提供接口**：应用通过套接字（Socket）等接口使用网络能力，而不是直接控制底层链路和路由器。

可以把互联网理解为一套公共基础设施：应用开发者更关心“能不能可靠发送数据”“时延大不大”，而不是每个比特具体经过了哪根光纤。

### 什么是协议

**协议（Protocol）** 是网络实体之间通信时共同遵守的规则，主要规定：

- 发送什么消息
- 消息的格式是什么
- 先发什么、后发什么
- 收到消息后该做什么
- 出错或超时时该怎么办

例如，浏览器请求网页、服务器返回网页，这背后就依赖 HTTP 协议；主机之间如何传输数据，又依赖 TCP/IP 等协议。

::: info

可以把协议理解成“机器世界里的交流规则”。人类打招呼讲究先后顺序和回应方式，网络中的通信也一样。

:::

### 互联网标准

- **IETF**：互联网工程任务组，是互联网标准的重要制定组织。
- **RFC**：Request for Comments，IETF 发布标准和技术文档的重要形式。

很多常见协议，如 IP、TCP、HTTP 的标准，都可以在 RFC 文档中找到。

## 1.2 网络边缘与接入网

### 网络边缘

网络边缘是端系统所在的部分，也是用户真正“使用网络”的地方。你在浏览器里打开网页、手机里发消息，本质上都是网络边缘的应用在进行通信。

从角色上看，网络边缘常见为两类主机：

- **客户机-服务器模式（C/S）**：客户端发起请求，服务器提供服务。
- **对等模式（P2P）**：端系统之间直接交换数据，彼此既是请求者，也可能是提供者。

### 接入网是什么

**接入网（Access Network）** 指端系统连接到第一个路由器的那一段网络。它回答的是一个非常基础的问题：

**用户设备是怎样接入互联网的？**

接入网常从两个角度来分析：

- **带宽有多大**：决定了能以多快的速率接入网络。
- **媒介是否共享**：决定了这条链路是你独享，还是需要和别人竞争使用。

::: tip 直观理解

接入网可以先粗略理解成“你家设备到互联网的第一段路”。

- 这段路如果是专用的，体验通常更稳定
- 这段路如果是共享的，就可能受到其他用户活动影响

:::

### 常见接入方式

| 接入类型            | 基本方式                   | 典型特点                                    | 典型速率特征                  |
| ------------------- | -------------------------- | ------------------------------------------- | ----------------------------- |
| DSL（数字用户线路） | 利用电话线接入             | 用户到局端通常是专线；常见为非对称接入      | 上行较低、下行较高            |
| HFC（混合光纤同轴） | 利用有线电视网络接入       | 同一区域用户共享接入链路；常用 FDM 划分频段 | 下行通常较高                  |
| 以太网              | 学校、企业等机构局域网接入 | 常见为交换式接入，速率范围大                | `10Mbps` 到 `10Gbps` 甚至更高 |
| WLAN / WiFi         | 无线局域网接入             | 通过无线接入点接入，信道共享                | 速率受标准和环境影响较大      |
| 蜂窝移动网络        | 基站接入                   | 覆盖广，支持移动性                          | 从 `3G` 到 `5G` 逐代提升      |

### 几个容易混淆的概念

- **接入网**：用户“连上互联网”的第一段网络。
- **物理媒介**：真正承载信号传播的介质，如光纤、铜缆、无线电。
- **链路**：两个相邻节点之间的通信通道，由某种物理媒介实现。

例如，家里用 WiFi 上网时：

- 手机到路由器之间通常是**无线链路**
- 路由器到运营商之间可能是**光纤或铜缆链路**
- 这些链路共同组成了你的接入过程

### 物理媒介简述

- **双绞线**：成本低，常见于以太网和传统电话线。
- **同轴电缆**：抗干扰较好，常见于有线电视网络。
- **光纤**：速率高、传输远、抗干扰能力强，是骨干网的核心媒介。
- **无线电链路**：部署灵活，适合移动接入，但容易受到干扰和衰减影响。
- **卫星链路**：覆盖广，但传播距离远，时延通常较大。

## 1.3 网络核心

### 网络核心是什么

网络核心可以理解为“由大量路由器互联形成的中间网络”。如果说网络边缘是用户与应用所在的区域，那么网络核心就是负责把数据从源主机送到目的主机的“中转系统”。

网络核心有两个关键任务：

- **转发（Forwarding）**：路由器收到一个分组后，根据分组首部信息，把它送到合适的输出链路。
- **路由（Routing）**：决定从源到目的地应该走哪条路径。

简单说：

- 转发关注“这一个分组下一步往哪走”
- 路由关注“整体路径怎么选”

### 分组交换

互联网的核心工作方式是 **分组交换（Packet Switching）**。

#### 什么是分组

应用产生的完整消息通常不会整体发送出去，而是会被拆成一个个较小的数据块，这些数据块就叫 **分组（Packet）**。

这样做的好处是：

- 单个数据块更容易在网络中转发
- 多个用户可以共享同一条链路
- 某个分组出错时，只需重传相关部分

#### 存储-转发

路由器通常采用 **存储-转发（Store-and-Forward）** 机制：

- 必须先把一个完整分组接收下来
- 然后才能把它转发到下一跳

若分组长度为 `L` bit，链路速率为 `R` bps，则把该分组“推”到链路上的发送时延为：

`d_trans = L / R`

如果一个分组需要经过两条相同速率的链路，且暂时忽略传播时延和排队时延，则端到端发送过程至少需要约 `2L/R` 的时间。

#### 排队、丢包与统计复用

分组交换中，链路资源并不是提前为某个用户保留的，而是多个用户**按需共享**。这种方式叫 **统计复用**。

它的优点是资源利用率高，但也带来两个直接结果：

- **排队**：当到达路由器的分组太多，来不及立刻发出时，就会先进入缓存队列等待。
- **丢包**：缓存空间有限；当队列满了，新到达的分组就会被丢弃。

### 电路交换

**电路交换（Circuit Switching）** 的思想与分组交换不同。它会在通信开始前，为通信双方预先分配一条端到端的专用资源通路。

特点如下：

- 通信前先建立连接
- 资源在通信期间被预留
- 其他用户不能占用这些资源
- 通信完成后再释放资源

传统电话网络是电路交换的典型例子。

### FDM 与 TDM

电路交换中，常见的资源划分方式有：

- **FDM（Frequency Division Multiplexing，频分复用）**：把一条链路的频率范围切成多个频段，每个用户占用一个频段。
- **TDM（Time Division Multiplexing，时分复用）**：把时间切成多个时隙，不同用户在不同时间片上轮流使用链路。

### 分组交换与电路交换对比

| 对比维度               | 分组交换             | 电路交换       |
| ---------------------- | -------------------- | -------------- |
| 资源使用方式           | 按需共享             | 预先保留       |
| 是否需要先建立连接     | 一般不需要           | 需要           |
| 对突发数据业务的适应性 | 强                   | 较差           |
| 资源利用率             | 高                   | 可能浪费       |
| 性能稳定性             | 可能排队、拥塞、丢包 | 较稳定、可预测 |
| 典型场景               | 互联网数据通信       | 传统电话网络   |

::: tip

互联网采用分组交换，不是因为它“任何方面都更强”，而是因为它更适合计算机中的突发式数据业务，且资源利用率更高。

:::

### 互联网的结构

互联网并不是一张由单一组织统一建设的大网，而是层层互联形成的：

- 家庭、学校、企业主机先接入 **接入 ISP**
- 接入 ISP 再与 **区域 ISP** 或更大的网络互联
- 更高层的 ISP 继续互联，形成覆盖更广的骨干网络
- 大型内容提供商（如云服务商、搜索服务商、视频平台）还会建设自己的专用网络，以提升访问效率

这就是“**网络的网络**”的真正含义。

## 1.4 网络性能指标

网络性能常从三个角度来衡量：**时延、丢包、吞吐量**。

### 四种节点时延

一个分组在路由器处经历的总时延可以写成：

`d_nodal = d_proc + d_queue + d_trans + d_prop`

其中四部分分别是：

1. **处理时延 `d_proc`**

   路由器检查首部、检查差错、决定从哪个端口发出的时间，通常很短。

2. **排队时延 `d_queue`**

   分组在缓存队列中等待发送的时间。网络越拥塞，排队时延通常越大。

3. **发送时延 `d_trans`**

   把分组的所有比特逐个“推”到链路上的时间：

   `d_trans = L / R`

4. **传播时延 `d_prop`**

   比特在物理链路中真正传播所花的时间：

   `d_prop = d / s`

   其中 `d` 是链路长度，`s` 是信号在媒介中的传播速度。

::: danger 注意

发送时延和传播时延完全不是一回事。

- 发送时延看的是“要把多少比特推上链路”
- 传播时延看的是“信号在链路上跑多远”

很多初学者会把这两者混为一谈，这是非常常见的错误。

:::

### 用生活类比理解发送时延与传播时延

可以把分组想象成一队汽车：

- **发送时延**：相当于收费站把整支车队一辆辆放上高速所花的时间。
- **传播时延**：相当于最后一辆车从收费站开到下一个站所花的路上行驶时间。

这个类比很适合帮助理解：为什么“链路很长”会影响传播时延，而“分组很大”会影响发送时延。

### 排队时延与流量强度

设：

- `L`：分组长度
- `a`：分组平均到达速率
- `R`：链路带宽

则 **流量强度** 常写作 `La/R`。

- 当 `La/R` 很小，排队时延通常很小
- 当 `La/R` 接近 `1`，排队时延会迅速增大
- 当 `La/R > 1`，说明到达速率长期超过处理能力，队列会持续增长

这说明：网络不一定是“链路断了”才变慢，很多时候只是因为排队过长。

::: info `La/R` 怎么理解

这个式子本质上是在比较：

- 分组“来得有多快”
- 链路“处理得有多快”

如果到达速度不断逼近甚至超过链路处理能力，队列就会越来越长，排队时延也会迅速变大。

:::

### 丢包

路由器缓存空间有限。如果新分组到达时，缓冲区已经满了，这个分组就会被直接丢弃，这就是 **丢包**。

丢包后会发生什么，要看具体协议：

- 有些协议会触发重传
- 有些应用可以容忍少量丢失，不一定重传

因此，“丢包”不总是意味着“整个通信失败”，但它往往意味着网络正处于拥塞或质量较差的状态。

### 吞吐量

**吞吐量（Throughput）** 是单位时间内实际成功传输的数据量。

常见区分有两种：

- **瞬时吞吐量**：某一时刻的传输速率
- **平均吞吐量**：较长时间内的平均传输速率

### 瓶颈链路

端到端路径上的吞吐量，通常由路径中最慢的那段链路决定，这条最慢的链路称为 **瓶颈链路**。

例如：

- 服务器出口速率是 `100Mbps`
- 中间链路速率是 `10Mbps`
- 用户接入速率是 `50Mbps`

那么端到端吞吐量通常不会超过 `10Mbps`。

::: tip

多条连接共享同一条瓶颈链路时，每条连接最终能分到多少带宽，常取决于竞争情况和协议机制。理想化情况下，若 `K` 条连接公平共享带宽 `R`，每条连接约为 `R/K`。

:::

### 往返时延 RTT

除了单向时延，网络中还常用 **RTT（Round-Trip Time，往返时延）** 表示一个报文从发送端到接收端、再返回响应所经历的总时间。

RTT 在后续很多协议中都很重要，例如：

- 浏览器访问网页时，建立连接需要消耗 RTT
- TCP 确认机制、超时重传都与 RTT 密切相关

## 1.5 协议分层与服务模型

### 为什么要分层

计算机网络是一个非常复杂的系统，包含主机、路由器、链路、信号、差错控制、应用协议等大量细节。为了便于设计、实现和维护，人们采用了 **分层（Layering）** 的思想。

分层的主要好处是：

- **降低复杂度**：每层只处理自己负责的问题。
- **便于复用与替换**：某一层的实现变化，不必影响所有层。
- **便于学习和分析**：可以逐层理解网络，而不用一次面对全部细节。

当然，分层也不是没有代价：

- 层次过多会带来额外开销
- 某些功能可能在不同层中出现重复

但总体上，分层仍然是构建复杂网络系统最有效的方法之一。

### 服务和协议有什么区别

这是初学者最容易混淆的点之一：

- **服务（Service）**：一层向上一层“提供什么能力”。
- **协议（Protocol）**：同一层的不同实体之间“如何协作来实现这种能力”。

可以简单记为：

- 服务回答“**能做什么**”
- 协议回答“**怎么做到**”

::: warning 服务和协议不要混

这是初学者最常混淆的一组概念：

- 服务：上一层“得到什么能力”
- 协议：同层实体“靠什么规则协作”

看到题目里在问“TCP 是什么”，通常既可能在问“它提供什么服务”，也可能在问“它作为协议如何工作”，要注意语境。

:::

例如，传输层可以向应用层提供“可靠传输”这样的服务；而 TCP 则是一种实现这类服务的协议。

### 互联网协议栈

互联网通常采用五层协议栈：

| 层次       | 主要任务                           | 典型协议               |
| ---------- | ---------------------------------- | ---------------------- |
| **应用层** | 为具体网络应用提供协议和数据格式   | HTTP、FTP、SMTP、DNS   |
| **传输层** | 实现进程到进程的逻辑通信           | TCP、UDP               |
| **网络层** | 实现主机到主机的数据转发与路由选择 | IP、RIP、OSPF、BGP     |
| **链路层** | 实现相邻节点之间的帧传输           | Ethernet、802.11、PPP  |
| **物理层** | 负责比特在媒介上的实际传输         | 电气/光学/无线信号规范 |

可以这样理解这五层：

- 应用层关心“我要完成什么应用功能”
- 传输层关心“应用进程之间怎样通信”
- 网络层关心“跨网络怎样把数据送到目标主机”
- 链路层关心“当前这一跳怎样传过去”
- 物理层关心“0 和 1 怎样变成真正的电信号、光信号或无线信号”

### ISO/OSI 参考模型

OSI 模型有七层，在互联网五层模型基础上多了两层：

- **表示层**：数据表示、加密、压缩等
- **会话层**：会话管理、同步、恢复等

在现代互联网实践中，这两部分功能往往直接由应用层或应用程序自身来实现，因此教材中更常使用五层互联网协议栈。

### 封装与解封装

发送数据时，数据会随着层次向下传递，并不断加上控制信息，这个过程叫 **封装（Encapsulation）**。

典型过程如下：

- 应用层生成 **报文（Message）**
- 传输层加上传输层首部，形成 **报文段（Segment）**
- 网络层加上网络层首部，形成 **数据报（Datagram）**
- 链路层加上链路层首部，形成 **帧（Frame）**
- 物理层把帧转换为比特流发送出去

接收方则反过来逐层去掉首部，这个过程叫 **解封装**。

### 不同设备处理到哪一层

- **端系统**：通常会处理从应用层到物理层的全部层次。
- **路由器**：主要工作在网络层，也会涉及链路层和物理层。
- **链路层交换机**：主要依据链路层信息转发帧。

### 互联网中的服务模型

在“服务模型”这个概念中，最重要的入门结论之一是：

**互联网网络层提供的是尽力而为（Best-Effort）服务。**

它的含义是：

- 网络会尽最大努力转发分组
- 但**不保证**一定送达
- 也**不保证**固定时延、固定带宽、固定到达顺序

这也是为什么后续还需要 TCP 这样的协议，去进一步提供可靠传输、拥塞控制等能力。

## 1.6 互联网发展史

### 早期基础

- **1961 年**：Kleinrock 用排队论分析分组交换思想。
- **1964 年**：Baran 进一步推动分组交换在网络中的应用。
- **1967 年**：ARPAnet 方案提出，成为互联网的早期雏形。
- **1969 年**：首个 ARPAnet 节点开始运行。

### 网络互连思想形成

- **1970 年代**：不同专用网络不断出现，网络之间互连成为核心问题。
- **1974 年**：Cerf 和 Kahn 提出网络互连体系结构思想。
- **这一时期形成了影响深远的原则**：
  - 网络之间互联时，尽量不要求内部大改
  - 服务模型采用尽力而为
  - 路由控制倾向分散化

这些思想深刻影响了今天互联网的架构。

### 协议与基础设施快速发展

- **1980 年代**：TCP/IP 成为主流协议体系。
- **SMTP、DNS、FTP** 等关键协议陆续标准化。
- **以太网** 快速部署，国家级网络不断出现，互联网规模迅速扩大。

### Web 推动商业化

- **1990 年代初**：Web 出现，HTML、HTTP、浏览器等技术推动互联网走向大众。
- **1990 年代中后期**：互联网开始商业化，大量网站和在线应用出现。
- **2000 年前后**：P2P、网络安全、宽带接入等议题迅速升温。

### 现代互联网

- 移动终端和无线接入迅速普及
- 云计算成为重要基础设施
- 大型内容提供商建设全球专用网络
- 社交网络、流媒体、移动应用成为主流场景

今天的互联网，已经从“连接几台计算机的实验网络”发展成支撑社会运行的重要基础设施。

## 本章小结

学完本章，建议你至少能回答下面这些问题：

1. 互联网从“组成视角”和“服务视角”分别是什么
2. 网络边缘、接入网、网络核心分别承担什么角色
3. 分组交换和电路交换有什么差别
4. 时延、丢包、吞吐量分别描述什么
5. 为什么网络要分层，服务和协议有什么区别
6. 互联网协议栈的五层各自做什么

::: details 参考答案

1. 互联网从“组成视角”看，是由主机、链路、路由器、交换机、ISP 等互联起来形成的万网之网；

   从“服务视角”看，是为各种网络应用提供通信能力的基础设施。

2. 网络边缘是用户主机和应用所在的地方；

   接入网负责把用户设备连到第一个路由器；

   网络核心由大量路由器互联而成，负责转发和路由。

3. 分组交换按需共享资源，不必预先建立连接，资源利用率高但可能排队和丢包；

   电路交换先建立连接并预留资源，性能更稳定但资源可能浪费。

4. 时延描述数据传输要花多久；

   丢包描述分组是否在网络中被丢弃；

   吞吐量描述单位时间内成功传输的数据量。

5. 分层是为了降低复杂度、便于实现和维护；

   服务强调“这一层向上一层提供什么能力”，协议强调“同层实体如何协作实现这种能力”。

6. 五层协议栈中，应用层负责网络应用，传输层负责进程到进程通信，网络层负责主机到主机转发与路由，链路层负责一跳上的帧传输，物理层负责比特在媒介上的实际传送。

:::

如果这些问题已经能说清楚，就说明你已经建立起了后续学习计算机网络的基本框架。

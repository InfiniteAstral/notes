# 第 1 章 计算机网络概述

## 1.1 互联网基础

### 核心组成

- **端系统（主机）**：运行网络应用的终端设备，分为客户端和服务器，服务器通常长期在线，部署在数据中心。

- **通信链路**：光纤、同轴电缆、无线电、卫星等，传输速率用 **带宽（bps）** 衡量。

- **分组交换设备**：路由器、交换机，负责转发数据分组。

- **ISP 层次**：接入 ISP、区域 ISP、全局 ISP，构成 "万网之网"，内容提供商（如 Google）会搭建私有网络优化服务。

### 协议的本质

协议定义了网络实体之间发送 / 接收消息的**格式、顺序、动作规则**，本质是通信双方的 "约定"，类似人类生活中的交互规则。

::: info

互联网标准由 IETF 制定，以 RFC 文档发布。

:::

## 1.2 网络边缘与接入网

### 网络边缘

端系统所在的网络边缘，是用户直接接触的部分，核心是为应用提供端到端的通信基础。

### 接入网类型

| 接入类型            | 特点                          | 速率特点                         |
| ------------------- | ----------------------------- | -------------------------------- |
| DSL（数字用户线路） | 复用电话线路，非对称          | 上行 < 1Mbps，下行 < 10Mbps      |
| HFC（混合光纤同轴） | 共享电缆线路，频分复用        | 上行~30Mbps，下行~42Mbps         |
| 以太网              | 机构接入，专用线路            | 10Mbps~10Gbps                    |
| 无线接入            | WLAN / 蜂窝网络，共享无线信道 | WiFi: 11~200Mbps，蜂窝：1~10Gbps |

## 1.3 网络核心：交换技术

### 分组交换

- 原理：将报文分割为固定长度的分组，每个分组独立转发，采用**存储-转发**机制。

- 存储-转发：路由器必须接收完整个分组，才能向下一跳转发，单跳发送时延为 `L/R`（L 为分组长度，R 为链路带宽）。

- 排队与丢包：当分组到达速率超过输出链路速率时，分组进入队列排队；队列满时发生丢包。

### 电路交换

- 原理：通信前为端到端分配专用资源（电路），通信结束后释放，类似电话网。

- 资源分配：采用 FDM（频分复用）或 TDM（时分复用）划分信道。

### 两种交换的对比

| 对比维度 | 分组交换                   | 电路交换               |
| -------- | -------------------------- | ---------------------- |
| 资源     | 共享，按需分配             | 专用，预留             |
| 适用场景 | 突发数据业务               | 实时业务（语音）       |
| 优势     | 资源利用率高，支持更多用户 | 性能可预测，无拥塞     |
| 劣势     | 可能拥塞、丢包             | 资源浪费，建立连接开销 |

## 1.4 网络性能指标

### 四种节点时延

分组在路由器的总时延为四部分之和：

`d_nodal = d_proc + d_queue + d_trans + d_prop`

1. **处理时延**：检查比特差错、确定输出链路，微秒级。

2. **排队时延**：分组等待输出链路的时间，取决于拥塞程度，用 **流量强度 `La/R`** 衡量：
   - `La/R -> 0`：排队时延很小

   - `La/R -> 1`：排队时延急剧增大

   - `La/R > 1`：排队时延趋于无穷

3. **发送时延**：将分组所有比特推送到链路的时间，`d_trans = L/R`。

4. **传播时延**：比特在链路中传播的时间，`d_prop = d/s`（d 为链路长度，s 为信号传播速度）。

::: danger 注意

发送时延是“推”数据的时间，传播时延是数据“跑”在链路上的时间，二者完全不同。

:::

### 丢包

路由器队列缓存有限，当队列满时，新到达的分组会被丢弃，即丢包。丢包可由源端重传恢复。

### 吞吐量

端到端的平均传输速率，瓶颈链路（路径上最小带宽的链路）决定了端到端的吞吐量。

::: tip

多连接共享瓶颈链路时，每个连接的吞吐量为 `R/K`（R 为瓶颈带宽，K 为连接数）。

:::

## 1.5 协议分层

### 分层的意义

将复杂系统模块化，每层实现特定服务，下层为上层提供服务，上层无需关心下层细节，便于维护和更新。

### 互联网协议栈（自顶向下）

| 层次       | 作用                 | 典型协议             |
| ---------- | -------------------- | -------------------- |
| **应用层** | 为网络应用提供服务   | HTTP、FTP、SMTP、DNS |
| **传输层** | 进程间的逻辑通信     | TCP、UDP             |
| **网络层** | 主机间的路由转发     | IP、RIP、OSPF、BGP   |
| **链路层** | 相邻节点的帧传输     | 以太网、802.11、PPP  |
| **物理层** | 比特在物理媒介的传输 | 电气 / 光学信号规范  |

### ISO/OSI 参考模型

额外增加了表示层（数据加密、压缩）、会话层（会话同步），这两层功能在互联网中由应用层实现。

### 封装

发送端逐层添加首部，接收端逐层解封装：

- 应用报文 → 添加传输首部 → 报文段 → 添加网络首部 → 数据报 → 添加链路首部 → 帧

- 路由器处理网络层，交换机处理链路层，端系统处理所有层。

## 1.6 互联网发展史

- 1960s：分组交换理论提出，ARPAnet 启动。

- 1970s：网络互连原则提出，TCP/IP 协议诞生，以太网部署。

- 1980s：TCP/IP 部署，DNS、SMTP、FTP 等协议标准化。

- 1990s：Web 出现，商业化，网络规模爆发。

- 2000s：移动互联网、P2P、云服务兴起。

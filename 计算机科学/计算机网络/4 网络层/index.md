# 第 4 章 网络层

网络层的核心是**主机间的逻辑通信**，负责将数据报从源主机路由到目的主机，是互联网的核心层次。

## 4.1 网络层基础

### 核心功能

1. **转发**：将数据报从路由器的输入端口转发到正确的输出端口。
2. **路由**：通过路由算法，确定源到目的的端到端路径。

### 网络服务模型

- **互联网**：尽力而为服务，不保证时延、带宽、有序交付。
- **其他网络**：`ATM` 的 `CBR`/`VBR`/`ABR`/`UBR`，提供不同的服务保证。

### 虚电路 vs 数据报网络

| 对比维度 | 虚电路网络                         | 数据报网络                     |
| -------- | ---------------------------------- | ------------------------------ |
| 连接     | 面向连接，建立虚电路               | 无连接，每个分组独立转发       |
| 地址     | 虚电路号（VC 号）                  | 目的 IP 地址                   |
| 状态     | 路由器维护连接状态                 | 路由器无连接状态               |
| 路由     | 连接建立时确定，所有分组走同一路径 | 每个分组独立选路，路径可能变化 |
| 适用     | 实时业务                           | 突发数据业务                   |

## 4.2 路由器工作原理

路由器的**输入端口、交换结构、输出端口**组成**数据面**，**路由处理器**是**控制面**。

1. **输入端口**：链路层解封装，查找转发表，排队。

2. **交换结构**：
   - **内存交换**：第一代路由器，CPU 拷贝分组，受内存带宽限制。
   - **总线交换**：共享总线，受总线带宽限制。
   - **互联交换**：交叉开关，无共享瓶颈，带宽最高。

3. **输出端口**：缓存排队，链路层封装，发送数据。
   - **缓存大小**：经验法则 `RTT * C`，`RTT` 是往返时间，`C` 是链路带宽。

## 4.3 IP：网际协议

IP 是互联网的网络层核心协议，提供无连接的尽力而为服务。

### IP 数据报格式

- **版本**：IPv4/IPv6。
- **首部长度**：IP 首部长度，单位 4 字节。
- **TOS**：服务类型，用于 QoS。
- **总长度**：数据报总长度，单位字节。
- **标识、标志、片偏移**：用于分片和重组。
- **TTL**：生存时间，每跳减 1，防止环路。
- **协议**：上层协议（TCP/UDP）。
- **首部校验和**：IP 首部校验。
- **源 / 目的 IP 地址**。
- **选项**：可选的扩展字段。

### IP 分片

当数据报长度超过链路 `MTU` 时，路由器将其分为多个分片，目的端重组，每个分片有独立的首部，片偏移指示分片的位置。

### IPv4 编址

IP 地址是 32 位的接口标识符，每个接口有一个 IP 地址，分为网络部分和主机部分。

1. **地址分类**

   | 类别 | 起始位 | 网络位 | 主机位 | 地址范围                        | 私有地址         |
   | ---- | ------ | ------ | ------ | ------------------------------- | ---------------- |
   | A 类 | 0      | 8      | 24     | `0.0.0.0` ~ `127.255.255.255`   | `10.0.0.0/8`     |
   | B 类 | 10     | 16     | 16     | `128.0.0.0` ~ `191.255.255.255` | `172.16.0.0/12`  |
   | C 类 | 110    | 24     | 8      | `192.0.0.0` ~ `223.255.255.255` | `192.168.0.0/16` |
   | D 类 | 1110   | -      | -      | `224.0.0.0` ~ `239.255.255.255` | 组播             |
   | E 类 | 1111   | -      | -      | `240.0.0.0` ~ `255.255.255.255` | 保留             |

2. **子网与 CIDR**
   - **子网**：将大的网络划分为小的子网，减少路由表项。
   - **`CIDR`（无类域间路由）**：网络部分长度任意，用`a.b.c.d/x`表示，最长前缀匹配：匹配目的 IP 的最长前缀的表项。
   - **路由聚合**：将多个子网的路由聚合为一个大的前缀，减少路由表大小。

3. **DHCP：动态主机配置协议**

   主机动态获取 IP 地址、网关、DNS 服务器地址，即插即用：

   ::: info 过程
   DHCP 发现（广播） → DHCP 提供 → DHCP 请求 → DHCP 确认（广播）
   :::

4. **NAT：网络地址转换**

   本地网络用一个公网 IP，所有本地主机共享：
   - NAT 路由器修改外出分组的源 IP / 端口，转换为公网 IP / 端口，维护 NAT 转换表。
   - 入站分组根据转换表，修改目的 IP / 端口，转发给本地主机。
   - **NAT 穿透**：UPnP、中继等方案，解决外部无法访问本地主机的问题。

### IPv6

为了解决 IPv4 地址耗尽的问题，IPv6 采用 128 位地址空间：

- 首部固定 40 字节，无分片、无校验和，处理更快。
- **流标签**：支持 `QoS`。
- **隧道技术**：IPv6 分组封装在 IPv4 分组中，穿越 IPv4 网络。

## 4.4 路由算法

路由算法的目标是找到源到目的的最小开销路径，将网络抽象为图，节点是路由器，边是链路。

### 算法分类

- **链路状态（`LS`）**：所有节点知道完整的拓扑，泛洪链路状态信息。
- **距离矢量（`DV`）**：节点只知道邻居，迭代更新距离矢量。

### 链路状态算法：Dijkstra

- 所有节点运行相同的算法，计算自己的最短路径树。
- **步骤**：初始化距离，迭代找到最近的节点，更新距离，直到所有节点处理完。
- **复杂度**：$O(n^2)$，收敛快，无计数到无穷问题。

### 距离矢量算法：Bellman-Ford

- 节点维护到所有目的的距离矢量，和邻居交换矢量，迭代更新。
- 好消息传得快，坏消息传得慢，存在**计数到无穷**的问题，毒性逆转可以解决两个节点的环路，无法解决三个以上的。
- 收敛慢，可能有路由环路。

### 分层路由

互联网太大，无法用单一的路由算法，因此划分为自治系统（`AS`）：

- **域内路由**：`AS` 内部的路由，IGP 协议。
- **域间路由**：`AS` 之间的路由，BGP 协议。

## 4.5 互联网路由协议

### 域内路由协议

1. **RIP：路由信息协议**
   - 距离矢量算法，跳数为度量，最大 15 跳。
   - 每 30 秒和邻居交换路由表，适合小网络。

2. **OSPF：开放式最短路径优先**
   - 链路状态算法，泛洪链路状态信息。
   - 支持认证、多路径、TOS 度量、分层区域，适合大网络。

### 域间路由协议：BGP

BGP 是互联网的域间路由协议，是互联网的粘合剂：

- **`eBGP`**：`AS` 之间交换可达性信息。
- **`iBGP`**：`AS` 内部传播 `BGP` 路由信息。
- **路径属性**：`AS-PATH`（经过的 `AS`）、`NEXT-HOP`（下一跳）。
- **路由选择**：基于本地偏好、`AS` 路径长度、热土豆路由等策略，不是最短路径。

## 4.6 网络管理

- **ICMP**：互联网控制报文协议，用于网络层的控制信息，ping、traceroute 都是基于 ICMP。
- **SNMP**：简单网络管理协议，用于管理网络设备，MIB 存储管理数据，代理和管理站交互。

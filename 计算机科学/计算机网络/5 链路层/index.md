# 第 5 章 链路层

链路层的核心任务是实现**相邻节点之间的帧传输**。如果说网络层关注的是“数据报如何跨多个网络到达远方主机”，那么链路层关注的就是：

- 在当前这一段链路上，数据如何被封装成帧
- 相邻节点如何共享这段链路
- 帧如何被正确识别、检测错误并交付给下一跳

这一章最容易让初学者困惑的地方在于：链路层的内容很杂，既有检错、纠错，也有以太网、ARP、交换机、VLAN、MPLS。学习时建议抓住三条主线：

- **服务主线**：链路层向网络层提供什么服务
- **局域网主线**：MAC 地址、ARP、以太网、交换机、VLAN 如何配合
- **共享媒介主线**：多路访问协议如何解决“大家同时想发”的问题

::: info 学习提示

链路层看起来杂，是因为它同时在处理三类问题：

- 一跳上怎样封装和传输帧
- 多个节点怎样共享同一条链路
- 局域网里怎样完成寻址与转发

把这三条线分开看，后面遇到 ARP、交换机、VLAN 时就不会觉得跳跃。

:::

## 5.1 链路层概述

### 链路层解决什么问题

链路层负责把网络层交下来的 **IP 数据报**，封装为可以在某一条具体链路上传输的**帧（Frame）**，并把它从一个节点送到物理相邻的另一个节点。

因此，链路层不是负责“端到端”传输，而是负责：

- **一跳一跳地传**
- **在每一段链路上各自处理**

### 节点、链路、帧

几个基础术语先分清：

- **节点**：主机、路由器、交换机等设备
- **链路**：连接相邻节点的通信信道，可以是有线、无线等
- **帧**：链路层的数据单位，用来承载网络层数据报

### 为什么不同链路需要不同的链路层协议

一条端到端路径上，往往会经过多种不同链路，例如：

- 主机到家庭路由器可能是以太网或 Wi-Fi
- 家庭路由器到运营商网络可能是光纤或电缆接入
- 核心网络内部可能使用另一类链路技术

因此：

- 同一个 IP 数据报
- 在不同链路上
- 可能被封装成不同格式的帧

这也是为什么链路层协议往往具有“强场景性”。

::: tip 一跳一换协议

链路层最值得先建立的直觉是：

- 端到端前进的是网络层数据报
- 每一跳重新封装的是链路层帧

所以同一个 IP 数据报在不同链路上，外面的帧格式完全可能不同。

:::

### 链路层通常实现在哪里

主机中的链路层通常由**网络适配器（网卡，NIC）** 来实现，往往由：

- 硬件
- 固件
- 驱动软件

共同完成。

网卡一般同时负责：

- 链路层功能
- 物理层功能

例如以太网卡、无线网卡等。

## 5.2 链路层提供的服务

### 成帧

链路层首先要做的是**成帧（Framing）**：

- 把网络层数据报前后加上首部和尾部
- 形成一个完整的帧

成帧的意义包括：

- 标识帧的起始和结束
- 携带源地址和目的地址
- 携带差错检测信息

### 透明传输

如果数据部分本身恰好出现了“像帧开始符、帧结束符”那样的特殊比特或字节序列，会发生什么？

为了避免接收方误判帧边界，就需要**透明传输**机制，例如：

- 字节填充
- 比特填充

它们的目标都是一样的：

- 让“控制信息看起来像控制信息”
- 让“普通数据即使长得像控制信息，也不会被误解”

### 链路访问

如果一条链路是共享媒介，例如老式总线以太网、无线局域网，那么多个节点可能同时想发送。

此时链路层还必须解决：

- 谁什么时候可以发
- 冲突发生时怎么办

这就是**链路访问控制（MAC, Medium Access Control）** 的问题。

### 差错检测与差错纠正

物理信号在传输中可能受噪声、衰减、干扰影响，导致比特错误。

因此链路层通常会提供：

- **差错检测**：发现是否出错
- **差错纠正**：有些情况下可直接纠正错误，而不用重传

### 可靠交付

有些链路层协议还会在**相邻节点之间**提供可靠交付，即：

- 帧出错或丢失后
- 由链路两端处理重传

但要注意：

- 这不等同于传输层的端到端可靠性
- 它只是“这一跳上尽量可靠”

### 为什么还需要链路级可靠，而不是只要 TCP 就够了

这是个很经典的问题。

原因是：

- 某些链路（尤其无线链路）误码率更高
- 如果每次都等端到端超时重传，代价可能太大
- 在本地链路快速修复错误，有时更高效

所以链路级可靠和端到端可靠并不冲突，它们解决的是不同层次的问题。

::: warning 差错检测、纠错、可靠交付不要混

这三者含义不同：

- **差错检测**：发现错了没有
- **差错纠正**：尽量在本地把错修好
- **可靠交付**：确保这一跳最终正确送达

它们常一起出现，但不是同一个概念。

:::

### 流量控制与双工方式

链路层在某些场景下还会涉及：

- **流量控制**：协调相邻节点收发速率
- **半双工**：同一时刻不能双向同时发送
- **全双工**：双方可以同时发送

例如交换机连接的现代以太网链路通常支持全双工，而早期共享以太网更接近半双工环境。

## 5.3 差错检测与纠错

### 差错检测不是绝对可靠的

差错检测的本质是：

- 发送方根据数据计算出冗余信息
- 接收方重新计算并比对
- 若不一致，则说明检测到错误

但要注意：

- **差错检测并不是 100% 可靠**
- 只是“漏检概率足够小”

通常：

- 冗余位越强
- 检错能力越好
- 但开销也越大

### 奇偶校验

最简单的检错方法是**奇偶校验**。

#### 一维奇偶校验

- 给一组比特增加 1 位奇偶校验位
- 可检测单比特错误

但它能力有限，对多个比特错误的检测能力较弱。

#### 二维奇偶校验

- 对行和列都做奇偶校验

这样不但能检测更多错误，在某些情况下还能定位并纠正单比特错误。

### 校验和

**校验和（Checksum）** 在传输层和网络层中也常见。

它的基本思想是：

- 把数据看成若干固定长度的整数
- 进行补码和运算
- 把结果作为校验字段发送

接收方再重新计算并核对。

它实现简单，但检错能力通常不如 CRC 强。

### CRC：循环冗余校验

**CRC（Cyclic Redundancy Check）** 是链路层中非常经典、非常强大的差错检测技术。

它的基本思想是：

- 把数据比特串看成一个二进制多项式
- 选择一个生成多项式 `G`
- 添加若干 CRC 比特，使得整个比特串能被 `G` 模 2 整除

接收方用同样的 `G` 去除：

- 若余数不为 0，则检测到错误

### CRC 为什么重要

CRC 被广泛使用，是因为它对很多常见错误模式都很敏感，尤其对**突发错误**检测能力很强。

教材中常见结论是：

- 它可以检测所有长度小于 `r + 1` 的突发错误

因此 CRC 被广泛用于：

- 以太网
- Wi-Fi
- ATM 等链路技术

## 5.4 多路访问协议

### 为什么会有多路访问问题

如果多台主机共享同一条广播信道，那么它们在同一时刻都可能想发送数据。

一旦两个或更多节点同时发送，就会发生：

- **冲突（Collision）**

冲突意味着：

- 信号互相干扰
- 这次发送作废
- 信道资源被浪费

因此必须设计协议来决定：

- 谁什么时候发
- 冲突后怎么恢复

### 理想多路访问协议希望达到什么

对于速率为 `R` 的共享信道，一个理想的多路访问协议希望：

- 只有一个节点发送时，它能以速率 `R` 发送
- 有 `M` 个节点都想发送时，每个节点平均得到 `R/M`
- 协议尽量简单
- 尽量去中心化

现实协议通常只能在这些目标之间权衡。

### 三大类 MAC 协议

多路访问协议大致可分为三类：

1. **信道划分协议**
2. **随机接入协议**
3. **轮流接入协议**

## 5.5 信道划分协议

### TDMA

**TDMA（时分多址）** 的思想是：

- 把时间划分成一个个时隙
- 每个节点分到固定时隙
- 只在自己的时隙里发送

优点：

- 高负载下公平
- 冲突少

缺点：

- 某节点暂时没数据时，它的时隙也可能空着，造成浪费
- 即使只有一个活跃节点，也不一定能独占整条链路

### FDMA

**FDMA（频分多址）** 的思想是：

- 把信道频谱划分为多个频带
- 每个节点长期占用其中一段频带

它和 TDMA 很像，只不过切分的不是时间，而是频率资源。

## 5.6 随机接入协议

### 随机接入的思想

随机接入协议不事先把信道切好，而是允许节点：

- 有数据就尝试发
- 若发生冲突，再想办法恢复

它的优点是：

- 低负载时效率较高
- 单个活跃节点可以充分利用信道

缺点是：

- 高负载下冲突开销会明显上升

### ALOHA

最原始的随机接入思想就是 **ALOHA**：

- 节点有帧就发
- 发生冲突就等待一段随机时间后重传

它非常简单，但效率较低。

### 时隙 ALOHA

**时隙 ALOHA** 在 ALOHA 基础上增加了同步要求：

- 时间被划分为离散时隙
- 节点只能在时隙开始时发送

这样做后，冲突窗口更小，因此效率提升到大约：

`0.37`

### 纯 ALOHA

纯 ALOHA 不要求时隙同步：

- 有帧就立即发送

虽然更简单，但冲突概率更高，最大效率只有大约：

`0.18`

因此它比时隙 ALOHA 更低效。

### CSMA：先听后发

**CSMA（Carrier Sense Multiple Access）** 的核心思想是：

- 发送前先侦听信道
- 如果信道空闲，就发
- 如果信道忙，就等

这和日常说话很像：

- 别人正在说，就先别抢话

### 为什么 CSMA 仍然会冲突

即使“先听后发”，冲突仍可能发生，因为：

- 信号传播需要时间
- 两个相距较远的节点，可能都误以为信道空闲
- 结果同时开始发送

这说明问题不在“听不听”，而在于：

- **传播时延让信息来不及传到所有节点**

### CSMA/CD：碰撞检测

**CSMA/CD（Collision Detection）** 在 CSMA 基础上进一步要求：

- 边发边检测
- 一旦发现碰撞，立即停止发送

这样可以减少整帧传完才发现冲突的浪费。

### 二进制指数退避

碰撞后不能立刻再次发送，否则还会反复撞车。

因此以太网中的 CSMA/CD 使用**二进制指数退避**：

- 第 `m` 次碰撞后
- 从更大的随机等待区间中选一个退避时间
- 等待后再重传

碰撞越多，等待范围越大，有助于把多个竞争者逐渐错开。

### 为什么无线里常讲 CSMA/CA，而不是 CSMA/CD

在无线环境中：

- 节点自己的发送信号往往非常强
- 容易盖过接收到的其他微弱信号

所以“边发边准确检测碰撞”比较困难。

因此无线局域网更常使用：

- **CSMA/CA（Collision Avoidance，碰撞避免）**

而不是 CSMA/CD。

## 5.7 轮流接入协议

### 轮询

**轮询（Polling）** 的做法是：

- 由一个主节点依次询问各从节点
- 被点到的节点才可以发送

优点：

- 冲突少
- 高负载时比较稳定

缺点：

- 有轮询开销
- 主节点可能成为单点故障

### 令牌传递

**令牌传递（Token Passing）** 的做法是：

- 令牌在节点间依次传递
- 拿到令牌的节点才可以发送

优点：

- 发送权明确
- 冲突少

缺点：

- 有令牌维护开销
- 令牌丢失或损坏会影响系统运行

### 三类协议的直觉对比

- **信道划分**：高负载时好，低负载时容易浪费
- **随机接入**：低负载时好，高负载时冲突多
- **轮流接入**：试图折中两者优点，但需要额外控制机制

## 5.8 局域网寻址：MAC 地址与 ARP

### IP 地址和 MAC 地址不要混

这是链路层中最容易混淆的地方之一。

- **IP 地址**：网络层地址，用于跨网络转发
- **MAC 地址**：链路层地址，用于本地链路上的帧传输

### MAC 地址是什么

MAC 地址通常是 **48 位**，常见写法如：

`1A-2F-BB-76-09-AD`

它通常具有这些特点：

- 存在于网卡中
- 在局域网通信中标识接口
- 是扁平地址，不像 IP 那样带网络层次结构

### IP 地址与 MAC 地址的差别

可以这样理解：

- **IP 地址** 更像“邮政地址”，决定跨网络怎么送
- **MAC 地址** 更像“当前这一栋楼里的门牌”，决定本地这一步交给谁

另外：

- IP 地址通常和所处子网有关，换网可能变化
- MAC 地址通常跟着网卡走，移动网卡时地址本身不变

### ARP 是什么

**ARP（Address Resolution Protocol，地址解析协议）** 的任务是：

- 已知目标 IP 地址
- 找到与之对应的 MAC 地址

### ARP 表

主机和路由器通常会维护一张 **ARP 表**，记录：

- `IP 地址 -> MAC 地址`

表项通常还带有：

- **TTL**

也就是过一段时间若不再使用，这条映射会被遗忘。

### ARP 工作过程

如果主机 A 想给同一局域网中的主机 B 发帧，但不知道 B 的 MAC 地址，就会：

1. 查询本地 ARP 表
2. 若没有，就发送 **ARP 广播请求**
3. 局域网中所有节点都能收到该广播
4. 拥有目标 IP 的主机 B 返回 **ARP 单播应答**
5. A 记住这条映射，并发送后续帧

### ARP 为什么像“即插即用”

ARP 的一个重要优点是：

- 不需要管理员手工配置每个主机的映射关系
- 节点自己就能动态建立 ARP 表

这也是局域网能方便接入和移动的关键原因之一。

### 跨局域网发送时，ARP 查的是谁

这是非常容易出错的点。

如果 A 要给另一个局域网中的 B 发 IP 数据报：

- IP 层中的目标地址仍然是 **B 的 IP**
- 但 A 在本地链路上封装帧时，目的 MAC 不是 B，而是**第一跳路由器接口的 MAC**

也就是说：

- **跨网传输时，链路层只关心“下一跳”**
- 不是直接关心“最终目的主机”

等数据报到了路由器，路由器再在下一段链路上重新封装新的帧。

::: info ARP 查的不是“最终目的主机”

只要跨了局域网，发送方在本地链路上真正要找的通常是：

- **下一跳路由器接口的 MAC 地址**

所以很多题目里：

- IP 目的地址没有变
- 但 MAC 目的地址会在每一跳重新变化

:::

## 5.9 以太网

### 以太网为什么重要

以太网是最流行的局域网技术之一，原因在于：

- 便宜
- 简单
- 扩展性强
- 速率持续演进

### 以太网的拓扑演进

早期以太网常见的是：

- **总线型拓扑**

所有节点共享一条总线，处于同一冲突域。

现代以太网更常见的是：

- **星型拓扑**

以交换机为中心，每台主机通过独立链路接到交换机上。

### 以太网帧结构

以太网帧常见字段包括：

- **前同步码 / 帧开始定界**
- **目的 MAC 地址**
- **源 MAC 地址**
- **类型字段**
- **数据（Payload）**
- **CRC**

其中：

- 目的地址和源地址用于本地交付
- 类型字段说明上层承载的是什么协议，例如 IP
- CRC 用于检错

### 以太网的几个基本性质

以太网通常被概括为：

- **无连接**
- **不可靠**

无连接表示：

- 发送前不需要先握手

不可靠表示：

- 网卡收到错帧时通常直接丢弃
- 不会由以太网本身去做端到端重传确认

因此丢失的数据是否恢复，通常由更高层决定，例如 TCP。

### CSMA/CD 在以太网中的角色

在传统共享以太网中，以太网使用：

- **无槽 CSMA/CD**

来处理共享媒介上的竞争访问。

但在现代交换式、全双工以太网中：

- 各链路彼此独立
- 不再是多个主机共享同一物理媒介

因此冲突问题大大减轻，CSMA/CD 的实际存在感也比早期低得多。

::: info `CSMA/CD` 和 `CSMA/CA`

可以把它们先粗略区分成：

- `CSMA/CD`：更典型地出现在早期共享有线环境
- `CSMA/CA`：更典型地出现在无线局域网环境

背后的关键差异不是字母，而是“无线里很难像有线那样边发边可靠检测碰撞”。

:::

## 5.10 以太网交换机

### 交换机是什么

交换机是典型的 **链路层设备**，它的核心特点是：

- 存储转发以太网帧
- 根据 MAC 地址进行转发

### 交换机和路由器的区别

两者都属于“存储转发”设备，但关注层次不同：

- **交换机**：看链路层首部，依据 MAC 地址转发帧
- **路由器**：看网络层首部，依据 IP 地址转发数据报

::: tip 交换机和路由器怎么快速区分

- 看到 **MAC 地址 / 帧 / 二层**，优先想到交换机
- 看到 **IP 地址 / 数据报 / 三层**，优先想到路由器

这是很多综合题里最基础的一步判断。

:::

### 交换机的优势

交换机带来的好处很大：

- 每条链路通常是独立冲突域
- 多对主机可同时通信
- 常支持全双工
- 主机对交换机的存在通常无感知，具有透明性

### 交换机为什么不需要手工配置全部转发表

因为交换机支持**自学习（Self-Learning）**。

其核心思想是：

- 交换机收到一个帧时
- 观察这个帧的**源 MAC 地址**
- 记住“这个源地址是从哪个接口进来的”

于是逐渐形成交换表。

### 交换机如何转发

交换机收到帧后，通常会做三步：

1. 学习源 MAC 与入接口的对应关系
2. 查找目的 MAC 是否在交换表中
3. 根据查找结果决定转发方式

具体来说：

- 若找到且目标就在入接口所在网段，则**过滤**，不用转发
- 若找到且在其他接口，则**定向转发**
- 若没找到，则**泛洪**到除入接口外的所有接口

### 交换机互联

多个交换机互联时，工作原理本质上并没有变：

- 仍然是自学习
- 仍然是根据 MAC 地址转发
- 仍然会在未知目的时进行泛洪

这使得较大的二层网络可以自动逐步学习和工作。

## 5.11 VLAN：虚拟局域网

### 为什么需要 VLAN

如果一个交换网络过大，会带来几个问题：

- 广播域太大
- ARP、DHCP 等广播流量会扩散到整个 LAN
- 安全隔离和管理不方便
- 用户移动位置后，网络归属可能不好调整

这就是引入 **VLAN（Virtual LAN）** 的动机。

### VLAN 的核心思想

VLAN 让一台物理交换机看起来像多台逻辑交换机：

- 不同端口可以被划入不同 VLAN
- 同一 VLAN 内像在一个独立局域网中
- 不同 VLAN 之间默认二层隔离

### 基于端口的 VLAN

最常见的是**基于端口的 VLAN**：

- 哪些端口属于同一 VLAN，由交换机配置决定

这样即使物理上在同一个交换机里，逻辑上也可以属于不同网络部门。

### VLAN 的价值

VLAN 带来的典型好处有：

- 隔离广播流量
- 提高安全性和私密性
- 便于按部门、业务划分网络
- 用户位置变动时更灵活

### 跨交换机的 VLAN

如果同一个 VLAN 跨越多个交换机，就需要在交换机之间传递：

- 这个帧属于哪个 VLAN

普通以太网帧本身不带 VLAN ID，因此需要额外机制。

### 802.1Q

**802.1Q** 就是为 VLAN 帧增加标签的标准。

它会在帧中加入 VLAN 相关字段，使交换机之间的**汇聚链路（Trunk）** 能识别：

- 该帧属于哪个 VLAN

### VLAN 之间如何通信

不同 VLAN 本质上像不同的局域网。

因此：

- **VLAN 内通信** 可以由交换机完成
- **VLAN 间通信** 通常需要借助路由器或三层交换能力

## 5.12 MPLS：链路虚拟化

### MPLS 是什么

**MPLS（Multi-Protocol Label Switching，多协议标签交换）** 的基本思想是：

- 在分组前面加一个较短的标签
- 转发时优先依据标签，而不是直接做完整的 IP 最长前缀匹配

### MPLS 的动机

MPLS 最初的重要动机包括：

- 希望转发更快
- 希望更灵活地控制路径
- 借鉴虚电路的一些思路

### MPLS 的特点

在 MPLS 网络中：

- 支持标签交换路由器根据标签转发
- 可实现流量工程
- 可支持快速重路由
- 可根据策略让不同流量走不同路径

因此 MPLS 不只是“提速技巧”，还提供了更强的路径控制能力。

## 5.13 一次网页请求中的链路层

### 为什么这一节重要

把协议分层学完之后，最容易出现的问题是：

- 每层都单独会背
- 但一到完整流程就不知道谁先做什么

所以这一节的重点不是新知识，而是把前面几章串起来。

### 访问网页时，链路层做了哪些事

假设用户第一次接入网络并访问一个网站，大致过程可能是：

1. **DHCP**
   - 主机先获取自己的 IP 地址、子网掩码、默认网关、DNS 服务器地址

2. **ARP**
   - 若要把分组交给默认网关，主机需要先知道网关接口的 MAC 地址

3. **DNS**
   - 查询目标网站域名对应的 IP 地址

4. **TCP 三次握手**
   - 与目标服务器建立传输层连接

5. **HTTP 请求 / 响应**
   - 浏览器发送请求并接收网页内容

### 链路层在这个过程中反复出现

虽然应用层上看你只是在“打开一个网页”，但每经过一跳：

- IP 数据报都会被重新封装成新的链路层帧
- 使用新的源 MAC、目的 MAC
- 在当前链路上传送到下一跳

这正体现了链路层的本质：

- **端到端的数据报不变地向前推进**
- **但每一跳上的帧封装都可能变化**

## 本章小结

学完链路层后，建议你至少能回答这些问题：

1. 链路层为什么只解决“相邻节点之间”的问题
2. 成帧和透明传输分别解决什么问题
3. 差错检测、差错纠正、可靠交付之间有什么区别
4. `TDMA`、`FDMA`、`ALOHA`、`CSMA`、`CSMA/CD`、轮询、令牌传递分别适合什么思路
5. 为什么 `CSMA` 明明先听后发，还是可能发生冲突
6. IP 地址和 MAC 地址分别起什么作用
7. 为什么跨局域网发送时，帧的目的 MAC 往往是下一跳路由器
8. 交换机如何通过自学习建立转发表
9. VLAN 为什么能隔离广播域，又为什么 VLAN 之间通信仍需要路由
10. MPLS 为什么说借鉴了虚电路思想

::: details 参考答案

1. 链路层只解决相邻节点之间的问题，因为它面向的是单条链路上的一跳传输；端到端通信要靠上层逐跳配合完成。
2. 成帧解决“如何把上层数据组织成可在链路上传送的帧”；

   透明传输解决“如何让帧中出现的特殊比特模式不被误认为控制信息”。

3. 差错检测是发现是否出错，

   差错纠正是尝试恢复错误，

   可靠交付是确保这一跳最终正确送达。

4. `TDMA`、`FDMA` 适合把信道预先划分给不同用户；

   `ALOHA`、`CSMA`、`CSMA/CD` 属于随机接入思路；

   轮询、令牌传递属于轮流接入思路。

5. 为什么 `CSMA` 先听后发仍可能冲突，因为不同站点感知到的信道状态存在传播时延，两个站点可能几乎同时判断“空闲”并发送。
6. IP 地址用于网络层标识主机或接口并支持跨网络转发；

   MAC 地址用于链路层在本地网络中标识网卡接口。

7. 跨局域网发送时，本地帧只需要先到下一跳，因此帧头中的目的 MAC 往往是默认网关或下一跳路由器接口的 MAC。
8. 交换机通过观察收到帧的源 MAC 地址和入端口，逐步学习“某个 MAC 在哪个端口后面”，从而建立转发表。
9. VLAN 通过逻辑划分广播域，让不同 VLAN 的广播彼此隔离；
   但 VLAN 之间若要通信，本质上已经是跨三层边界，因此仍需要路由设备参与。
10. MPLS 借鉴虚电路思想，是因为它在分组进入 MPLS 网络时先绑定标签，后续节点依据标签而不是完整 IP 前缀进行转发。

:::

如果这些问题已经能比较顺畅地解释出来，说明你对链路层已经建立起了比较扎实的整体框架。

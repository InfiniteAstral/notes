# 第 5 章 链路层

链路层的核心是**相邻节点的帧传输**，将 IP 数据报封装为帧，在链路上传输。

## 5.1 链路层基础

- **节点**：主机、路由器、交换机。
- **链路**：相邻节点之间的信道，有线 / 无线。
- **帧**：链路层的分组，封装 IP 数据报。
- **适配器**：网卡，实现链路层和物理层的功能。

### 链路层服务

- **成帧**：封装 IP 数据报为帧，添加首部尾部。
- **链路访问**：共享信道的访问控制。
- **差错检测 / 纠正**：检测传输中的比特错误。
- **可靠传输**：相邻节点的可靠交付。
- **流量控制**：相邻节点的速率匹配。

## 5.2 差错检测与纠错

- **奇偶校验**：检测单比特错误。
- **校验和**：将数据视为 16 位序列，补码和，用于 UDP/TCP/IP。
- **CRC（循环冗余校验）**：更强大的检错，基于生成多项式，能检测所有小于 `r + 1` 比特的突发错误，广泛用于以太网、WiFi。

## 5.3 多路访问协议

共享广播信道下，协调节点的传输，避免冲突：

### 信道划分协议

- **TDMA**：时分多址，将时间划分为时隙，每个节点分配时隙。
- **FDMA**：频分多址，将频谱划分为频带，每个节点分配频带。
- **优点**：高负载时公平高效，缺点：低负载时利用率低，单个节点无法用全部带宽。

### 随机接入协议

- **ALOHA**：有数据就发，冲突了就重传，效率只有 18%。
- **时隙 ALOHA**：时间划分为时隙，节点只能在时隙开始发送，效率 37%。
- **CSMA**：载波侦听多址，先听后发，但是传播时延导致冲突仍然可能发生。
- **CSMA/CD**：载波侦听碰撞检测，发送时检测碰撞，碰撞了就中止，二进制指数退避，用于以太网。

### 轮流接入协议

- **轮询**：主节点轮询从节点，依次传输。
- **令牌传递**：令牌依次传递，拿到令牌才能发送。
- 结合了信道划分和随机接入的优点，高低负载都有不错的性能。

## 5.4 局域网技术

### MAC 地址与 ARP

- **MAC 地址**：48 位的物理地址，烧录在网卡，全局唯一，扁平结构，可携带。

- **ARP**：地址解析协议，将 IP 地址解析为 MAC 地址：
  - 本地 ARP 表缓存 IP-MAC 映射，TTL 超时。
  - 广播 ARP 查询，目的主机单播 ARP 回复。

### 以太网

以太网是最流行的局域网技术，便宜、简单、可扩展。

- **拓扑**：星型拓扑，交换机居中，每个链路独立，无冲突。
- **帧结构**：前同步码、目的 MAC、源 MAC、类型、数据、CRC。
- **CSMA/CD**：二进制指数退避，半双工下的碰撞处理。
- 无连接、不可靠，上层负责可靠传输。

### 以太网交换机

链路层的存储转发设备，转发以太网帧：

- **自学习**：收到帧时，学习源 MAC 的接口，维护转发表。
- **转发 / 过滤**：根据目的 MAC，转发到对应的接口，泛洪未知目的的帧。
- **隔离冲突**：每个链路是独立的冲突域，支持全双工，同时传输。
- **透明**：主机感知不到交换机的存在，即插即用。

### VLAN：虚拟局域网

将物理交换机划分为多个虚拟的局域网，隔离广播流量：

- **基于端口的 VLAN**：端口分组，属于同一个 VLAN 的端口才能互相通信。
- **`802.1Q`**：VLAN 帧格式，添加 VLAN 标签，跨交换机传输时携带 VLAN ID。

## 5.5 MPLS：多协议标签交换

- 基于标签的转发，用短标签代替 IP 地址，转发更快。
- 支持快速重路由、源路由，用于 VPN、流量工程。

## 5.6 网页请求的完整生命周期

一次访问 Google 的完整过程，覆盖所有层的协议：

1. **DHCP**：主机获取 IP、网关、DNS 地址。
2. **ARP**：获取网关的 MAC 地址。
3. **DNS**：查询 [www.google.com](https://www.google.com) 的 IP 地址。
4. **TCP 三次握手**：建立到 Web 服务器的 TCP 连接。
5. **HTTP**：发送请求，接收响应，显示网页。

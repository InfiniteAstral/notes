# 第 4 章 存储器

## 例题部分

### <span id="例-4-1">例 4.1</span>：设 CPU 有 $16$ 根地址线、$8$ 根数据线，并用 $\overline{\text{MREQ}}$ 作为访存控制信号（低电平有效），用 $\overline{\text{WR}}$ 作为读/写控制信号（高电平为读，低电平为写）。现有下列存储芯片：$1\,\text{K}\times 4$ 位 RAM、$4\,\text{K}\times 8$ 位 RAM、$8\,\text{K}\times 8$ 位 RAM、$2\,\text{K}\times 8$ 位 ROM、$4\,\text{K}\times 8$ 位 ROM、$8\,\text{K}\times 8$ 位 ROM 及 74138 译码器和各种门电路，如图所示。![译码器和门电路](https://static.owo.cab/notes/image/cs/poco/chapter04/译码器和门电路.webp)画出 CPU 与存储器的连接图，要求如下：<ol><li>主存地址空间分配：<br>$6000\text{H}\sim 67\text{FF}\text{H}$ 为系统程序区。<br>$6800\text{H}\sim 6\text{BFF}\text{H}$ 为用户程序区。</li><li>合理选用上述存储芯片，说明各选几片。</li><li>详细画出存储芯片的片选逻辑图。</li></ol>

:::: details 知识点讲解、元件解释和解题方法

这道题虽然表面上是在画 CPU 与存储器的连接图，但真正考查的是一整套前置知识能否串起来使用。读者如果第一次接触这类题，最容易出现的情况是：每个词都认识，但不知道为什么先算容量、为什么要选 ROM 而不是 RAM、为什么有些地址线直接接芯片，有些却要送去译码器。

所以在正式进入后面的“四步解题”之前，最好先按下面几个层次建立整体认识。

1. **这道题到底在做什么**

   题目并不是单纯要求“画图”，而是要求你同时完成三件事：
   - 分配主存地址空间；
   - 根据用途和容量合理选芯片；
   - 设计片选逻辑并画出连接关系。

   因而整体思路应当是：

   **题目给出地址范围和器件条件 → 先算各区容量 → 再按功能和位宽选芯片 → 决定哪些地址线做片内寻址、哪些地址线做片选 → 最后组合出片选逻辑并画图。**

2. **先读懂题目给出的已知条件**

   CPU 有 $16$ 根地址线，表示它最多能形成

   $$
   2^{16}=65\,536=64\,\text{K}
   $$

   个不同地址。若结合本题 $8$ 位数据总线来理解，就是从 $0000\text{H}$ 到 $\text{FFFF}\text{H}$ 的 $64\,\text{KB}$ 编址空间。

   CPU 有 $8$ 根数据线，表示一次并行传送 $8$ 位，也就是 $1$ 字节，因此题中各地址区的容量都可以按字节理解。

   两个控制信号也都很关键：
   - $\overline{\text{MREQ}}$ 是访存控制信号，且低电平有效。只有当它为 $0$ 时，CPU 才真的在访问主存，所以它常被直接拿来参与片选逻辑。
   - $\overline{\text{WR}}$ 是读写控制信号，高电平表示读，低电平表示写。这个信号对 RAM 尤其重要，因为 RAM 既支持读，也支持写。

3. **再读懂存储芯片规格是什么意思**

   类似 $1\,\text{K}\times 4$、$2\,\text{K}\times 8$、$8\,\text{K}\times 8$ 的写法中，前面的数表示有多少个存储单元，后面的数表示每个存储单元有多少位。

   例如：
   - $1\,\text{K}\times 4$ 表示有 $1024$ 个单元，每个单元 $4$ 位；
   - $2\,\text{K}\times 8$ 表示有 $2048$ 个单元，每个单元 $8$ 位。

   这里必须区分两个量：
   - **单元数**决定芯片内部需要多少根地址线；
   - **位宽**决定芯片要占用多少根数据线。

   因此：
   - $1\,\text{K}=2^{10}$，所以 $1\,\text{K}$ 芯片需要 $10$ 根地址线；
   - $2\,\text{K}=2^{11}$，所以 $2\,\text{K}$ 芯片需要 $11$ 根地址线；
   - $4\,\text{K}=2^{12}$，所以 $4\,\text{K}$ 芯片需要 $12$ 根地址线。

   后面画图时，正是据此决定 $A_{10}\sim A_0$ 或 $A_9\sim A_0$ 应接到哪些芯片上。

4. **先算地址区容量，再判断需要多大的存储块**

   存储器区间容量的计算公式是

   $$
   \text{容量}=\text{末地址}-\text{首地址}+1
   $$

   原因是首地址和末地址都属于这个区间。

   本题中：
   - 系统程序区为 $6000\text{H}\sim 67\text{FF}\text{H}$，容量为
     $$
     67\text{FF}\text{H}-6000\text{H}+1=0800\text{H}=2048=2\,\text{K}
     $$
     即 $2\,\text{K}\times 8$ 位；
   - 用户程序区为 $6800\text{H}\sim 6\text{BFF}\text{H}$，容量为
     $$
     6\text{BFF}\text{H}-6800\text{H}+1=0400\text{H}=1024=1\,\text{K}
     $$
     即 $1\,\text{K}\times 8$ 位。

   因为本题按字节编址，所以也可以直接看作：
   - 系统程序区为 $2\,\text{KB}$；
   - 用户程序区为 $1\,\text{KB}$。

5. **除了算容量，还要观察固定不变的高位地址**

   真正负责“选中哪一片芯片”的，往往不是全部地址线，而是某一段地址区中保持不变的高位地址。

   例如：
   - 对于 $6000\text{H}\sim 67\text{FF}\text{H}$，可归纳出
     $$
     A_{15}A_{14}A_{13}A_{12}A_{11}=0\,1\,1\,0\,0
     $$
   - 对于 $6800\text{H}\sim 6\text{BFF}\text{H}$，可归纳出
     $$
     A_{15}A_{14}A_{13}A_{12}A_{11}=0\,1\,1\,0\,1
     $$

   低位地址在区间内变化，高位地址却稳定不变，这正是后面把高位地址送入 74138 做译码的依据。

6. **选片时必须同时考虑“容量”和“功能”**

   ROM 适合存放系统程序、监控程序、固化程序这类运行中通常不应被改写的内容；RAM 适合存放用户程序、临时数据和运行结果，因为它支持读写。

   所以本题中：
   - 系统程序区应优先放在 ROM 中；
   - 用户程序区应优先放在 RAM 中。

   如果只按容量选片，不看存储器类型是否合适，逻辑上就已经错了。

7. **理解两种基本操作：扩位与扩容**

   **扩位**是指芯片位宽不够时，用多片芯片并联，共用地址线，分别承担不同的数据位；**扩容**是指芯片单元数不够时，用多片芯片分担不同地址段。

   本题最典型的是扩位：CPU 数据总线是 $8$ 位，而给定 RAM 中没有 $1\,\text{K}\times 8$，只有 $1\,\text{K}\times 4$。因此要把两片 $1\,\text{K}\times 4$ RAM 并联起来：
   - 一片负责高 $4$ 位数据；
   - 一片负责低 $4$ 位数据；
   - 两片共用同一组地址线，因此访问的是同一个地址。

   所以两片合起来得到的是 $1\,\text{K}\times 8$，而不是 $2\,\text{K}\times 4$。

8. **理解地址线、数据线和片选逻辑的分工**

   芯片内部寻址只需要足以区分其自身单元数的那几根低位地址线。例如：
   - $2\,\text{K}\times 8$ ROM 有 $2^{11}$ 个单元，所以只需要 $11$ 根地址线，即 $A_{10}\sim A_0$；
   - $1\,\text{K}\times 4$ RAM 只有 $2^{10}$ 个单元，所以只需要 $A_9\sim A_0$。

   CPU 更高位的地址线不是无用的，它们要和访存控制信号一起去形成片选逻辑，其作用是决定“当前到底选中哪一片芯片”。

9. **看懂题图中的关键元件**

   **74138 译码器**是一个 **3 线-8 线译码器**，作用是把 $3$ 位输入译成 $8$ 个输出中的某一个。它的关键引脚包括：
   - $G_1$：高电平有效使能端；
   - $\overline{G}_{2A}$、$\overline{G}_{2B}$：低电平有效使能端；
   - $C、B、A$：变量输入端；
   - $\overline{Y_0}\sim \overline{Y_7}$：输出端，且**低电平有效**。

   它的工作条件是

   $$
   G_1=1,\quad \overline{G}_{2A}=0,\quad \overline{G}_{2B}=0
   $$

   使能后会根据 $CBA$ 的取值，让某一个 $\overline{Y_i}$ 变成 $0$，其余保持 $1$。它特别适合做片选，因为它天然适合完成“选中一片，其余不选”的工作。

   **门电路**的作用不是重复译码器的工作，而是把“译码器已经选中的某一大块地址”继续细分。当题目要求的地址范围只是某个译码块的一部分时，就要借助门电路再和某一位地址线组合。本题中用 $\overline{Y}_5$ 再配合 $A_{10}$，本质上就是在做进一步筛选。

10. **顺便建立对芯片常见引脚的整体印象**

芯片引脚大致可以分成四类：

- 地址引脚：决定芯片内部访问哪个单元；
- 数据引脚：负责与 CPU 数据总线交换数据；
- 片选引脚：决定该芯片当前是否参与工作；
- 读写控制引脚：决定当前是读还是写。

对本题而言：

- RAM 需要接 CPU 的读写控制信号；
- ROM 在这里只承担“读出系统程序”的任务；
- 因而 ROM 的相关控制端只需接成能正常读出的状态。

后面题解里把 $\overline{\text{PD}}/\text{Progr}$ 接地，就是为了让 ROM 工作在题目需要的读出方式下。

这里的 $\overline{\text{PD}}/\text{Progr}$ 可以理解为“**工作方式控制引脚**”。从名字上就能看出，它和两种状态有关：

- `Progr` 对应 **编程/写入** 状态；
- `PD` 常可理解为 **正常读出时需要满足的控制状态**。

由于这个引脚带上划线，说明它是**低电平有效**的控制端。因此，在本题这种“ROM 只负责存放系统程序，运行时只需要读出、不需要现场写入”的场景下，就必须把它接到能保证芯片处于正常读出工作方式的有效电平。题解中把它接地，本质上就是给这个引脚一个稳定的低电平，使芯片不进入与本题无关的其他工作状态，而是稳定工作在题目所需的读出方式下。

可以把这个引脚简单理解成：**它不决定“读哪个单元”，而是决定“这片 ROM 现在按什么模式工作”**。地址线负责选单元，数据线负责传数据，片选信号决定芯片是否参与工作，而像 $\overline{\text{PD}}/\text{Progr}$ 这样的引脚，则是用来保证芯片处于正确的功能模式。

::: warning 易错点

- 地址区间容量一定要用“末地址减首地址再加 $1$”。
- 两片 $1\,\text{K}\times 4$ 共用地址线、分别接高低数据位，是扩位后的 $1\,\text{K}\times 8$，不是 $2\,\text{K}\times 4$。
- 带横线的信号名通常表示**低电平有效**，例如 $\overline{\text{MREQ}}$、$\overline{Y_4}$；在这种题里，$0$ 往往才意味着“动作成立”。
- 不是所有地址线都直接接芯片地址端。低位地址线做片内寻址，高位地址线做片选译码。

:::

到这里，这道题的预备知识其实已经齐了。真正开始画图时，你可以按下面这套顺序推进：先算各区容量，再按“功能 + 容量 + 位宽”选片；然后确定低位地址线如何直接接芯片，高位地址线如何送 74138；最后再看是否需要借助门电路对译码结果继续细分。后面的“四步解题”，本质上就是把这条主线具体落实到本题的数据和器件条件上。

::::

**第一步，先将十六进制地址范围写成二进制地址码，并确定其总容量。**

$$
\begin{aligned}
&A_{15}\ \cdots\ A_{12}
A_{11}\ \cdots\ A_{8}
A_{7}\ \cdots\ A_{4}
A_{3}\ \cdots\ A_{0}
\\[6pt]
&\left.
\begin{array}{l}
0\ 1\ 1\ 0 \qquad 0\ 0\ 0\ 0 \qquad 0\ 0\ 0\ 0 \qquad 0\ 0\ 0\ 0 \\
\cdots \\
0\ 1\ 1\ 0 \qquad 0\ 1\ 1\ 1 \qquad 1\ 1\ 1\ 1 \qquad 1\ 1\ 1\ 1
\end{array}
\right\}
\quad
\begin{array}{l}
\text{系统程序区} \\
2\,\text{K}\times 8\text{ 位}
\end{array}
\\[10pt]
&\left.
\begin{array}{l}
0\ 1\ 1\ 0 \qquad 1\ 0\ 0\ 0 \qquad 0\ 0\ 0\ 0 \qquad 0\ 0\ 0\ 0 \\
\cdots \\
0\ 1\ 1\ 0 \qquad 1\ 0\ 1\ 1 \qquad 1\ 1\ 1\ 1 \qquad 1\ 1\ 1\ 1
\end{array}
\right\}
\quad
\begin{array}{l}
\text{用户程序区} \\
1\,\text{K}\times 8\text{ 位}
\end{array}
\end{aligned}
$$

**第二步**，根据地址范围的容量以及该范围在计算机系统中的作用，选择存储芯片。

根据 $6000\text{H}\sim 67\text{FF}\text{H}$ 为系统程序区的范围，应选择 $1$ 片 $2\,\text{K}\times 8$ 位的 ROM，若选择 $4\,\text{K}\times 8$ 位或 $8\,\text{K}\times 8$ 位的 ROM，都超出了 $2\,\text{K}\times 8$ 位的系统程序区范围。

根据 $6800\text{H}\sim 6\text{BFF}\text{H}$ 为用户程序区的范围，选 $2$ 片 $1\,\text{K}\times 4$ 位的 RAM 芯片正好满足 $1\,\text{K}\times 8$ 位的用户程序区要求。

**第三步，分配 CPU 的地址线。**

将 CPU 的低 $11$ 位地址 $A_{10}\sim A_0$ 与 $2\,\text{K}\times 8$ 位的 ROM 地址线相连；将 CPU 的低 $10$ 位地址 $A_9\sim A_0$ 与 $2$ 片 $1\,\text{K}\times 4$ 位的 RAM 地址线相连。剩下的高位地址与访存控制信号 $\overline{\text{MREQ}}$ 共同产生存储芯片的片选信号。

**第四步，片选信号的形成。**

由图给出的 74138 译码器输入逻辑关系可知，必须保证控制端 $G_1$ 为高电平，$\overline{G}_{2A}$ 与 $\overline{G}_{2B}$ 为低电平，才能使译码器正常工作。根据第一步写出的存储器地址范围得出，$A_{15}$ 始终为低电平，$A_{14}$ 始终为高电平，它们正好分别与译码器的 $\overline{G}_{2A}$（低）和 $G_1$（高）对应。而访存控制信号 $\overline{\text{MREQ}}$（低电平有效）又正好可与 $\overline{G}_{2B}$（低）对应。剩下的 $A_{13},A_{12},A_{11}$ 可分别接到译码器的 $C,B,A$ 输入端。输出 $\overline{Y}_4$ 有效时，选中 $1$ 片 ROM；$\overline{Y}_5$ 与 $A_{10}$ 同时有效为低电平时，与门输出选中 $2$ 片 RAM，如图所示。

::: details 接线时可能存在的问题

1. **<span id="why-use-y5-and-a10-to-select-2-ram-of-8k-8">为什么要同时使用 $\overline{Y}_5$ 与 $A_{10}$ 作为 $2$ 片 RAM 的片选信号？为什么不可以直接使用 $\overline{Y}_5$？</span>**

   关键原因：**$\overline{Y}_5$ 只能选中一个完整的 $2\,\text{K}$ 地址块，而题目给 RAM 的地址范围只有其中前半个 $1\,\text{K}$。**

   在本题中，74138 的输入端分别接的是 $C=A_{13}$、$B=A_{12}$、$A=A_{11}$。因此当

   $$
   A_{13}A_{12}A_{11}=101
   $$

   时，译码器输出 $\overline{Y}_5=0$，这对应的是一个完整的 $2\,\text{K}$ 地址块，也就是

   $$
   6800\text{H}\sim 6\text{FFF}\text{H}
   $$

   但题目真正分配给用户程序区的 RAM 地址范围只有

   $$
   6800\text{H}\sim 6\text{BFF}\text{H}
   $$

   它只是上面这个 $2\,\text{K}$ 大块中的前一半。换句话说，若只用 $\overline{Y}_5$ 来片选，那么 RAM 会在整个

   $$
   6800\text{H}\sim 6\text{FFF}\text{H}
   $$

   范围内都被选中，这比题目要求的范围多出了

   $$
   6\text{C00}\text{H}\sim 6\text{FFF}\text{H}
   $$

   这一段地址。

   为什么偏偏用 $A_{10}$ 来继续细分？因为两片 RAM 组成的是 $1\,\text{K}\times 8$，而 $1\,\text{K}$ 存储器内部只需要 $10$ 根地址线，所以 RAM 的片内地址只接到 $A_9\sim A_0$。这就意味着 $A_{10}$ 不参与 RAM 的片内寻址，只能用来区分“这一大块地址中的前半块还是后半块”。

   在这里：
   - 当 $A_{10}=0$ 时，对应前半块地址 $6800\text{H}\sim 6\text{BFF}\text{H}$；
   - 当 $A_{10}=1$ 时，对应后半块地址 $6\text{C00}\text{H}\sim 6\text{FFF}\text{H}$。

   所以必须让 **$\overline{Y}_5$ 有效且 $A_{10}=0$** 时，RAM 才被选中。这样做的本质，就是先用 74138 选中一个 $2\,\text{K}$ 大块，再用 $A_{10}$ 把这个大块切成前后两个 $1\,\text{K}$ 小块，只取前一半给 RAM。

   如果直接使用 $\overline{Y}_5$，就会产生地址镜像：前 $1\,\text{K}$ 和后 $1\,\text{K}$ 的某些地址会映射到同一片物理 RAM 单元上，因为 RAM 内部根本没有接收 $A_{10}$ 这一位地址信息。

2. **74138 译码器的真值表是什么？$C, B, A$ 分别为 $100$ 和 $101$ 时输出有何变化？为什么选 $\overline{Y}_5$ 和 $\overline{Y}_4$？**

   74138 是一个 **3 线-8 线译码器**。在满足使能条件

   $$
   G_1=1,\quad \overline{G}_{2A}=0,\quad \overline{G}_{2B}=0
   $$

   时，它会根据输入端 $CBA$ 的二进制值，从 $\overline{Y_0}$ 到 $\overline{Y_7}$ 中选中一路输出。要注意：**74138 的输出是低电平有效的**，也就是说，被选中的那一路输出为 $0$，其余输出保持为 $1$。

   对应关系如下：

   | $C$ | $B$ | $A$ | 有效输出           |
   | --- | --- | --- | ------------------ |
   | $0$ | $0$ | $0$ | $\overline{Y_0}=0$ |
   | $0$ | $0$ | $1$ | $\overline{Y_1}=0$ |
   | $0$ | $1$ | $0$ | $\overline{Y_2}=0$ |
   | $0$ | $1$ | $1$ | $\overline{Y_3}=0$ |
   | $1$ | $0$ | $0$ | $\overline{Y_4}=0$ |
   | $1$ | $0$ | $1$ | $\overline{Y_5}=0$ |
   | $1$ | $1$ | $0$ | $\overline{Y_6}=0$ |
   | $1$ | $1$ | $1$ | $\overline{Y_7}=0$ |

   因而：
   - 当 $CBA=100$ 时，对应二进制数 $4$，所以输出变为 $\overline{Y_4}=0$；
   - 当 $CBA=101$ 时，对应二进制数 $5$，所以输出变为 $\overline{Y_5}=0$。

   之所以本题正好用到 $\overline{Y_4}$ 和 $\overline{Y_5}$，不是人为“挑中”了这两个输出，而是由地址范围本身决定的。由于题中把

   $$
   C=A_{13},\quad B=A_{12},\quad A=A_{11}
   $$

   所以 74138 实际译码的就是高位地址 $A_{13}A_{12}A_{11}$。

   对系统程序区 $6000\text{H}\sim 67\text{FF}\text{H}$ 而言，其高位可归纳为

   $$
   A_{15}A_{14}A_{13}A_{12}A_{11}=0\,1\,1\,0\,0
   $$

   因此后三位

   $$
   A_{13}A_{12}A_{11}=100
   $$

   所以自然对应 $\overline{Y_4}$。这一路正好可用来片选 ROM。

   对用户程序区 $6800\text{H}\sim 6\text{BFF}\text{H}$ 而言，其高位可归纳为

   $$
   A_{15}A_{14}A_{13}A_{12}A_{11}=0\,1\,1\,0\,1
   $$

   因此后三位

   $$
   A_{13}A_{12}A_{11}=101
   $$

   所以自然对应 $\overline{Y_5}$。这一路先选出 RAM 所在的那个 $2\,\text{K}$ 地址块，再由 $A_{10}$ 进一步细分成题目真正需要的 $1\,\text{K}$ RAM 区间。

### 例 4.2：假设 CPU 及其他芯片同 [**例 4.1**](#例-4-1)，画出 CPU 与存储器的连接图。要求主存的地址空间满足下述条件：最小 $8\,\text{K}$ 地址为系统程序区，与其相邻的 $16\,\text{K}$ 地址为用户程序区，最大 $4\,\text{K}$ 地址空间为系统程序工作区。详细画出存储芯片的片选逻辑并指出存储芯片的种类及片数。

::: details 知识点讲解、元件解释和解题方法

这道题和例 4.1 的基本解法是一致的，仍然是“先看地址范围，再定容量与芯片，再分配地址线，最后形成片选信号”。因此，例 4.1 中已经讲过的内容，如存储芯片规格怎么看、地址区容量怎么算、地址线和数据线如何分工、74138 的基本作用等，这里不再重复展开。下面只讲这道题相对例 4.1 新出现或更值得注意的地方。

1. **先读懂题目里的几个新表述**

   这题和例 4.1 最大的不同，是**地址分配方式的表达变了**。题目不再直接给出某一段明确的十六进制区间，而是说：
   - **最小 $8\,\text{K}$ 地址为系统程序区**；
   - **与其相邻的 $16\,\text{K}$ 地址为用户程序区**；
   - **最大 $4\,\text{K}$ 地址空间为系统程序工作区**。

   这里的关键词要这样理解：
   - **最小地址空间**：指从整个主存编址空间的最低地址开始分配；
   - **相邻**：指紧挨着前一段地址空间继续往上排；
   - **最大地址空间**：指整个编址空间中最高地址的那一段；
   - **系统程序工作区**：本质上仍然是 RAM 区，只不过它服务于系统程序运行时的临时工作需要。

   因为 CPU 有 $16$ 根地址线，所以整个地址空间是从

   $$
   0000\text{H}\sim \text{FFFF}\text{H}
   $$

   因而本题三段空间实际上就是：
   - 最小 $8\,\text{K}$：$0000\text{H}\sim 1\text{FFF}\text{H}$；
   - 与其相邻的 $16\,\text{K}$：$2000\text{H}\sim 5\text{FFF}\text{H}$；
   - 最大 $4\,\text{K}$：$\text{F}000\text{H}\sim \text{FFFF}\text{H}$。

2. **这次最关键的新变化：用户程序区跨了两个 $8\,\text{K}$ 块**

   例 4.1 中每一类存储区都比较“整齐”，基本可以直接对上一块固定容量的芯片；而这题里，用户程序区要求的是 **相邻 $16\,\text{K}$**，这意味着它不是一个单独的 $8\,\text{K}$ 译码块，而是**连续两个 $8\,\text{K}$ 块拼起来**。

   由于给定芯片中有 $8\,\text{K}\times 8$ 位 RAM，所以最自然的做法就是：
   - 用 $1$ 片 $8\,\text{K}\times 8$ 位 ROM 放系统程序区；
   - 用 $2$ 片 $8\,\text{K}\times 8$ 位 RAM 顺次构成相邻的 $16\,\text{K}$ 用户程序区；
   - 再用 $1$ 片 $4\,\text{K}\times 8$ 位 RAM 构成最高地址处的系统程序工作区。

   这也是为什么本题的片选信号会同时出现 $\overline{Y}_1$ 和 $\overline{Y}_2$：因为用户程序区不是只占一个块，而是占了两个相邻的 $8\,\text{K}$ 译码块。

3. **为什么这次把 $A_{15}A_{14}A_{13}$ 直接送到 74138 的 $CBA$**

   例 4.1 中，由于地址区的高位有很明显的固定模式，所以可以把一部分高位地址接到 74138 的使能端，另一部分接到输入端；而本题的地址分布更像是在整个 $64\,\text{K}$ 空间里，按 $8\,\text{K}$ 为单位去切块：
   - $0000\text{H}\sim 1\text{FFF}\text{H}$ 是第 $0$ 块；
   - $2000\text{H}\sim 3\text{FFF}\text{H}$ 是第 $1$ 块；
   - $4000\text{H}\sim 5\text{FFF}\text{H}$ 是第 $2$ 块；
   - $\cdots$
   - $\text{E}000\text{H}\sim \text{FFFF}\text{H}$ 是第 $7$ 块。

   每一块正好都是 $8\,\text{K}=2^{13}$，因此只要看高 $3$ 位地址

   $$
   A_{15}A_{14}A_{13}
   $$

   就能判断当前访问的是哪一个 $8\,\text{K}$ 块。于是本题最直接的做法，就是把

   $$
   C=A_{15},\quad B=A_{14},\quad A=A_{13}
   $$

   送入 74138，让它按 $8$ 个 $8\,\text{K}$ 块来译码。

   这样一来：
   - $000$ 对应 $\overline{Y}_0$，正好选中最低的 $8\,\text{K}$ ROM；
   - $001$ 对应 $\overline{Y}_1$，选中第一片 $8\,\text{K}$ RAM；
   - $010$ 对应 $\overline{Y}_2$，选中第二片 $8\,\text{K}$ RAM；
   - $111$ 对应 $\overline{Y}_7$，对应最高的那个 $8\,\text{K}$ 地址块。

4. **为什么这次把 $G_1$ 直接接到 $+5\,\text{V}$**

   图中的 $+5\,\text{V}$ 只是表示给 74138 的 $G_1$ 一个固定高电平，也就是逻辑 $1$。这说明本题不再像例 4.1 那样，需要用某个高位地址去控制 $G_1$ 是否使能，而是让 74138 在“只要发生访存”时就处于可工作状态。

   因此本题采取的是：
   - $G_1$ 固定接高电平；
   - $\overline{G}_{2A}$ 与 $\overline{G}_{2B}$ 都接 $\overline{\text{MREQ}}$。

   这样做的含义是：**只要当前确实是访存周期，74138 就进行地址译码；否则不工作。**

5. **为什么最高的 $4\,\text{K}$ RAM 不能直接只用 $\overline{Y}_7$**

   这是本题最容易忽略的新点。$\overline{Y}_7$ 对应的是最高的整个 $8\,\text{K}$ 块，也就是

   $$
   \text{E}000\text{H}\sim \text{FFFF}\text{H}
   $$

   但题目要求的系统程序工作区只有**最大 $4\,\text{K}$**，也就是最高地址中的后一半：

   $$
   \text{F}000\text{H}\sim \text{FFFF}\text{H}
   $$

   因而和例 4.1 中“$\overline{Y}_5$ 还要再结合 $A_{10}$”的道理类似，本题里也必须把 $\overline{Y}_7$ 对应的这个 $8\,\text{K}$ 大块再切成两半。

   对一个 $4\,\text{K}\times 8$ 位 RAM 来说，片内寻址只需要 $12$ 根地址线，即 $A_{11}\sim A_0$；因此 $A_{12}$ 不参与 RAM 内部寻址，而是恰好可以用来区分这整个 $8\,\text{K}$ 块中的前半和后半：
   - 当 $A_{12}=0$ 时，对应 $\text{E}000\text{H}\sim \text{EFFF}\text{H}$；
   - 当 $A_{12}=1$ 时，对应 $\text{F}000\text{H}\sim \text{FFFF}\text{H}$。

   题目要求取的是最高的那一半，所以必须在 $\overline{Y}_7$ 的基础上，再结合 $A_{12}=1$ 这一条件。图中“先反相，再与 $\overline{Y}_7$ 相与”的逻辑，本质上就是为了构造这个更精确的片选信号。

6. **这题和例 4.1 的解法差异可以这样抓**

   如果要把两题的区别压缩成最值得记的几句话，可以这样理解：
   - 例 4.1 是在一个较小地址区间内做较细的局部划分，因此用到了“高位地址接使能端、局部地址再细分”的思路。
   - 例 4.2 是先把整个 $64\,\text{K}$ 空间按 $8\,\text{K}$ 块整体切开，再从这些块中挑出连续的几块分配给 ROM 和 RAM。
   - 例 4.2 中最值得注意的新点，不是 ROM 和 RAM 本身，而是：
     - 用户区跨了两个 $8\,\text{K}$ 块；
     - 最高的 $4\,\text{K}$ 区只占 $\overline{Y}_7$ 对应块的一半，因此还要继续利用 $A_{12}$ 细分。

读到这里，再看后面的四步解题，重点就不再是“74138 怎么用”，而是“为什么这次要按 $8\,\text{K}$ 为单位整体译码，以及为什么最高的 $4\,\text{K}$ 还要再切半”。

**为什么系统程序工作区的片选信号来自 $\overline{Y}_7$ 和 $A_{12}=1$ 这两个条件？**

这个问题和前面 [**为什么要同时使用 $\overline{Y}_5$ 与 $A_{10}$ 作为 $2$ 片 RAM 的片选信号**](#why-use-y5-and-a10-to-select-2-ram-of-8k-8) 是同一类问题：**74138 先选中一个较大的地址块，然后再借助某一位更低的地址线，把这个大块继续细分成题目真正需要的小块。**

这里之所以要同时看 $\overline{Y}_7$ 和 $A_{12}=1$，是因为 **$\overline{Y}_7$ 只能选中最高的整个 $8\,\text{K}$ 地址块，而系统程序工作区实际只需要这个大块中的后一半 $4\,\text{K}$。**

在本题中，74138 的输入端接的是

$$
C=A_{15},\quad B=A_{14},\quad A=A_{13}
$$

因此当

$$
A_{15}A_{14}A_{13}=111
$$

时，译码器输出 $\overline{Y}_7=0$，对应的是最高的整个 $8\,\text{K}$ 地址块：

$$
\text{E}000\text{H}\sim \text{FFFF}\text{H}
$$

但题目真正要求的系统程序工作区只有**最大 $4\,\text{K}$**，也就是这整个 $8\,\text{K}$ 地址块中的后一半：

$$
\text{F}000\text{H}\sim \text{FFFF}\text{H}
$$

因此，如果只用 $\overline{Y}_7$ 作为片选信号，那么这片 $4\,\text{K}\times 8$ 位 RAM 会在整个

$$
\text{E}000\text{H}\sim \text{FFFF}\text{H}
$$

范围内都被选中，这显然超过了题目要求。多出来的那一段就是

$$
\text{E}000\text{H}\sim \text{EFFF}\text{H}
$$

为什么这里继续细分时要用的是 $A_{12}$？因为这片 RAM 的容量是 $4\,\text{K}\times 8$，而

$$
4\,\text{K}=2^{12}
$$

所以它的片内地址只需要 $12$ 根地址线，即 $A_{11}\sim A_0$。这意味着 $A_{12}$ **不参与这片 RAM 的片内寻址**，因此它正好可以用来区分这个 $8\,\text{K}$ 大块中的前半和后半：

- 当 $A_{12}=0$ 时，对应前半块 $\text{E}000\text{H}\sim \text{EFFF}\text{H}$；
- 当 $A_{12}=1$ 时，对应后半块 $\text{F}000\text{H}\sim \text{FFFF}\text{H}$。

而题目要的是“最大 $4\,\text{K}$ 地址空间”，也就是最高地址处的那一半，所以必须让 **$\overline{Y}_7$ 有效且 $A_{12}=1$** 时，这片 RAM 才被选中。

如果不加上 $A_{12}=1$ 这个条件，而是直接使用 $\overline{Y}_7$，就会出现和前面例子相同的现象：由于 RAM 内部只接收 $A_{11}\sim A_0$，没有接收 $A_{12}$，因此前半块和后半块中某些地址会映射到同一个物理存储单元上，形成地址镜像。

### 例 4.6：设有 4 个模块组成的四体存储器结构，每个体的存储字长为 32 位，存取周期为 200 ns。假设数据总线宽度为 32 位，总线传输周期为 50 ns，试求顺序存储和交叉存储的存储器带宽。

顺序存储（高位交叉编址）和交叉存储（低位交叉编址）连续读出 4 个字的**信息量**是 $32\times 4 = 128$ 位。

**顺序存储存储器**连续读出 4 个字的**时间**是

$$200\,\text{ns} \times 4 = 800\,\text{ns} = 8 \times 10^{-7}\,\text{s}$$

**交叉存储存储器**连续读出4 个字的**时间**是

$$200\,\text{ns} + 50\,\text{ns} \times (4-1) = 350\,\text{ns} = 3.5 \times 10^{-7}\,\text{s}$$

**顺序存储器**的**带宽**是

$$128/(8\times 10^{-7}) = 16\times 10^7\,\text{bps}$$

**交叉存储器**的**带宽**是

$$128/(3.5\,\times 10^{-7}) = 37\times 10^7\,\text{bps}$$

::: info 顺序存储和交叉存储

顺序存储和交叉存储，区别在于**连续地址如何分配到多个存储体**。

**顺序存储**又常称为**高位交叉编址**。它的特点是：一大段连续地址先集中放在同一个存储体中，只有高位地址变化时，才会访问下一个存储体。这样做的结果是，若 CPU 连续读取若干个相邻字，这些字往往仍落在同一个存储体里，只能等前一次存取结束后，再进行下一次存取，因此访问是“一个接一个”地串行完成，带宽较低。

**交叉存储**又常称为**低位交叉编址**。它的特点是：把连续地址轮流分配到不同的存储体中。例如 4 个存储体时，连续地址可按“第 0 体、第 1 体、第 2 体、第 3 体、再回到第 0 体”的方式循环分布。这样一来，当 CPU 连续读取若干个相邻字时，相邻字通常位于不同存储体中，于是一个存储体还在恢复或完成本次存取时，下一个存储体已经可以开始工作，从而把多个存储体的工作时间交叠起来。

因此，两者的核心差别可以概括为：

- **顺序存储**：连续字常落在同一存储体中，连续访问时难以并行，速度较慢；
- **交叉存储**：连续字分散在不同存储体中，连续访问时可交叉进行，带宽更高。

这也是本题中为什么交叉存储连续读出 $4$ 个字所需时间明显短于顺序存储：前者利用了多个存储体“轮流接力”工作的特点，而后者基本只能等待同一存储体一次一次地完成读出。

:::

### 例 4.7：假设 CPU 执行某段程序时，共访问 Cache 命中 2000 次，访问主存 50 次。已知 Cache 的存取周期为 50 ns，主存的存取周期为 200 ns。求 Cache-主存系统的命中率、效率和平均访问时间。

1. **Cache 的命中率**为

   $$\dfrac{2000}{2000+50} \approx 0.976$$

2. 由题可知，**访问主存的时间**是访问 Cache 时间的 **4 倍**（$200/50 = 4$）。

   设访问 Cache 的时间为 $t$，访问主存的时间为 $4t$，**Cache-主存系统的访问效率**为 $e$，则

   $$\begin{aligned} e & =\dfrac{\text{访问 Cache 的时间}}{\text{平均访问时间}}\times 100\%\\ & = \frac{t}{\frac{2000}{2050}t + \left(1-\frac{2000}{2050}\right)4t}\times 100\%\\ & \approx 93.2\% \end{aligned}$$

3. **平均访问时间**为

   $$50\,\text{ns}\times \dfrac{2000}{2050} + 200\,\text{ns}\times \left(1-\dfrac{2000}{2050}\right) \approx 53.7\,\text{ns}$$

### 例 4.8：假设主存容量为 512 KB，Cache 容量为 4 KB ，每个字块为 16 个字，每个字 32 位。<br>1. Cache 地址有多少位？可容纳多少块？<br>2. 主存地址有多少位？可容纳多少块？<br>3. 在直接映射方式下，主存的第几块映射到 Cache 中的第 5 块（设起始字块为第 1 块）？<br>4. 画出直接映射方式下主存地址字段中各段的位数。

::: details CPU、Cache 和主存

在做这类题之前，最好先把这三个名词之间的关系理顺：

- **CPU**：中央处理器，是发出访存请求的一方。程序执行过程中，CPU 需要不断读取指令和数据，也可能把结果写回存储器。
- **主存**：也就是内存，容量较大，但访问速度比 CPU 慢得多。程序和数据最终都要放在主存中。
- **Cache**：位于 CPU 和主存之间的一层高速小容量存储器。它的容量远小于主存，但速度更快，作用是把 CPU 最近可能会反复访问的数据暂时保存起来，减少 CPU 直接访问主存的次数。

它们三者的关系可以简单理解成：**主存负责“存得下”，Cache 负责“取得快”，CPU 负责“来使用”。**

CPU 访问数据时，一般并不是每次都直接找主存，而是先查 Cache：

1. CPU 给出一个主存地址；
2. 先看这个地址所在的数据块是否已经在 Cache 中；
3. 如果在，就称为**命中**，CPU 直接从 Cache 读，速度很快；
4. 如果不在，就称为**失效**，此时需要到主存中把该地址所在的整个字块调入 Cache，再由 CPU 访问。

注意这里调入 Cache 的通常不是“一个字”，而是**一个字块**。这是因为程序访问常具有局部性：刚访问过的数据及其附近的数据，很可能很快再次被访问。因此，把整块一起调入 Cache，往往能提高后续命中率。

也正因为 Cache 容量远小于主存，所以主存中的很多块不可能同时全部放进 Cache，这就引出了后面要讨论的映射方式问题：**主存的一个块装入 Cache 时，到底允许放到 Cache 的哪些位置上。** 后面的直接映射、全相联映射和组相联映射，研究的正是这个问题。

:::

1. 根据 Cache 容量为 $4\,\text{KB}$（$2^{12} = 4\,\text{K}$），**Cache 地址为 12 位**。由于每字 32 位，则 Cache 共有 $4\,\text{KB}/4\,\text{B} = 1\,\text{K}$ 字。因每个字块 16 个字，故 **Cache 中有 $1\,\text{K}/16 = 64$ 块**。
2. 根据主存容量为 $512\,\text{KB}$ （$2^{19} = 512\,\text{K}$），**主存地址为 19 位**。由于每字 32 位，则主存共有 $512\,\text{KB}/4\,\text{B} = 128\,\text{K}$ 字。因每个字块 16 个字，故主存中共 $128\,\text{K}/16 = 8\,192$ 块。
3. 在直接映射方式下，由于 Cache 共有 $64$ 块，主存共有 $8\,192$ 块，因此**能映射到 Cache 第 $5$ 块的主存块号可写成 $64k+5\ (k=0,1,2,\dots,127)$**，即主存的 $5$，$64+5$，$2\times 64+5$，$\dots$，$8192 - 64 + 5$ 块。

   ::: details 如何计算？

   在直接映射方式下，主存第 $j$ 块映射到 Cache 的第几块，只需看

   $$j \bmod (\text{Cache 块数})$$

   的结果。  
   本题中 Cache 有 $64$ 块，所以凡是满足

   $$j \bmod 64 = 5$$

   的主存块，都会映射到 Cache 的第 $5$ 块。因而可写成

   $$j = 64k+5 \qquad (k=0,1,2,\dots)$$

   再结合主存一共 $8\,192$ 块，就得到答案中的那一串结果。

   :::

4. 在直接映射方式下，主存地址字段的各段位数分配如图所示。其中**字块内地址为 $6$ 位**（$4$ 位表示 $16$ 个字，$2$ 位表示每字 $32$ 位），缓存共 $64$ 块，故**缓存字块地址为 $6$ 位**，**主存字块**标记为主存地址长度与 Cache 地址长度之差，**即 $19-12=7$ 位**。

   **主存地址各字段的分配：**

   ![主存地址各字段的分配](https://static.owo.cab/notes/image/cs/poco/chapter04/主存地址各字段的分配.drawio.svg)

   ::: details 如何分配各字段？

   直接映射方式下，主存地址通常按“**标记 + Cache 块号 + 块内地址**”来划分。计算时可以按下面三步来做：
   - **先确定块内地址位数**：本题每块有 $16$ 个字，每字 $32$ 位，即每字 $4\,\text{B}$，所以块大小是 $16\times 4=64\,\text{B}$，对应

     $$64=2^6$$

     故块内地址为 $6$ 位。

   - **再确定 Cache 块号位数**：Cache 共 $64$ 块，而

     $$64=2^6$$

     所以 Cache 块号也需要 $6$ 位。

   - **最后求标记位数**：主存地址共 $19$ 位，去掉块内地址和 Cache 块号后，剩余部分就是标记位，因此

     $$19-6-6=7$$

     故主存字块标记为 $7$ 位。

   所以本题直接映射方式下的主存地址字段分配就是：**标记 $7$ 位，Cache 块号 $6$ 位，块内地址 $6$ 位。**

   :::

::: details 直接映射方式、全相联映射方式和二路组相联映射方式

这三种映射方式，本质上都在回答同一个问题：**主存中的一个字块，装入 Cache 时，究竟允许放到 Cache 的哪些位置上。**

1. **直接映射方式**

   在直接映射中，主存的每一个字块只能映射到 Cache 中**唯一固定**的一块。也就是说，主存块号一旦确定，对应的 Cache 块号也就确定了，不能随意改放别处。

   如果设 Cache 共有 $2^c$ 块，那么主存第 $j$ 块只能映射到 Cache 的第

   $$j\bmod 2^c$$

   块。

   因此，直接映射的主存地址通常分成三段：
   - **主存字块标记**：用来判断当前 Cache 这一块里装的到底是不是所需的主存块；
   - **Cache 块号**：直接指出应到 Cache 的哪一块去查找；
   - **块内地址**：指出在该块内部访问哪个字或哪个字节。

   它的优点是结构最简单，查找速度快，硬件实现也容易；缺点是冲突最明显。凡是映射到同一 Cache 块的多个主存块，只要轮流访问，就会频繁把彼此顶掉，产生较高的冲突失效。

2. **全相联映射方式**

   在全相联映射中，主存的任意一个字块都可以放到 Cache 的**任意一块**中，没有固定位置限制。也就是说，装入时只要 Cache 中有空块，或者替换算法选中了某块，就可以把这个主存块放进去。

   由于没有“固定块号”这一说，因此全相联映射的主存地址通常只分成两段：
   - **主存字块标记**；
   - **块内地址**。

   查找时，需要把主存块标记与 Cache 中所有块的标记同时比较，判断它是否已经在 Cache 中。

   它的优点是位置最灵活，最不容易因为“固定映射冲突”而发生失效；缺点是硬件代价最高，因为需要并行比较多个标记，控制逻辑和实现成本都比较大。

3. **二路组相联映射方式**

   二路组相联映射可以看成是前两者之间的折中。它先把 Cache 分成若干组，每组有 $2$ 块。主存中的一个字块先按某种规则映射到**唯一的一组**，但到了该组之后，可以放在组内的任意一块中。

   因此，它既不像直接映射那样“只能放一块”，也不像全相联那样“可以放到任意块”，而是“**只能放到某一组内，但组内有两个候选位置**”。

   如果 Cache 共 $2^c$ 块，采用二路组相联，那么组数就是

   $$\frac{2^c}{2}=2^q$$

   其中 $q=c-1$。这时主存块号先按“模组数”的方式确定应落到哪一组，再在该组的两块中任选一块存放。

   所以，二路组相联方式下，主存地址通常分成三段：
   - **主存字块标记**；
   - **组号**；
   - **块内地址**。

   查找时，不是只查一块，也不是查所有块，而是只在目标组中的 $2$ 块里比较标记。

   它的优点是：相对于直接映射，冲突明显减轻；相对于全相联，硬件复杂度又低得多。因此它是一种非常常见的折中方案。

4. **怎样理解这三者的关系**

   可以把这三种方式理解成“位置限制从严到松”的三个层次：
   - **直接映射**：主存块只能放到 **1 个固定位置**；
   - **二路组相联**：主存块只能先落到 **1 个固定组**，但在组内可放到 **2 个位置**；
   - **全相联映射**：主存块可以放到 **任意位置**。

   因而一般有这样一个趋势：
   - **直接映射**：实现最简单，但冲突失效较多；
   - **全相联映射**：冲突最少，但硬件最复杂；
   - **组相联映射**：在冲突率和实现复杂度之间取折中。

5. **做题时最该抓住什么**

   这三种方式在计算题里最常考的，不是文字定义本身，而是**主存地址字段如何划分**。判断方法可以记成：
   - **直接映射**：地址 = 标记 + Cache 块号 + 块内地址；
   - **全相联映射**：地址 = 标记 + 块内地址；
   - **二路组相联映射**：地址 = 标记 + 组号 + 块内地址。

   所以后面遇到设计主存地址格式的题时，关键就是先算出：
   - 块内地址多少位；
   - Cache 一共有多少块；
   - 若是组相联，则一共有多少组；
   - 最后由“总地址位数减去已知部分”得到标记位数。

6. **还有哪些映射方式需要掌握**

   对《计算机组成原理》这类课程和题目来说，最核心、最常考的就是这三类：
   - **直接映射方式**；
   - **全相联映射方式**；
   - **组相联映射方式**。

   其中，“二路组相联”只是组相联的一种特例。更一般地，还应知道：
   - **四路组相联**；
   - **八路组相联**；
   - 更一般的 **$n$ 路组相联**。

   它们的分析方法和二路组相联完全一样，只是“一组内有多少块”这个条件变了。

   因此，如果已经理解了二路组相联，那么再看四路、八路，本质上只是把“每组 $2$ 块”改成“每组 $4$ 块”或“每组 $8$ 块”。

   所以从复习角度说：
   - **必须牢固掌握**：直接映射、全相联映射、组相联映射；
   - **组相联中要会推广**：二路、四路、八路以及一般的 $n$ 路组相联。

:::

## 练习题部分

### 4.2：计算机中哪些部件可用于存储信息，按其速度、容量和价格/位排序说明。

计算机中**寄存器**、**Cache**、**主存**、**硬盘**可以用于存储信息。一般来说，速度越高，价格就越高；容量越大，价格就越高，而且容量越大，速度必越低。人们追求大容量、高速度、低位价的存储器，可惜这是很难达到的。寄存器通常都制作在 CPU 芯片内，寄存器中的数直接在 CPU 内部参与运算，CPU 内可有十几个、几十个寄存器，它们的速度最快，价位最高，容量最小。主存用来存放将要参与运行的程序和数据，其速度与 CPU 速度差距较大，为了使它们之间速度更好地匹配，在主存与 CPU 之间插入了一种比主存速度更快、容量更小的高速缓冲存储器 Cache，显然其价位要高于主存。所以，按其速度、容量和价格/位排序如下：

- 按**速度由高至低**排序为：寄存器 > Cache > 主存 > 硬盘；
- 按**容量由小至大**排序为：寄存器 < Cache < 主存 < 硬盘；
- 按**价格/位由高至低**排序为：寄存器 > Cache > 主存 > 硬盘。

### 4.3：存储器的层次结构主要体现在什么地方，为什么要分这些层次，计算机如何管理这些层次？

存储器的层次结构主要体现在 **Cache-主存** 和 **主存-辅存** 这两个存储层次上。

- **Cache-主存层次**在存储系统中主要**对 CPU 访存起加速作用**，即从整体运行的效果分析，CPU 访存速度加快，接近于 Cache 的速度，而寻址空间和位价却接近于主存。
- **主存-辅存层次**在存储系统中主要**起扩容作用**，即从程序员的角度看，他所使用的存储器其容量和位价接近于辅存，而速度接近于主存。

综合上述两个存储层次的作用，从整个存储系统来看，就**达到了速度快、容量大、位价低的优化效果**。

**主存与 Cache** 之间的信息调度功能**全部由硬件自动完成**。而**主存与辅存**层次的调度目前**广泛采用虚拟存储技术实现**，即将主存与辅存的一部分通过软硬结合的技术组成虚拟存储器，程序员可使用这个比主存实际空间（物理地址空间）大得多的虚拟地址空间（逻辑地址空间）编程，当程序运行时，再由软、硬件自动配合完成虚拟地址空间与主存实际物理空间之间的转换。因此，这两个层次上的调度或转换操作对于程序员来说都是透明的。

### 4.4：说明存取周期和存取时间的区别。

**主要区别**：

- **存取时间仅为完成一次操作的时间**；
- **存取周期不仅包含操作时间，还包含操作后线路的恢复时间**。

即：

$$\text{存取周期} = \text{存取时间} + \text{恢复时间}$$

### 4.5：什么是存储器的带宽？若存储器的数据总线宽度为 $32$ 位，存取周期为 $200\,\text{ns}$，则存储器的带宽是多少？

- **存储器的带宽**指**单位时间内从存储器进出信息的最大数量**。
- 存储器带宽 $=1/200\,\text{ns}\times 32$ 位 $=160\,\text{M}$ 位/秒 $=20\,\text{MB}$/秒 $=5\,\text{M}$ 字/秒

  ::: danger 注意
  字长 $32$ 位，不是 $16$ 位。（注：$1\,\text{ns}$ = $10^{-9}\,\text{s}$）
  :::

### 4.6：机字长为 $32$ 位，存储容量是 $64\,\text{KB}$，按字编址其寻址范围是多少？若主存以字节编址，试画出主存字地址和字节地址的分配情况。

存储容量是 $64\,\text{KB}$ 时，**按字节编址的寻址范围就是 $64\,\text{K}$**，

如**按字编址**，其**寻址范围为：$64\,\text{KB}/32$ 位 $=(64\times 1024\times 8)/32=16\times 1024=16\,\text{K}$**

### 4.8：试比较静态 RAM 和动态 RAM。

- **静态 RAM**（SRAM）用**双稳态触发器**存储信息；**动态 RAM**（DRAM）用 **MOS 电路中的栅极电容**存储信息。
- 由于电容上的电荷会泄漏，**DRAM 需要定期刷新**；而 **SRAM 在不断电时不需要刷新**。
- **DRAM 集成度高、功耗低、成本低**，适于作**大容量主存**；**SRAM 速度快**，适于作**高速缓冲存储器（Cache）**。

::: info 两者的主要区别如下：

| 特性         | SRAM             | DRAM       |
| ------------ | ---------------- | ---------- |
| 存储信息     | 触发器           | 电容       |
| 破坏性读出   | 否               | 是         |
| 是否需要刷新 | 不需要           | 需要       |
| 行列地址     | 同时送           | 分两次送   |
| 运行速度     | 快               | 慢         |
| 集成度       | 低               | 高         |
| 发热量       | 大               | 小         |
| 存储成本     | 高               | 低         |
| 功耗         | 高               | 低         |
| 可靠性       | 高               | 低         |
| 使用性       | 使用方便         | 不方便     |
| 适用场合     | 高速小容量存储器 | 大容量主存 |

:::

### 4.9：什么叫刷新？为什么要刷新？说明刷新有几种方法。

- **刷新**：指对 **DRAM** 定期进行的**全部重写过程**。
- **刷新原因**：由于 DRAM 依靠电容存储信息，而电容上的电荷会逐渐泄漏，若不及时补充，原存信息就会衰减甚至丢失，因此必须定期刷新。

常用的刷新方法有**三种**：

- **集中式刷新**：在最大刷新间隔时间内，集中安排一段时间对全部存储单元逐行刷新，存在 **CPU 访存死时间**。
- **分散式刷新**：在每个读/写周期之后插入一个刷新周期，因此**无 CPU 访存死时间**，但会降低系统的有效工作速度。
- **异步式刷新**：是**集中式**和**分散式**的折中方案，在不影响正常访问过多的前提下完成刷新。

### 4.11：一个 $8\text{K}\times 8$ 位的动态 RAM 芯片，其内部结构排列成 $256\times 256$ 形式，读/写周期为 $0.1\,\mu\text{s}$。试问采用集中刷新、分散刷新及异步刷新三种方式的刷新间隔各为多少？

芯片内部按 $256\times 256$ 排列，因此共有 $256$ 行，需要在最大刷新周期内把这 $256$ 行全部刷新一遍。设最大刷新周期为 $2\,\text{ms}$。

刷新 $1$ 行需要 $0.1\,\mu\text{s}$，因此连续刷新全部 $256$ 行所需时间为

$$256\times 0.1\,\mu\text{s} = 25.6\,\mu\text{s}$$

- **集中刷新**：在最大刷新周期内，先正常读/写一段时间，等接近 $2\,\text{ms}$ 时，再把 $256$ 行集中连续刷新完。因此，若把题目中的“刷新间隔”理解为“两次集中刷新批次之间的间隔”，则其刷新间隔为

  $$2\,\text{ms}$$

  这种方式的缺点是会出现一段不能正常访存的**刷新死时间**，其长度正是上面算出的

  $$25.6\,\mu\text{s}$$

- **分散刷新**：每个读/写周期后都插入一个刷新周期。由于一个读/写周期和一个刷新周期都为 $0.1\,\mu\text{s}$，因此相邻两次刷新操作之间的时间间隔为

  $$0.1\,\mu\text{s}+0.1\,\mu\text{s}=0.2\,\mu\text{s}$$

  若继续问“完成一轮全部刷新需要多久”，则为

  $$256\times 0.2\,\mu\text{s}=51.2\,\mu\text{s}$$

- **异步刷新**：在最大刷新周期 $2\,\text{ms}$ 内，把 $256$ 次行刷新均匀地分散开来。于是每刷新一行的时间间隔为

  $$\frac{2\,\text{ms}}{256}=7.8125\,\mu\text{s}$$

因此，这道题若按通常口径回答“刷新间隔”，则三种方式分别为：

- 集中刷新：$2\,\text{ms}$
- 分散刷新：$0.2\,\mu\text{s}$
- 异步刷新：$7.8125\,\mu\text{s}$

### 4.13：设有一个 $64\text{K}\times 8$ 位的 RAM 芯片，试问该芯片共有多少个基本单元电路（简称存储基元）？欲设计一种具有上述同样多存储基元的芯片，要求对芯片字长的选择应满足地址线和数据线的总和为最小，试确定这种芯片的地址线和数据线，并说明有几种解答。

**存储基元总数 $=64\,\text{K}\times 8$ 位 $=512\,\text{K}$ 位 $=2^{19}$ 位**

如要满足地址线和数据线总和最小，应尽量把存储元安排在字内，因为地址位数和字数成 2 的幂的关系，可较好地压缩级数。

设地址线根数为 $a$，数据线根数为 $b$，

则片容量为：$2^a\times b=2^{19}$；$b=2^{(19-a)}$

- 若 $a=19,b=1$，总和 $=19+1=20$；
- 若 $a=18,b=2$，总和 $=18+2=20$；
- 若 $a=17,b=4$，总和 $=17+4=21$；
- 若 $a=16,b=8$，总和 $=16+8=24$；

可见，地址线与数据线总和的最小值为 $20$，共有 **$2$ 种解答**：

- $a=19,\ b=1$，即设计成 $2^{19}\times 1$ 位芯片；
- $a=18,\ b=2$，即设计成 $2^{18}\times 2$ 位芯片。

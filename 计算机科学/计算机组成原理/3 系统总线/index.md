# 第 3 章 系统总线

## 例题部分

### 例 3.1：假设总线的时钟频率为 100 MHz，总线的传输周期为 4 个时钟周期，总线的宽度为 32 位，试求总线的数据传输率。若想提高一倍数据传输率，可采取什么措施？

1. 根据总线时钟频率为 $100\,\text{MHz}$，得

   $1$ 个时钟周期为 $1/100\,\text{MHz} = 0.01\,\mu\text{s}$

   总线传输周期为 $0.01\,\mu\text{s} \times 4 = 0.04\,\mu\text{s}$

   由于总线的宽度为 $32$ 位 ＝ $4\,\text{B}$（字节）

   故总线的数据传输率为 $4\,\text{B} / 0.04\,\mu\text{s} = 100\,\text{MBps}$

2. 若想提高一倍数据传输率，可以在不改变总线时钟频率的前提下，将数据线的宽度改为 $64$ 位，也可以仍保持数据宽度为 $32$ 位，但使总线的时钟频率增加到 $200\,\text{MHz}$ 。

### 例 3.2：在异步串行传输系统中，假设每秒传输 120 个数据帧，其字符格式规定包含 1 个起始位、7 个数据位、1 个奇校验位、1 个终止位，试计算波特率。

根据题目给出的字符格式，一帧包含 $1+7+1+1=10$ 位

- 两种传输率的异步串行传送字符格式：

  ![两种传输率的异步串行传送字符格式](https://static.owo.cab/notes/image/cs/poco/chapter03/两种传输率的异步串行传送字符格式.webp)

故波特率为 $(1+ 7 + 1 + 1) \times 120 = 1\,200\,\text{bps} = 1\,200$ 波特

::: info 比特率和波特率

- **波特率**：单位时间内传送的**码元个数**，单位为**波特**（Baud）。
- **比特率**：单位时间内传送的**二进制信息位个数**，单位通常为 $\text{bps}$。

当**一个码元只表示 1 位二进制信息**时，比特率与波特率**数值相等**；若**一个码元可表示多位二进制信息**，则比特率会**大于**波特率。

在**异步串行通信**中，若把起始位、校验位、终止位都算在内，求得的通常是**波特率**；若只统计有效数据位，则得到的是**比特率**。

:::

### 例 3.3：画图说明用异步串行传输方式发送十六进制数据 95H 。要求字符格式为：1 位起始位、8 位数据位、1 位偶校验位、1 位终止位。

- 数据95H的传送波形：

  ![数据95H的传送波形](https://static.owo.cab/notes/image/cs/poco/chapter03/数据95H的传送波形.webp)

由于异步串行通信字符格式中包含若干附加位，如起始位、终止位、校验位，而且终止位又有 1 位、1.5 位、2 位之分，若只考虑有效数据位，可用比特率来衡量异步串行通信的数据传输速率，即单位时间内传送二进制有效数据的位数，单位用 $\text{bps}$ 表示。

为了提高速度，将异步串行传送中这些附加位去掉，就可以采用同步传送，在同步传送时，数据块开始处要用同步字符 `SYN` 来指明，如图所示。

- 同步串行传送格式：

  ![同步串行传送格式](https://static.owo.cab/notes/image/cs/poco/chapter03/同步串行传送格式.webp)

  ::: tip

  图中有两个 `SYN`，是因为同步传送开始时通常要连续发送若干同步字符，使接收端可靠地建立同步，然后再连续接收后续数据。

  :::

同步串行传送速度高于异步串行传送速度，可达 $500$ 千波特，而异步通信传送一般为 $50\sim19\,200$ 波特。

::: info 同步传送和异步传送的区别

- **异步传送**：通信双方**没有统一的公共时钟**来严格规定每一位数据的发送时刻，因此每发送一个字符，通常都要加上**起始位**和**终止位**，必要时还会加**校验位**，接收方据此判断一帧数据从哪里开始、在哪里结束。
- **同步传送**：通信双方按**统一的时序**收发数据，发送时通常是**一组数据连续传送**，只需在数据块开始处用同步字符（如 `SYN`）或其他同步方式表明传输开始，不必像异步传送那样给**每个字符**都加起始位和终止位。

因此，**异步传送**的优点是**实现简单、适合间歇式低速通信**，缺点是附加位较多、传输效率较低；**同步传送**的优点是**附加开销小、传输效率高、适合连续高速传输**，但对时序配合的要求更高，控制也更复杂。

:::

## 练习题部分

### 3.1：什么是总线？总线传输有何特点？为了减轻总线的负载，总线上的部件都应具备什么特点？

1. **什么是总线**：总线是连接多个部件的信息传输线，是个部件共享的传输介质。
2. **总线传输特点**：在某一时刻，只允许有一个部件向总线发送信息，而多个部件可以同时从总线上接受相同的信息。
3. **为减轻总线上的负载**：各种 I/O 设备要通过 I/O 接口接在总线上，而且还要通过三态门挂在总线上，没有数据交换时置成高阻态。

   :::: info 什么是三态门
   - **三态门**是在普通 `0`、`1` 两种输出状态之外，还能输出第三种状态“**高阻态**”的门电路。
   - 当三态门输出为**高阻态**时，可把它看成该设备的输出端暂时与总线**断开**，此时它**不能向总线写入数据**，从而避免多个设备同时向总线输出而发生冲突。

   因此，在总线系统中，只有当前获得总线使用权的设备，才应打开三态门向总线送数据；其他设备在不发送数据时应置为高阻态。

   ::: details 为什么三态门会主动向总线输出高（`1`） / 低（`0`）电平？

   这里容易误解的一点是：**三态门并不是脱离 I/O 设备单独决定输出什么**，而是位于设备输出端和总线之间，起“**允许接入或断开总线**”的作用。可以把它理解成一扇受控的门。
   - 当设备需要向总线发送数据时，设备内部逻辑本来就已经产生了某一位数据 `0` 或 `1`，三态门在**使能**状态下，只是把这个 `0` 或 `1` **传到总线**上，并对总线提供足够的驱动能力，所以看起来像是“三态门在输出 `0/1`”。
   - 因此，三态门输出 `1`，并不是“不管 I/O 设备本身输出什么都强行压成 `1`”；恰恰相反，**它输出的高低电平正是设备当前要送出的那一位数据**。设备输出是 `1`，它就把 `1` 送上总线；设备输出是 `0`，它就把 `0` 送上总线。
   - 之所以要强调“主动输出高/低电平”，是因为总线是一条有电容、有负载的共享线路，发送方不能只“声明”自己是 `0` 或 `1`，而必须真正把线路驱动到相应电平，接收方才能稳定读到数据。
   - 当设备**未被选中**或**不该发送数据**时，三态门进入**高阻态**，此时不是输出 `0`，也不是输出 `1`，而是相当于从总线上断开，让其他设备去驱动总线。

   :::

   ::::

### 3.2：总线如何分类？什么是系统总线？系统总线又分为几类，它们各有何作用，是单向的，还是双向的，它们与机器字长、存储字长、存储单元有何关系？

1. 按照**连接部件**的不同，总线可以分为**片内总线**、**系统总线**和**通信总线**。

   ::: details 片内总线、系统总线和通信总线
   - **片内总线**：指**芯片内部**各功能部件之间的连线，用来连接运算器、控制器、寄存器等内部单元。
   - **系统总线**：指**计算机主机内部**连接 CPU、主存和 I/O 接口等部件的信息传输线，是本章讨论的重点。
   - **通信总线**：指**计算机系统之间**或计算机与其他外部设备之间进行信息传送的总线，用于更大范围的数据通信。

   :::

2. **系统总线**：是连接 CPU、主存、 I/O 各部件之间的信息传输线。
3. 系统总线按照**传输信息**不同又分为**地址线**、**数据线**和**控制线**。
   - **地址线**是**单向的**，其根数越**多**，寻址空间越**大**，即 CPU 能访问的存储单元的个数越**多**；
   - **数据线**是**双向的**，其根数与存储字长**相同**，是机器字长的整数倍。
   - **控制线**是**双向的**。

   ::: details 为什么它们的传输方向不同？

   传输方向不同，根本原因在于它们承担的任务不同：
   - **地址线**用来指出“要访问谁”。在一般的主存和 I/O 访问中，地址信息主要由 **CPU 发出**，主存或 I/O 设备只负责按照这个地址被选中，而不需要再把地址反向送回 CPU，所以地址线通常是**单向**的。
   - **数据线**用来传送“真正的数据内容”。CPU 既可能把数据**写入**主存或 I/O 设备，也可能从主存或 I/O 设备中**读出**数据，因此数据既可能从 CPU 流向外部部件，也可能从外部部件流向 CPU，所以数据线必须是**双向**的。
   - **控制线**用来传送“操作命令和状态信息”。一方面，CPU 会发出读命令、写命令、中断响应等控制信号；另一方面，主存和 I/O 设备也可能向 CPU 返回完成信号、中断请求、准备就绪等状态信息，所以控制线一般也是**双向**的。

   可以简单记成：
   - **地址线传“访问目标”**，所以多为 CPU 单向发出；
   - **数据线传“数据本身”**，所以读写都要走；
   - **控制线传“命令与应答”**，所以双方都可能发送。

   :::

### 3.3：常用的总线结构有几种？不同的总线结构对计算机的性能有什么影响？举例说明。

常用的总线结构有**单总线结构**、**双总线结构**、**三总线结构**、**四总线结构**。

1. **单总线结构**：将 CPU、主存、I/O 设备都挂在一组总线上，允许 I/O 之间或 I/O 与主存之间直接交换信息。因为所有的传送都通过这组共享总线，极易形成计算机系统的瓶颈，不允许两个以上部件在同一时刻向总线传输信息，必然会影响系统工作效率的提高，ISA、EISA 总线是单总线结构。

   ![单总线结构](https://static.owo.cab/notes/image/cs/poco/chapter03/单总线结构.drawio.svg)

2. **双总线结构**：双总线结构的特点是将速度较低的设备从总线上分离出来，形成主存总线与 I/O 总线分开的结构。通道是一个具有特殊功能的处理器，CPU 将一部分功能下放给通道，使其对 I/O 设备具有统一管理的功能，系统的吞吐能力可以相当大。如果将不同速率的 I/O 设备进行分类，然后将它们连接在不同的通道上，将成为总线结构。

   ![双总线结构](https://static.owo.cab/notes/image/cs/poco/chapter03/双总线结构.drawio.svg)

3. **三总线结构**：
   - 主存总线用于 CPU 与主存间的传输；I/O 总线供 CPU 与各类 I/O 之间传递信息，DMA 总线用于高速外设（磁盘，磁带等）与主存之间直接交换信息。

     ![三总线结构-1](https://static.owo.cab/notes/image/cs/poco/chapter03/三总线结构-1.drawio.svg)

   - 处理器于高速缓冲存储器 Cache 之间增加一条局部总线，Cache 可以通过系统总线与主存传输信息，I/O 与主存之间也不必通过 CPU。还有一条扩展总线，可以支持相当多的 I/O 设备。

     ![三总线结构-2](https://static.owo.cab/notes/image/cs/poco/chapter03/三总线结构-2.drawio.svg)

   ::: info 两种三总线结构的差异

   这两种三总线结构虽然都叫“三总线”，但第三条总线承担的任务并不一样，系统组织的重点也不同。
   - **第一种三总线结构**的重点是把**高速外设与主存之间的数据交换**单独分离出来。除主存总线、I/O 总线外，再增加一条 **DMA 总线**，使高速 I/O 设备可以直接与主存交换数据，减少对 CPU 的占用。
   - **第二种三总线结构**的重点是按**速度层次**来组织系统。CPU 与 Cache 之间增设一条 **局部总线**，以满足高速部件之间的快速通信；同时在系统总线之外再设置 **扩展总线**，用于连接更多、速度较低的外部设备。

   可以简单理解为：前一种更强调 **DMA 直接传输**，后一种更强调 **高速部件与低速设备分层连接**。

   :::

4. **四总线结构**：在三总线基础上，增加一条与计算机系统机密相连的高速总线，这种结构对高速设备而言，其自身的工作可以很少依赖处理器，同时它们又比扩展总线上的设备更贴近处理器。

   ![四总线结构](https://static.owo.cab/notes/image/cs/poco/chapter03/四总线结构.drawio.svg)

### 3.4：为什么要设置总线判优控制？常见的集中式总线控制有几种，各有何特点，哪种方式响应时间最快，哪种方式对电路故障最敏感？

1. 如**多个主设备同时要使用总线时**，就由总线控制器的判优、仲裁逻辑按一定的优先级顺序，确定那个主设备能使用总线，只有获得总线使用权的主设备才能开始传送数据。
2. 常见的集中式总线控制有三种：
   1. **链式查询**：其特点是，只需**很少几根线**就能按一定优先次序实现总线控制，并且**很容易扩充设备**，但**对电路故障很敏感**。
   2. **计数器定时查询**：其特点是，**计数可以从“0”开始**，此时设备的优先次序是固定的，计数**也可以从终止点开始**，即是一种循环方式，此时设备使用总线的优先级相等，**计数器的初始值还可以由程序设置**，故优先次序可以改变。此外，对电路故障**不如**链式查询方式**敏感**，但**增加了主控线数**，**控制也较复杂**。
   3. **独立请求方式**：其特点是，**响应速度快**，**优先次序控制灵活**（通过程序改变），但**控制线数量多**，**总线控制更复杂**。

   **独立请求方式**响应时间**最快**，**链式查询**对电路故障**最敏感**。

   ::: details 细说链式查询、计数器定时查询和独立请求方式

   这三种方式都属于**集中式仲裁**，共同点是：总线使用权最终都由**中央仲裁器**决定。它们的主要区别，不在于“是否要仲裁”，而在于**仲裁信号怎样分配到各设备**、**优先级怎样确定**。
   - **链式查询**

     当多个设备同时发出总线请求时，仲裁器先发出一个总线允许信号，这个允许信号按照预先连接好的链路，依次经过各个设备。离仲裁器最近的设备先检查自己是否提出了请求：
     - 如果它**提出了请求**，就立即截获这个允许信号，并取得总线使用权，后面的设备就收不到允许信号。
     - 如果它**没有提出请求**，就把允许信号继续传给下一个设备。

     因此，链上的先后次序本身就决定了优先级，**越靠前的设备优先级越高**。这种方法只需要较少的控制线，结构也直观，所以容易扩充。

     它对电路故障最敏感，原因也正是在这条“链”上：如果某个设备或某段连线发生故障，导致允许信号不能继续往后传，那么它后面的所有设备都可能再也得不到总线使用权。也就是说，前面一个环节出问题，可能连带影响后面一串设备。

   - **计数器定时查询**

     这种方式下，仲裁器内部有一个计数器。计数器的值依次对应各个设备的编号，仲裁器按顺序查询“当前编号对应的设备是否正在申请总线”。
     - 如果当前被查询到的设备**没有请求**，计数器继续加一，查询下一个设备。
     - 如果当前被查询到的设备**有请求**，它就获得总线使用权。

     因此，它的本质是“**按编号轮流点名**”。如果计数总是从 `0` 开始，那么编号越靠前的设备越容易先被查询到，优先级就是固定的；如果每次从上次停止的位置继续计数，就形成循环查询，各设备获得总线的机会更均衡；如果初值可由程序设定，优先级还可以按需要调整。

     它对电路故障不如链式查询那样敏感，是因为仲裁器并不依赖一条设备与设备之间逐级传递的许可链路；某个设备故障，通常不会像链式查询那样把其后所有设备都“卡死”。但它的缺点是控制逻辑比链式查询更复杂，而且为了编号、查询和识别，往往需要更多控制线。

   - **独立请求方式**

     这种方式中，每个设备都有自己独立的**请求线**，通常也有自己独立的**允许线**与仲裁器相连。某个设备要用总线时，就通过自己的请求线单独向仲裁器申请；仲裁器根据预定优先级或程序设定规则，直接把允许信号发给某个设备。

     它响应时间最快，原因是仲裁器可以**直接看到所有设备的请求**，不需要像链式查询那样一级一级往下传允许信号，也不需要像计数器定时查询那样一个编号一个编号地轮流查询，所以一旦发生竞争，仲裁器能较快作出决定。

     它的优先级控制也最灵活，因为仲裁器可以通过硬件或程序直接改变判优规则。但代价也很明显：设备越多，请求线和允许线就越多，布线复杂，硬件成本也更高。

   **三种方式可以这样对比**：
   - **链式查询**：结构最简单、控制线最少，但优先级固定，而且由于许可信号逐级传递，**对电路故障最敏感**。
   - **计数器定时查询**：复杂度和性能介于两者之间，优先级可固定、可循环、也可编程调整，对故障的敏感程度**低于链式查询**。
   - **独立请求方式**：仲裁最快、优先级控制最灵活，因此**响应时间最快**，但控制线最多、硬件实现最复杂。

   :::

### 3.5：解释概念：总线宽度、总线带宽、总线复用、总线的主设备（或主模块）、总线的从设备（或从模块）、总线的传输周期、总线的通信控制。

- **总线宽度**：通常指**数据总线的根数**；
- **总线带宽**：**总线的数据传输率**，指**单位时间内**总线上传输数据的**位数**；
- **总线复用**：指**同一条信号线可以分时传输不同的信号**；
- **总线的主设备（主模块）**：指一次总线传输期间，**拥有总线控制权的设备（模块）**；
- **总线的从设备（从模块）**：指一次总线传输期间，**配合主设备完成数据传输的设备（模块）**，它只能**被动**接受主设备发来的命令；
- **总线的传输周期**：指**总线完成一次完整而可靠的传输所需时间**；
- **总线的通信控制**：指**总线传送过程中双方的时间配合方式**。

### 3.6：试比较同步通信和异步通信。

**同步通信**：指**由统一时钟控制的通信**，**控制方式简单**，**灵活性差**，当系统中各部件工作速度差异较大时，总线工作效率明显下降。**适合于速度差别不大的场合**。

**异步通信**：指**没有统一时钟控制的通信**，部件间采用**应答方式**进行联系，**控制方式**较同步**复杂**，**灵活性高**，当系统中**各部件工作速度差异较大**时，有利于提高总线工作效率。

:::: details 细说同步通信和异步通信

这两种通信方式的核心区别，不在于“传的数据是什么”，而在于**发送方和接收方靠什么来配合时间**。

**同步通信**依靠**统一时钟**来约定双方在什么时候发送、什么时候接收。可以理解为：通信双方都跟着同一个节拍工作，时钟一到，就按预定时序完成地址、命令和数据的传送。因此它的优点是时序规则明确，控制电路较简单，传送过程也容易组织；但缺点是各部件必须尽量跟上同一个节拍，如果某些部件速度较慢，为了等它们，整个系统往往只能按较保守的速度工作，所以当各部件速度差异较大时，效率会受到影响。

**异步通信**没有统一的公共时钟来强制规定每一步发生的准确时刻，而是靠**请求、回答、应答**这类 [**联络信号**](#联络信号) 来协调。发送方准备好后先发出请求，接收方准备好后再给出回答，双方根据这些握手信号完成一次传送。因此它的优点是灵活，能较好适应速度差异较大的部件；慢设备不会硬拖着快设备长期空等。缺点是控制过程比同步通信复杂，因为每次传送都要处理双方的联络与确认。

::: details 联络信号<span id="联络信号"></span>

在异步通信中，由于没有统一时钟来规定“现在该谁发、谁收”，所以双方必须借助一些专门的控制信号来互相通知状态，这些信号就叫**联络信号**。

最常见的联络信号就是**请求**和**回答**：

- **请求信号**通常由**发送方**发出，用来表示“我已经准备好，可以开始这次数据传送了”。
- **回答信号**通常由**接收方**发出，用来表示“我已经收到请求，或我已经准备好接收/已经接收完毕，可以继续下一步了”。

也就是说，异步通信不是靠统一时钟的节拍来配合，而是靠“**我准备好了**”和“**我知道了/我也准备好了**”这样的联络过程来推进。

因此，后面讲的**不互锁、半互锁和全互锁**，本质上都是在讨论：发送方和接收方在使用这些联络信号时，**彼此需要等待到什么程度**、**撤销信号时要不要再看对方的状态**。

:::

可以从适用场景来理解它们的差别：

- **同步通信**适合各部件速度比较接近、系统时序容易统一的场合。
- **异步通信**适合各部件速度差别较大、需要靠应答机制协调配合的场合。

还需要掌握的通信方式有，至少在本章里还有两类：

- **异步通信中的三种互锁关系**：不互锁、半互锁、全互锁。这部分对应后面的 [3.7](#3-7)，属于异步通信内部更细的配合方式。
- **半同步通信**：它既保留了同步通信“有统一时钟”的特点，又增加了等待响应信号来适应慢设备，对应后面的 [3.8](#3-8)。可以把它看成介于同步通信和异步通信之间的一种折中方式。

如果按考试复习的角度，本章关于通信控制，通常至少要掌握这三类：**同步通信、异步通信、半同步通信**；其中异步通信还要进一步掌握**不互锁、半互锁、全互锁**三种应答关系。

::::

### 3.7<span id="3-7"></span>：画图说明异步通信中请求与回答有哪几种互锁关系。

异步通信的应答方式可分为不互锁、半互锁和全互锁三种，如下图。

![异步通信中请求与回答的互锁](https://static.owo.cab/notes/image/cs/poco/chapter03/异步通信中请求与回答的互锁.drawio.svg)

::: details 不互锁、半互锁和全互锁

这里的“互锁”可以理解为：**发送方和接收方在撤销各自信号时，是否还要等待对方的信号变化**。图中的“请求”一般由发送方发出，“回答”由接收方发出，三种方式的区别主要就在于双方配合得有多紧。

- **不互锁**：发送方发出请求后，接收方收到请求便给出回答；随后，发送方**不必等待接收方撤销回答**，到时间就可以撤销自己的请求；接收方也不必等待发送方再作进一步确认，在完成应答后即可撤销回答。也就是说，双方只是完成了“请求到回答”这一层联系，后续撤销动作彼此约束较少，所以控制最简单，但配合也最松，对时序条件要求较严格。
- **半互锁**：发送方发出请求后，接收方收到请求再发出回答；这时发送方在撤销请求之前，要先**看到接收方已经给出回答**，也就是发送方要等对方一下；但接收方撤销回答时，不再要求继续等待发送方的进一步动作。因此它比不互锁更可靠，因为发送方至少能确认“对方确实收到了请求”，但约束还没有全互锁那么强。
- **全互锁**：发送方发出请求，接收方收到后给出回答；发送方只有在**收到回答后**才能撤销请求，而接收方又只有在**看到请求已经撤销后**，才撤销自己的回答。也就是说，请求的撤销要等回答，回答的撤销又要等请求的撤销，双方前后都互相制约，配合最严密，因此可靠性最高，对速度差异的适应性也最好，但控制过程最复杂、一次传送所花的时间也相对更长。

可以顺着图来记：

- **不互锁**：只在“发出请求后引出回答”这一点上发生联系，后面撤销信号基本各自进行。
- **半互锁**：发送方撤销请求前，要先等到回答出现。
- **全互锁**：发送方等回答，接收方还要再等请求撤销，双方首尾都要互相确认。

因此，这三种方式在一般理解上可以看成是：**不互锁最快但最松，半互锁居中，全互锁最稳但最慢**。

:::

### 3.8<span id="3-8"></span>：为什么说半同步通信同时保留了同步通信和异步通信的特点？

半同步通信既有**统一时钟控制**这一同步通信的特点，又通过增设 **WAIT** 信号、插入等待周期，允许不同速度的部件协调工作，因而兼有异步通信的灵活性。

因此，半同步通信可以看成是**同步通信与异步通信的折中方式**，其工作效率和控制复杂度一般也介于两者之间。

::: details 半同步通信

半同步通信的关键点在于：**整体上仍按统一时钟工作，但当某一方来不及完成操作时，允许临时插入等待周期**。因此它既不像纯同步通信那样要求所有部件都严格跟上统一节拍，也不像纯异步通信那样每一步都完全依赖请求、回答来握手。

它为什么同时具有同步和异步两种特点，可以分别从两方面理解：

- 从**同步**的一面看，地址、命令、数据信号的发出和识别，仍然以系统时钟为基准，整个系统在大框架上依旧是按统一节拍运行的。
- 从**异步**的一面看，当接收方或执行部件速度较慢、暂时来不及完成操作时，可以通过 **WAIT** 响应信号通知发送方“再等一会儿”，系统于是插入若干等待周期，等慢设备准备好后再继续传送。

也就是说，半同步通信并没有放弃时钟控制，而是在时钟控制之上增加了一种“**必要时允许等待**”的协调机制。

这种方式的优点是：

- 比纯异步通信的控制方式更简单，因为系统仍有统一时钟作为基本依据；
- 比纯同步通信更灵活，因为它允许不同速度的部件通过等待周期来配合；
- 在统一时钟下工作，整体可靠性和实现上的规整性通常较好。

它的缺点是也很明显：

- 为了照顾慢设备，需要插入等待周期，因此系统时钟频率不能一味提高；
- 从整体性能看，它的速度通常达不到理想的高速同步传送；
- 控制方式虽然比异步通信简单，但又比纯同步通信更复杂。

因此，半同步通信通常适用于**系统速度要求不是特别高，但系统中各部件速度差异又比较明显**的场合。可以把它理解成：在“统一时钟”这个大前提下，对慢设备作出有限让步，从而换取更好的适应性。

:::

### 3.10：什么是总线标准？为什么要设置总线标准？目前流行的总线标准有哪些？什么是即插即用，哪些总线有这一特点？

1. 所谓总线标准，可视为**系统与各模块、模块与模块之间的一个互连的标准界面**。这个界面对它两端的模块都是透明的，即界面的任一方只需根据总线标准的要求完成自身一方接口的功能要求，而无须了解对方接口与总线的连接要求。因此，按总线标准设计的接口可视为通用接口。
2. **采用总线标准可以为计算机接口的软硬件设计提供方便**。对硬件设计而言，使**各个模块的接口芯片设计相对独立**；对软件设计而言，更**有利于接口软件的模块化设计**。
3. 目前流行的总线标准有以下几种：
   - **ISA 总线**；
   - **EISA 总线**；
   - **VESA（VL-BUS）总线**；
   - **PCI 总线**；
   - **AGP 总线**；
   - **RS-232C 总线**；
   - **USB 总线**。

   ::: details 流行的总线标准

   这些总线标准并不都属于同一层次，有的主要用于早期微机内部扩展，有的用于图形显示，有的用于外部设备通信。掌握时可先抓住它们各自的主要用途：
   - **ISA 总线**：早期 PC 中常见的**系统扩展总线**，结构较简单，但速度较低，属于较早期的总线标准。
   - **EISA 总线**：可看作 ISA 的增强型标准，在兼容 ISA 的基础上提高了数据传输能力和系统扩展能力。
   - **VESA（VL-BUS）总线**：主要为了适应当时高速图形显示和高速外设的需要而提出，传输速度较 ISA、EISA 更高。
   - **PCI 总线**：一种应用非常广泛的局部总线标准，性能较好，支持即插即用，在很长一段时间内是微机系统中的主流扩展总线。
   - **AGP 总线**：主要面向**图形加速卡**设计，可看作针对显示系统提出的专用高速接口标准。
   - **RS-232C 总线**：一种常见的**串行通信标准**，常用于计算机与外部设备之间进行串行数据传送。
   - **USB 总线**：一种通用的外部设备连接标准，支持热插拔和即插即用，广泛用于键盘、鼠标、移动存储设备等外设连接。

   可以粗略地记成：**ISA、EISA、VESA、PCI、AGP** 更偏向计算机内部或扩展接口标准，**RS-232C、USB** 更偏向外部设备通信标准。

   :::

4. **即插即用（Plug and Play）**：即**任何扩展卡只要插入系统便可工作**。PCI 设备中配有存放设备具体信息的寄存器，这些信息可供 BIOS（基本输入输出系统）和操作系统层的软件自动配置 PCI 总线部件和插件，使系统使用方便，无须进行复杂的手动配置。**PCI、USB 等总线有即插即用的特点**。

### 3.13：什么是总线的数据传送速率，它与哪些因素有关？

**总线的数据传输速率**即**总线带宽**，指**单位时间内总线上传输数据的位数**，通常用**每秒**传输信息的**字节数**来衡量。

它与**总线宽度**和**总线频率**有关，总线宽度越**宽**，频率越**快**，数据传输率越**高**。

### 3.14：设总线的时钟频率为 8 MHz，一个总线周期等于一个时钟周期。如果一个总线周期中并行传送 16 位数据，试问总线的带宽是多少?

$f = 8\,\text{MHz}$，$T = 1/f = 1/(8 \,\text{M})$ 秒，一个总线周期即是一个时钟周期

所以，总线带宽$= 16/(1/(8\,\text{M})) = 128\,\text{Mbps}$

### 3.15：在一个 32 位的总线系统中，总线的时钟频率为 66 MHz，假设总线最短传输周期为 4 个时钟周期，试计算总线的最大数据传输率。若想提高数据传输率，可采取什么措施?

1. 总线传输周期 $= 4 \times 1/(66\,\text{M})$ 秒

   总线的最大数据传输率 $= 32/(4/(66\,\text{M})) = 528\,\text{Mbps}$

2. 若想提高数据传输率，可以**提高总线时钟频率**、**增大总线宽度**或者**减少总线传输周期包含的时钟周期个数**。

### 3.16：在异步串行传送系统中，字符格式为：1 个起始位、8 个数据位、1 个校验位、2 个终止位。若要求每秒传送 120 个字符，试求传送的波特率和比特率。

一帧包含：$1+8+1+2=12$ 位

故波特率为：$(1+8+1+2)\times 120 = 1440\,\text{bps}$

比特率为：$8\times 120 = 960\,\text{bps}$

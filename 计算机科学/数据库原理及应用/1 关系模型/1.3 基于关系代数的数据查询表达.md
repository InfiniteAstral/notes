# 1.3 基于关系代数的数据查询表达

## 环境说明

- Student (<u>sNo</u>, sName, sSex, sAge, sDept)
- Course (<u>cNo</u>, cName, cPNo, cCredit)
- SC (<u>sNo</u>, <u>cNo</u>, score)

## 数据查询表达

### 查询所有软件学院的学生信息

关系代数表达式：

$$\sigma_{\text{sDept}='\text{软件学院}'}(\text{Student})$$

### 查询所有年龄小于等于 20 岁的男学生信息

关系代数表达式：

$$\sigma_{\text{sAge}\leq 20 \land \text{sSex}='\text{M}'}(\text{Student})$$

### 查询所有软件学院年龄小于 20 岁的男学生姓名及年龄

关系代数表达式：

$$\prod_{\text{sName},\text{sAge}}(\sigma_{\text{sDept}='\text{软件学院}' \land \text{sAge}<20 \land \text{sSex}='\text{M}'}(\text{Student}))$$

### 查询 `001` 号学生所选择的所有课程号

关系代数表达式：

$$\prod_{\text{cNo}}(\sigma_{\text{sNo}='\text{001}'}(\text{SC}))$$

### 查询选修了 `001` 号课或者选修了 `002` 号课程的学生信息

关系代数表达式：

$$\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}' \lor \text{cNo}='\text{002}'}(\text{SC})) \Join \text{Student}$$

也可以写成：

$$\left(\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC})) \cup \prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{002}'}(\text{SC}))\right) \Join \text{Student}$$

### 查询选修了 `001` 号课并且选修了 `002` 号课程的学生信息

关系代数表达式：

$$\left(\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC})) \cap \prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{002}'}(\text{SC}))\right) \Join \text{Student}$$

也可以利用除法写成：

$$\left(\prod_{\text{sNo},\text{cNo}}(\text{SC}) \div \prod_{\text{cNo}}(\sigma_{\text{cNo}='\text{001}' \lor \text{cNo}='\text{002}'}(\text{Course}))\right) \Join \text{Student}$$

::: tip 注
不能直接写成 $\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}' \land \text{cNo}='\text{002}'}(\text{SC})) \Join \text{Student}$，因为同一个元组的 `cNo` 不可能同时等于 `001` 和 `002`。
:::

### 查询选修了 `001` 号课但没选修 `002` 号课程的学生信息

关系代数表达式：

$$\left(\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC})) - \prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{002}'}(\text{SC}))\right) \Join \text{Student}$$

::: tip 注
不能先对 `SC` 做差再投影，即不能写成 $\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC})-\sigma_{\text{cNo}='\text{002}'}(\text{SC})) \Join \text{Student}$。因为差运算是在整个元组上进行的，而不是仅仅针对学号进行比较。
:::

### 查询所有没选 `001` 号课程的学生信息

关系代数表达式：

$$\left(\prod_{\text{sNo}}(\text{Student}) - \prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC}))\right) \Join \text{Student}$$

::: tip 注
不能写成 $\prod_{\text{sNo}}(\sigma_{\text{cNo}\neq '\text{001}'}(\text{SC})) \Join \text{Student}$。因为某个学生只要选过其他课程，就会出现在该结果中，即使他同时也选了 `001` 号课程。
:::

### 查询仅仅选了 `001` 号课程的学生信息

关系代数表达式：

$$\left(\prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC})) - \prod_{\text{sNo}}(\sigma_{\text{cNo}\neq '\text{001}'}(\text{SC}))\right) \Join \text{Student}$$

也可以写成：

$$\left(\prod_{\text{sNo}}(\text{SC}) - \prod_{\text{sNo}}(\text{SC} - \sigma_{\text{cNo}='\text{001}'}(\text{SC}))\right) \Join \text{Student}$$

::: tip 注
“仅仅选了 `001` 号课程”表示该学生选过 `001`，并且没有选过除 `001` 以外的任何课程，因此它比“选了 `001` 但没选 `002`”的条件更强。
:::

### 查询仅仅没选 `001` 号课程的学生信息

关系代数表达式：

$$\left(\prod_{\text{sNo}}(\text{Student}) - \prod_{\text{sNo}}(\sigma_{\text{cNo}='\text{001}'}(\text{SC}))\right) \Join \text{Student}$$

::: tip 注
这里“仅仅没选 `001` 号课程”可以理解为“没有选修 `001` 号课程”，并不要求学生必须选修其他课程。因此，没有任何选课记录的学生也应包含在结果中。
:::

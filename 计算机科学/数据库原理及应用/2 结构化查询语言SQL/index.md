# 第 2 章 结构化查询语言SQL

**SQL 简介**：

1. **定义**：结构化查询语言（Structured Query Language，SQL）是访问关系数据库的标准语言。
2. **核心目标**：使用户能够建立数据库和关系结构，完成基本的数据管理任务，执行简单和复杂查询，并以尽可能少的用户操作实现这些功能。
3. **起源与发展**
   - 1970 年：E.F. Codd 提出关系数据模型，奠定了理论基础。
   - 1974 年：D. Chamberlin 定义了结构化英语查询语言（SEQUEL）。
   - 1976 年：修订为 SEQUEL/2，后来更名为 SQL。
   - 20 世纪 70 年代后期：IBM 开发了基于 SEQUEL/2 的 System R；第一个商业化关系数据库管理系统 Oracle 发布。
   - 1982 至 1983 年：IBM 推出 SQL/DS 和 DB2。
   - 标准化过程：SQL-86 → SQL-89 → SQL-92（SQL2）→ SQL-99（SQL3）→ SQL-2003 → SQL-2008 → SQL-2011。
4. **核心特点**：非过程化语言（以关系代数和关系演算为基础），易学易用。
5. **核心功能及对应动词**

   | SQL 功能 | 对应动词                     |
   | -------- | ---------------------------- |
   | 数据查询 | `SELECT`                     |
   | 数据定义 | `CREATE`, `DROP`, `ALTER`    |
   | 数据操纵 | `INSERT`, `UPDATE`, `DELETE` |
   | 数据控制 | `GRANT`, `REVOKE`            |

6. **应用方式**：交互式方式（直接在终端输入语句）、嵌入式方式（嵌入到过程化程序设计语言中）。

# 2.3 SQL应用

## SQL 的局限性

SQL是面向数据查询和操作的**声明式语言**，擅长数据存储与检索，但存在以下固有局限：

- 不适合**复杂的数值计算、逻辑处理和业务流程编排**
- 无法直接开发专用**用户界面**（如图形化界面、Web界面）
- 原生不支持同时访问**多个异构数据库**

## 解决方案

为弥补 SQL 的不足，数据库应用通常采用以下**三种模式**：

1. **嵌入式 SQL**：将 SQL 语句直接嵌入通用程序设计语言（如 C、Java），SQL 负责数据操作，宿主语言负责业务逻辑和界面
2. **应用编程接口（API）**：通过标准库函数与数据库交互，无需预编译 SQL
3. **混合语言**：设计同时支持 SQL 和通用程序设计语言特性的新语言

## 阻抗失配（Impedance Mismatch）

数据库关系模型与程序设计语言的对象/结构模型之间的差异导致的不匹配问题，是程序式 SQL 的**核心挑战**：

- 数据类型不兼容：数据库的 `NULL`、日期、大对象等类型与程序语言类型映射困难
- **结果集匹配问题**：SQL 返回的二维结果集与程序语言的对象、集合结构不匹配
- **遍历与处理问题**：需要手动遍历结果集并转换为程序数据结构
- **内存管理差异**：数据库数据持久化存储，程序数据在内存中临时存在

## 嵌入式 SQL（Embedded SQL）

将 SQL 语句直接嵌入**宿主语言**（如 C、C++）源代码中混合编写，需经过**预编译处理**。

### 核心特点

- **预编译阶段**将 SQL 语句转换为宿主语言可识别的函数调用
- 通过通信区（**SQLCA**）传递执行状态和错误信息（`SQLCODE`、`SQLSTATE`）

### 静态嵌入式 SQL

- 数据库访问模式在**编译时固定**，硬编码到程序中
- **不允许**用宿主变量代替表名、列名等数据库对象名
- 编译时确定：保留字、宿主变量数量与类型、访问的数据库对象

### 动态嵌入式 SQL

- 数据库访问模式在**运行时确定**，SQL 语句可动态生成
- 将完整 SQL 语句存入宿主变量，传递给 DBMS 执行
- **灵活性更高**，适合动态查询场景

### 静态与动态嵌入式 SQL 对比

| 特性     | 静态嵌入式 SQL | 动态嵌入式 SQL     |
| :------- | :------------- | :----------------- |
| 执行时机 | 编译时确定     | 运行时确定         |
| 灵活性   | 低，模式固定   | 高，支持动态 SQL   |
| 性能     | 高，预编译优化 | 低，运行时解析优化 |
| 适用场景 | 固定业务逻辑   | 动态查询、通用工具 |

### 执行流程

- **静态 SQL**：预编译 → 宿主语言编译 → 绑定 → 执行
- **动态 SQL**：运行时生成 SQL → 解析 → 验证 → 优化 → 生成执行计划 → 执行

::: info 例

1. 静态嵌入式 SQL（C 语言）

   ```c
   EXEC SQL BEGIN DECLARE SECTION;
   float increment;
   EXEC SQL END DECLARE SECTION;

   // 静态更新员工工资
   EXEC SQL UPDATE Staff
   SET salary = salary + :increment
   WHERE staffNo = 'SL21';
   ```

2. 动态嵌入式 SQL（C 语言）

   ```c
   EXEC SQL BEGIN DECLARE SECTION;
   char buffer[100];
   EXEC SQL END DECLARE SECTION;

   // 动态生成并执行SQL
   sprintf(buffer, "UPDATE Staff SET salary = salary + %f WHERE staffNo = 'SL21'", increment);
   EXEC SQL EXECUTE IMMEDIATE :buffer;
   ```

:::

## 应用编程接口（API）

提供标准函数库供程序语言调用以访问数据库，无需预编译 SQL，是目前**最常用**的数据库交互方式。

### 主流标准

- **ODBC（Open Database Connectivity）**：跨语言、跨数据库的通用接口标准
- **JDBC（Java Database Connectivity）**：Java 语言专属的数据库 API，属于`java.sql`包

### JDBC查询步骤

1. **加载 JDBC 驱动**
2. **建立数据库连接**（`Connection`）
3. **创建 `Statement` 对象**执行 SQL 语句
4. **处理 `ResultSet` 结果集**
5. **关闭资源**（`ResultSet`、`Statement`、`Connection`）

::: info 例
JDBC 查询学生信息示例（Java）

```java
import java.sql.*;

public class JdbcExample {

    public static void main(String[] args) {
        Connection conn = null;
        Statement stmt = null;
        ResultSet rs = null;

        try {
            // 1. 加载驱动
            Class.forName("org.postgresql.Driver");

            // 2. 建立连接
            conn = DriverManager.getConnection(
                    "jdbc:postgresql://localhost:5432/school",
                    "username",
                    "password"
            );

            // 3. 执行SQL
            stmt = conn.createStatement();
            rs = stmt.executeQuery("SELECT sNo, sName FROM Student WHERE dNo = '01'");

            // 4. 处理结果集
            while (rs.next()) {
                String sNo = rs.getString("sNo");
                String sName = rs.getString("sName");
                System.out.println(sNo + " - " + sName);
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            // 5. 关闭资源
            try {
                if (rs != null) {
                    rs.close();
                }
            } catch (SQLException e) {
            }
            try {
                if (stmt != null) {
                    stmt.close();
                }
            } catch (SQLException e) {
            }
            try {
                if (conn != null) {
                    conn.close();
                }
            } catch (SQLException e) {
            }
        }
    }
}

```

:::

## 过程语言/SQL

结合过程化语言的**控制结构**（分支、循环、变量）与 SQL 的数据操作能力，扩展 SQL 的表达能力。

- 主要用于编写**存储过程、触发器、函数**等数据库端程序
- 不同数据库有各自的实现：PostgreSQL 的 **PL/pgSQL**、Oracle 的 **PL/SQL**、MySQL 的 **SQL/PSM**

::: info 例
PL/pgSQL编写的加权平均分计算函数

```sql
CREATE OR REPLACE FUNCTION weighted_AVG(IN sNoIn student.sNo%TYPE)
RETURNS NUMERIC AS $weighted_AVG$
DECLARE
  weightedScore NUMERIC;
BEGIN
  SELECT CAST(SUM(CAST(score * credit AS NUMERIC)) / SUM(credit) AS NUMERIC)
  INTO weightedScore
  FROM sc, course c
  WHERE sc.cNo = c.cNo
    AND score IS NOT NULL
    AND credit IS NOT NULL
    AND sc.sNo = sNoIn
  GROUP BY sc.sNo;

  RETURN weightedScore;
END;
$weighted_AVG$ LANGUAGE plpgsql;
```

:::

## MVC架构（Model-View-Controller）

数据库应用最常用的**分层架构**，将应用职责分离为三个独立部分，降低耦合度。

### 核心组件

1. **模型（Model）**：封装数据和业务逻辑，负责与数据库交互（增删改查）
2. **视图（View）**：负责用户界面输出，展示模型数据
3. **控制器（Controller）**：接收用户输入，调用模型处理业务，选择视图展示结果

### 典型流程

```
用户请求 → 控制器 → 模型（JDBC访问数据库） → 控制器 → 视图 → 用户
```

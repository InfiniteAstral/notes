# 5.3 Neo4j 图数据库

## 图数据库简介

图数据库是专门用于存储和查询**高度关联数据**的数据库，它以图（节点、边和属性）的形式存储数据，非常适合处理复杂的关系查询。

- **性能优势**：在处理深度关系查询时，比关系型数据库快数千倍
- **典型应用**：社交网络、知识图谱、推荐系统、欺诈检测

## Neo4j 简介

`Neo4j` 是目前最流行的开源图数据库，遵循**属性图数据模型**。

### 属性图数据模型的基本元素

1. **节点（Node）**：表示实体，类似于关系型数据库的行
2. **关系（Relationship）**：表示节点之间的关联，总是有方向的
3. **属性（Property）**：节点和关系都可以有属性，是键值对
4. **标签（Label）**：用于对节点进行分组，类似于关系型数据库的表名

## Cypher 查询语言（CQL）

`Cypher` 是 `Neo4j` 的声明式查询语言，类似于 `SQL`，专门用于查询图数据。

::: info 常用 CQL 命令示例

```cypher
// 1. 创建节点
CREATE (n:Person { name: "刘备", age: 40 });
CREATE (n:Person { name: "关羽", age: 38 });

// 2. 创建关系
MATCH (a:Person { name: "刘备" }), (b:Person { name: "关羽" })
CREATE (a)-[:结义兄弟 { 排行: "大哥" }]->(b);

// 3. 查询节点和关系
// 查询所有Person节点
MATCH (n:Person) RETURN n;
// 查询刘备的所有结义兄弟
MATCH (a:Person { name: "刘备" })-[:结义兄弟]->(b)
RETURN b.name;

// 4. 更新节点属性
MATCH (n:Person { name: "刘备" })
SET n.nickname = "玄德";

// 5. 删除节点和关系
MATCH (n:Person { name: "刘备" })-[r]-()
DELETE n, r;
```

:::

## Neo4j 应用实例

1. **社交网络分析**：查找用户的朋友、朋友的朋友，计算用户影响力
2. **知识图谱**：构建领域知识图谱，实现智能问答和推理
3. **推荐系统**：基于用户的兴趣和行为，推荐相似的物品或用户
4. **欺诈检测**：分析交易和用户之间的关系，识别异常模式和欺诈行为

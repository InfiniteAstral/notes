# 5.2 MongoDB 文档数据库

## NoSQL 简介

**NoSQL（Not Only SQL）**是一类**非关系型数据库**的总称，主要用于解决传统关系型数据库在**海量数据、高并发和高可扩展性**方面的不足。

### NoSQL 的四大类型

| 类型                                                     | 代表产品  | 数据模型 | 典型应用           |
| -------------------------------------------------------- | --------- | -------- | ------------------ |
| **<span style="white-space: nowrap;">键值数据库</span>** | Redis     | 键值对   | 缓存、会话存储     |
| **列族数据库**                                           | Cassandra | 列族     | 分布式数据存储     |
| **文档数据库**                                           | MongoDB   | BSON文档 | Web应用、内容管理  |
| **图数据库**                                             | Neo4j     | 图结构   | 社交网络、知识图谱 |

## MongoDB 简介

`MongoDB` 是一个基于 **BSON（二进制 JSON）** 文档模型的 `NoSQL` 数据库，是目前最流行的文档数据库之一。

### MongoDB 与关系型数据库的对比

| <span style="white-space: nowrap;">关系型数据库</span> | MongoDB                                                               |
| ------------------------------------------------------ | --------------------------------------------------------------------- |
| 数据库（Database）                                     | 数据库（Database）                                                    |
| 表（Table）                                            | 集合（Collection）                                                    |
| 行（Row）                                              | 文档（Document）                                                      |
| 列（Column）                                           | 字段（Field）                                                         |
| 主键（Primary Key）                                    | `_id` 字段（默认自动生成）                                            |
| 表连接（Join）                                         | 通常不采用传统表连接，必要时可通过聚合管道中的 `$lookup` 实现关联查询 |

## MongoDB 基本概念

1. **文档**：`MongoDB` 的基本数据单元，是一组键值对，类似于 `JSON` 对象
2. **集合**：一组文档的集合，类似于关系型数据库的表
3. **数据库**：一组集合的集合

## MongoDB 常用操作

```javascript
// 1. 创建/切换数据库
use testdb;

// 2. 创建集合
db.createCollection("students");

// 3. 插入文档
db.students.insertOne({
  name: "张三",
  age: 20,
  gender: "男",
  major: "计算机科学"
});

// 4. 查询文档
// 查询所有文档
db.students.find();
// 查询年龄大于18的学生
db.students.find({ age: { $gt: 18 } });

// 5. 更新文档
db.students.updateOne(
  { name: "张三" },
  { $set: { age: 21 } }
);

// 6. 删除文档
db.students.deleteOne({ name: "张三" });

// 7. 创建索引
db.students.createIndex({ name: 1 });
```

## GridFS

`GridFS` 是 `MongoDB` 用于存储**大于 16 MB**的大文件的规范，它将大文件分割成多个 `256 KB` 的块（chunk）进行存储。

- 使用两个集合存储文件：`fs.files`（存储文件元数据）和 `fs.chunks`（存储文件块）
- 提供命令行工具 `mongofiles` 用于上传、下载和删除文件

## 复制与分片

1. **复制（Replication）**：将数据同步到多个服务器，提供**数据冗余**和**高可用性**
   - **副本集（Replica Set）**：由一个主节点和多个从节点组成，主节点处理写操作，从节点处理读操作
2. **分片（Sharding）**：将数据分散存储在多个服务器上，解决**海量数据存储**和**高并发访问**问题
   - **分片键（Shard Key）**：用于将数据分布到不同分片的字段

# 5.1 XML 数据传输格式

## XML 概述

**XML（可扩展标记语言，eXtensible Markup Language）**是一种用于**存储和传输数据**的标记语言。

- **设计目标**：传输和存储数据，而非显示数据
- **特点**：标签自定义、自我描述性、独立于软硬件平台
- **作用**：数据交换、配置文件、存储非结构化数据

## XML 基本语法

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!-- 这是一个XML注释 -->
<bookstore>
  <book category="CHILDREN">
    <title>Harry Potter</title>
    <author>J K. Rowling</author>
    <year>2005</year>
    <price>29.99</price>
  </book>
  <book category="WEB">
    <title>Learning XML</title>
    <author>Erik T. Ray</author>
    <year>2003</year>
    <price>39.95</price>
  </book>
</bookstore>
```

### 语法规则

1. 所有 `XML` 元素必须有闭合标签
2. `XML` 标签对大小写敏感
3. `XML` 必须正确嵌套
4. `XML` 文档必须有且只有一个根元素
5. `XML` 属性值必须加引号

### 实体引用

用于表示 `XML` 中的特殊字符：

| 实体引用 | 字符 | 含义   |
| -------- | ---- | ------ |
| `&lt;`   | `<`  | 小于号 |
| `&gt;`   | `>`  | 大于号 |
| `&amp;`  | `&`  | 和号   |
| `&apos;` | `'`  | 单引号 |
| `&quot;` | `"`  | 双引号 |

### CDATA 段

`CDATA` 段中的内容不会被 `XML` 解析器解析，用于包含大量特殊字符：

```xml
<![CDATA[
  这里的内容不会被解析：< > & " '
]]>
```

## 良构与有效的 XML

1. **良构的 XML**：符合 `XML` 基本语法规则的 `XML` 文档
2. **有效的 XML**：不仅良构，而且符合文档类型定义（`DTD`）或 `XML Schema`（`XSD`）的约束

## XML 解析技术

| 解析技术  | 特点                                    | 适用场景                            |
| --------- | --------------------------------------- | ----------------------------------- |
| **DOM**   | 基于树结构，将整个 `XML` 文档加载到内存 | 多次访问 `XML`、需要修改 `XML` 结构 |
| **SAX**   | 基于事件驱动，逐行解析，不加载整个文档  | 大 `XML` 文件、只需要读取部分内容   |
| **DOM4J** | 开源、高性能、易用的 `Java XML API`     | `Java` 开发中最常用                 |

## XPath 语言

`XPath` 是用于在 `XML` 文档中查找信息的语言，使用路径表达式来选取 `XML` 文档中的节点或节点集。

::: info 常用 XPath 表达式示例

- `/bookstore/book/title`：选取所有 `book` 元素下的 `title` 元素
- `//title`：选取文档中所有的 `title` 元素
- `//book[@category='WEB']`：选取 `category` 属性为 `WEB` 的所有 `book` 元素
- `//book[price>30]`：选取 `price` 大于 `30` 的所有 `book` 元素

:::
